# LangChain、LangGraph 与 DeepAgents：官方关系与选型指南

> **核心结论先行：**
>
> 1. **LangChain** 是用于构建 LLM 应用的通用框架。
> 2. **LangGraph** 是 LangChain 生态中用于编排有状态 Agent 的运行时。
> 3. **DeepAgents** **不是**一个新框架，而是 **LangGraph 的一种高级应用模式/参考实现**。

---

## 一、定位总览

| 项目           | 官方定位         | 抽象层级 | 独立性                    |
| :------------- | :--------------- | :------- | :------------------------ |
| **LangChain**  | LLM 应用框架     | 中高层   | ✅ 独立框架               |
| **LangGraph**  | Agent 编排运行时 | 底层     | ✅ LangChain 生态核心组件 |
| **DeepAgents** | 高级 Agent 模式  | 顶层封装 | ❌ **依附于 LangGraph**   |

---

## 二、官方架构关系

DeepAgents 建立在 LangGraph 之上，而 LangGraph 通常与 LangChain 集成使用。

```
┌──────────────────────────────────────────────────┐
│ 应用层 (Your Application) │
├──────────────────────────────────────────────────┤
│ │
│ ┌──────────────────────────────┐ │
│ │ LangChain │ │
│ │ （模型 / 工具 / 记忆 / RAG） │ │
│ └──────────────┬───────────────┘ │
│ │ │
│ ▼ │
│ ┌──────────────────────────────┐ │
│ │ LangGraph │ │
│ │ （图执行 / 状态 / 持久化） │ │
│ │ │ │
│ │ • 节点 (Nodes) │ │
│ │ • 边 (Edges) │ │
│ │ • 检查点 (Checkpoints) │ │
│ │ • 人在回路 (HITL) │ │
│ └──────────────┬───────────────┘ │
│ │ │
│ ▼ │
│ ┌──────────────────────────────┐ │
│ │ DeepAgents │ │
│ │ （预置图 + 规划 + 子 Agent） │ │
│ │ │ │
│ │ • 自主规划 (Planning) │ │
│ │ • 任务分解 (Sub-agents) │ │
│ │ • 长周期执行 │ │
│ └──────────────────────────────┘ │
│ │
└──────────────────────────────────────────────────┘
```

## 三、详细区别对比

| 功能维度     | LangChain               | LangGraph                  | DeepAgents              |
| :----------- | :---------------------- | :------------------------- | :---------------------- |
| **核心职责** | 提供组件抽象            | 提供执行引擎               | 提供最佳实践模板        |
| **状态管理** | 基础记忆 (Buffer)       | **强状态 (Stateful)**      | 强状态 (继承 LangGraph) |
| **执行模式** | 线性 / 简单循环         | **图结构 (DAG/Cycle)**     | 复杂图结构              |
| **人在回路** | 手动实现                | **原生支持 (Interrupt)**   | 原生支持                |
| **持久化**   | 依赖外部存储            | **内置 Checkpointer**      | 内置                    |
| **适用场景** | RAG、简单问答、工具调用 | 复杂工作流、审批流、长任务 | 深度研究、复杂代码生成  |

## 四、如何选择？

### ✅ 使用 LangChain

- 你需要快速构建一个 **RAG 系统**、聊天机器人或简单的工具调用应用。
- 你不需要复杂的循环或状态回溯。

### ✅ 使用 LangGraph

- 你的 Agent 需要 **多步决策**、**分支逻辑**或**循环执行**。
- 你需要 **生产级特性**：断点续传、状态恢复、人工干预。
- 你需要精确控制 Agent 的执行流。

### ✅ 使用 DeepAgents（模式）

- 你想要一个 **“开箱即用”的强自主性 Agent**。
- 你的任务涉及 **深度研究、多文件处理、复杂规划**。
- 你希望直接复用 LangGraph 的最佳实践，而不想从零设计图结构。

---

## 五、常见误区澄清

❌ **误区**：DeepAgents 是 LangChain 的下一代替代品。  
✅ **正解**：DeepAgents 是基于 LangGraph 构建的**高级封装**，LangChain 依然是所有应用的基础框架。

❌ **误区**：三者必须选其一。  
✅ **正解**：它们是**互补的**。你用 LangChain 定义模型和工具，用 LangGraph 编排它们，而 DeepAgents 则是这种编排的一个高级范例。
