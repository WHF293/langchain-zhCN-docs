# 第十五章 解释器（Interpreter）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

解释器在 QuickJS 运行时中执行 JavaScript 代码，提供轻量级、安全的代码执行环境。它支持编程式工具调用（Programmatic Tool Calls, PTC）、编程式子代理以及有状态工作流，是 Deep Agents 自动化能力的核心组件。

## QuickJS 运行时

QuickJS 是一个轻量级的 JavaScript 引擎，具有以下特点：

| 特性 | 说明 |
|:-----|:-----|
| **轻量级** | 内存占用小，启动速度快 |
| **ES2020 支持** | 支持现代 JavaScript 语法 |
| **沙箱化** | 无文件系统和网络访问（默认） |
| **确定性执行** | 相同输入产生相同输出 |
| **低延迟** | 适合高频调用场景 |

### 基本使用

```typescript
import { Interpreter } from "deepagents";

const interpreter = new Interpreter({
  // 运行时配置
  runtime: "quickjs",
  // 超时时间（毫秒）
  timeout: 5000,
  // 内存限制（字节）
  memoryLimit: 1024 * 1024,
  // 允许的全局对象
  globals: ["console", "JSON", "Math", "Date"],
});

// 执行 JavaScript 代码
const result = await interpreter.run(`
  const numbers = [1, 2, 3, 4, 5];
  const sum = numbers.reduce((a, b) => a + b, 0);
  return { sum, average: sum / numbers.length };
`);
console.log(result); // { sum: 15, average: 3 }
```

## 编程式工具调用（PTC）

编程式工具调用允许在 JavaScript 代码中直接调用 Deep Agents 的工具，实现复杂的数据处理和工作流编排：

```typescript
import { Interpreter, PTC } from "deepagents";

const interpreter = new Interpreter({
  runtime: "quickjs",
  tools: {
    // 注册可用工具
    readFile: PTC.tool("fs:read"),
    writeFile: PTC.tool("fs:write"),
    httpGet: PTC.tool("http:get"),
    queryDb: PTC.tool("db:query"),
  },
});

// 在代码中使用工具
const result = await interpreter.run(`
  // 读取配置文件
  const config = await readFile("config.json");
  
  // 从 API 获取数据
  const data = await httpGet(config.apiUrl + "/users");
  
  // 处理数据
  const processed = data.map(user => ({
    name: user.name,
    email: user.email.toLowerCase(),
    active: user.lastLogin > "2024-01-01",
  }));
  
  // 写入结果
  await writeFile("output/users.json", JSON.stringify(processed, null, 2));
  
  return { count: processed.length, active: processed.filter(u => u.active).length };
`);
```

### PTC 工具注册

```typescript
// 方式一：使用 PTC.tool() 注册
const tools = {
  readFile: PTC.tool("fs:read", {
    // 工具描述
    description: "读取文件内容",
    // 参数验证
    schema: {
      type: "object",
      properties: {
        path: { type: "string", description: "文件路径" },
      },
      required: ["path"],
    },
  }),
};

// 方式二：使用自定义函数
const tools = {
  customTool: PTC.define({
    name: "customTool",
    description: "自定义工具",
    parameters: {
      type: "object",
      properties: {
        input: { type: "string" },
      },
    },
    // 工具实现
    execute: async (params, context) => {
      return { result: params.input.toUpperCase() };
    },
  }),
};
```

## 编程式子代理

编程式子代理允许在 JavaScript 代码中创建和控制子代理：

```typescript
import { Interpreter, ProgrammaticSubAgent } from "deepagents";

const interpreter = new Interpreter({
  runtime: "quickjs",
  subAgents: {
    // 注册子代理
    analyzer: ProgrammaticSubAgent.create({
      name: "analyzer",
      model: "gpt-4",
      systemPrompt: "你是数据分析专家",
      tools: ["readFile", "queryDb"],
    }),
    summarizer: ProgrammaticSubAgent.create({
      name: "summarizer",
      model: "gpt-3.5-turbo",
      systemPrompt: "你是内容摘要专家",
    }),
  },
});

// 在代码中使用子代理
const result = await interpreter.run(`
  // 使用分析子代理处理数据
  const analysis = await analyzer.run({
    prompt: "分析 sales_data.csv 的销售趋势",
    context: { filePath: "/data/sales_data.csv" },
  });
  
  // 使用摘要子代理生成报告
  const summary = await summarizer.run({
    prompt: "将以下分析结果总结为一段话：\\n" + analysis.result,
  });
  
  return {
    analysis: analysis.result,
    summary: summary.result,
    tokensUsed: analysis.tokens + summary.tokens,
  };
`);
```

### 子代理编排

```typescript
const result = await interpreter.run(`
  // 并行执行多个子代理
  const [marketAnalysis, competitorAnalysis, customerAnalysis] = await Promise.all([
    analyzer.run({ prompt: "分析市场趋势" }),
    analyzer.run({ prompt: "分析竞争对手" }),
    analyzer.run({ prompt: "分析客户画像" }),
  ]);
  
  // 汇总结果
  const report = await summarizer.run({
    prompt: \`
      基于以下三个维度的分析，生成综合报告：
      1. 市场分析：\${marketAnalysis.result}
      2. 竞争分析：\${competitorAnalysis.result}
      3. 客户分析：\${customerAnalysis.result}
    \`,
  });
  
  return report.result;
`);
```

## 有状态工作

解释器支持有状态的工作流，可以在多次调用之间保持状态：

```typescript
import { Interpreter, StatefulWorkflow } from "deepagents";

// 创建有状态工作流
const workflow = new StatefulWorkflow({
  interpreter: new Interpreter({ runtime: "quickjs" }),
  // 初始状态
  initialState: {
    step: 0,
    results: [],
    errors: [],
  },
  // 状态持久化
  persistence: {
    driver: "memory", // 或 "file"、"redis"
    key: "workflow_123",
  },
});

// 定义工作流步骤
workflow.step("fetch_data", async (state, tools) => {
  const data = await tools.httpGet("https://api.example.com/data");
  return { ...state, data, step: 1 };
});

workflow.step("process_data", async (state, tools) => {
  const processed = state.data.map(item => ({
    ...item,
    score: item.value * 2,
  }));
  return { ...state, processed, step: 2 };
});

workflow.step("save_results", async (state, tools) => {
  await tools.writeFile("results.json", JSON.stringify(state.processed));
  return { ...state, saved: true, step: 3 };
});

// 执行工作流（可中断恢复）
const result = await workflow.run();

// 或逐步执行
const step1Result = await workflow.runStep("fetch_data");
const step2Result = await workflow.runStep("process_data");
const step3Result = await workflow.runStep("save_results");
```

### 工作流恢复

```typescript
// 从持久化状态恢复工作流
const restoredWorkflow = await StatefulWorkflow.restore({
  key: "workflow_123",
  interpreter: new Interpreter({ runtime: "quickjs" }),
});

// 继续执行未完成的步骤
const result = await restoredWorkflow.resume();
```

## 确定性转换

解释器支持确定性转换，确保相同输入始终产生相同输出：

```typescript
import { Interpreter, DeterministicTransform } from "deepagents";

// 创建确定性转换
const transform = new DeterministicTransform({
  interpreter: new Interpreter({ runtime: "quickjs" }),
  // 禁用随机性来源
  deterministic: true,
  // 固定时间
  fixedTime: new Date("2024-01-01T00:00:00Z"),
  // 固定随机种子
  seed: 42,
});

// 相同输入总是产生相同输出
const result1 = await transform.run(`
  return {
    timestamp: Date.now(),  // 固定时间
    random: Math.random(),  // 固定种子
    uuid: generateUUID(),   // 确定性生成
  };
`);

const result2 = await transform.run(`
  return {
    timestamp: Date.now(),
    random: Math.random(),
    uuid: generateUUID(),
  };
`);

console.log(result1 === result2); // true
```

### 确定性转换应用场景

| 场景 | 说明 |
|:-----|:-----|
| **数据转换** | ETL 管道中的数据格式转换 |
| **缓存键生成** | 基于输入生成确定性缓存键 |
| **测试** | 单元测试中需要可重现的结果 |
| **审计** | 需要可追溯的转换记录 |
| **分布式一致性** | 多节点执行需要相同结果 |

### 使用示例

```typescript
// 数据转换管道
const pipeline = new DeterministicTransform({
  interpreter: new Interpreter({ runtime: "quickjs" }),
  deterministic: true,
});

const transformed = await pipeline.run(`
  // 输入数据
  const input = ${JSON.stringify(rawData)};
  
  // 确定性转换
  const result = input
    .filter(item => item.active)
    .map(item => ({
      id: item.id,
      name: item.name.trim().toLowerCase(),
      score: Math.round(item.score * 100) / 100,
    }))
    .sort((a, b) => a.id.localeCompare(b.id));
  
  return result;
`);
```

> **提示**：确定性模式会禁用所有随机性来源（`Math.random()`、`Date.now()`、`crypto` 等），确保转换的可重现性。

## 解释器配置

```typescript
const interpreter = new Interpreter({
  // 运行时
  runtime: "quickjs",
  // 执行超时（毫秒）
  timeout: 5000,
  // 内存限制（字节）
  memoryLimit: 1024 * 1024,
  // 栈大小限制
  stackSize: 1024 * 64,
  // 全局对象
  globals: {
    console: "log-only", // 仅允许 console.log
    JSON: true,
    Math: true,
    Date: true,
    // 禁止的全局对象
    fetch: false,
    XMLHttpRequest: false,
    process: false,
  },
  // 模块支持
  modules: {
    enabled: true,
    // 允许导入的模块
    allowed: ["lodash", "date-fns"],
    // 模块解析器
    resolver: (specifier) => {
      return require.resolve(specifier);
    },
  },
  // 错误处理
  onError: (error, code) => {
    console.error("解释器错误:", error);
    console.error("执行代码:", code);
  },
});
```

> **注意**：解释器的超时设置非常重要，过短可能导致正常代码无法完成，过长可能导致资源浪费。建议根据实际场景调整。
