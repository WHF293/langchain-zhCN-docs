# 第十六章 配置文件（Configuration）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

Deep Agents 使用分层配置系统，支持 Harness 配置文件、提供者配置文件和模型配置包。配置可以通过 YAML/JSON 文件、环境变量或编程方式定义，实现灵活的部署和管理。

## 配置层次结构

配置按以下优先级加载（从高到低）：

```
1. 环境变量
2. 命令行参数
3. 项目级配置文件 (.deepagents.yaml)
4. 用户级配置文件 (~/.deepagents/config.yaml)
5. 全局默认配置
```

## Harness 配置文件

Harness 配置文件定义代理的核心行为和能力：

```yaml
# .deepagents.yaml
# Harness 配置文件

# 代理配置
agent:
  # 模型配置
  model:
    provider: "openai"
    name: "gpt-4"
    temperature: 0.7
    maxTokens: 4096
  
  # 系统提示
  systemPrompt: |
    你是一个专业的数据分析助手。
    请用中文回复，并提供详细的分析过程。
  
  # 工具配置
  tools:
    - name: "fs:read"
      enabled: true
    - name: "fs:write"
      enabled: true
    - name: "shell:exec"
      enabled: false
  
  # 权限配置
  permissions:
    - type: "allow"
      pattern: "fs:read:*"
    - type: "allow"
      pattern: "fs:write:/workspace/**"
    - type: "deny"
      pattern: "shell:*"

# 后端配置
backend:
  type: "composite"
  backends:
    fs:
      type: "filesystem"
      root: "/workspace"
      ignore:
        - "node_modules/**"
        - ".git/**"
    shell:
      type: "local"
      cwd: "/workspace"
      timeout: 30000

# 沙箱配置
sandbox:
  provider: "e2b"
  apiKey: "${E2B_API_KEY}"
  runtime: "node"
  timeout: 60
```

## 提供者配置文件

提供者配置文件定义外部服务的连接信息：

```yaml
# providers.yaml

providers:
  # OpenAI 配置
  openai:
    apiKey: "${OPENAI_API_KEY}"
    organization: "${OPENAI_ORG_ID}"
    baseUrl: "https://api.openai.com/v1"
    timeout: 30
    maxRetries: 3

  # Anthropic 配置
  anthropic:
    apiKey: "${ANTHROPIC_API_KEY}"
    baseUrl: "https://api.anthropic.com"
    version: "2024-01-01"

  # 本地模型配置
  local:
    type: "ollama"
    baseUrl: "http://localhost:11434"
    model: "llama2"

  # 沙箱提供者配置
  sandbox:
    e2b:
      apiKey: "${E2B_API_KEY}"
      template: "node"
    langsmith:
      apiKey: "${LANGSMITH_API_KEY}"
      runtime: "node"
```

## 模型配置包

模型配置包为不同模型提供优化的配置参数：

```typescript
import { ModelConfigPackage } from "deepagents";

// 注册模型配置包
ModelConfigPackage.register("gpt-4", {
  // 模型参数
  params: {
    temperature: 0.7,
    maxTokens: 4096,
    topP: 1,
    frequencyPenalty: 0,
    presencePenalty: 0,
  },
  // 工具支持
  tools: {
    enabled: true,
    maxTools: 10,
    parallelToolCalls: true,
  },
  // 上下文窗口
  contextWindow: 128000,
  // 支持的功能
  features: {
    vision: true,
    functionCalling: true,
    jsonMode: true,
  },
});

// 使用配置包
const agent = new Agent({
  model: "gpt-4",
  configPackage: "gpt-4", // 自动应用优化配置
});
```

### 内置配置包

| 模型 | 配置包名称 | 特点 |
|:-----|:-----------|:-----|
| GPT-4 | `gpt-4` | 高质量、高成本、支持工具 |
| GPT-4 Turbo | `gpt-4-turbo` | 128K 上下文、支持视觉 |
| GPT-3.5 Turbo | `gpt-3.5-turbo` | 快速、低成本 |
| Claude 3 Opus | `claude-3-opus` | 高质量、长上下文 |
| Claude 3 Sonnet | `claude-3-sonnet` | 平衡性能和成本 |
| Claude 3 Haiku | `claude-3-haiku` | 快速、低成本 |

## 注册键（Registry Keys）

注册键用于标识和引用配置组件：

```typescript
import { Registry } from "deepagents";

// 注册模型
Registry.registerModel("custom-model", {
  provider: "openai",
  name: "gpt-4",
  params: { temperature: 0.5 },
});

// 注册工具
Registry.registerTool("custom-tool", {
  name: "custom-tool",
  description: "自定义工具",
  parameters: {
    type: "object",
    properties: {
      input: { type: "string" },
    },
  },
  execute: async (params) => {
    return { result: params.input };
  },
});

// 注册后端
Registry.registerBackend("custom-backend", {
  type: "composite",
  backends: {
    fs: { type: "filesystem", root: "/workspace" },
  },
});

// 使用注册键
const agent = new Agent({
  model: Registry.getModel("custom-model"),
  tools: [Registry.getTool("custom-tool")],
  backend: Registry.getBackend("custom-backend"),
});
```

### 注册键命名约定

```
<namespace>:<type>:<name>

示例：
- model:openai:gpt-4
- tool:fs:read
- backend:composite:default
- sandbox:e2b:node
```

## 合并语义

当多个配置源存在时，按以下规则合并：

```typescript
// 基础配置
const baseConfig = {
  model: { provider: "openai", name: "gpt-4" },
  tools: ["fs:read", "fs:write"],
  permissions: [{ type: "allow", pattern: "fs:read:*" }],
};

// 覆盖配置
const overrideConfig = {
  model: { temperature: 0.5 }, // 深度合并
  tools: ["shell:exec"], // 数组合并（去重）
  permissions: [{ type: "deny", pattern: "shell:*" }], // 数组合并
};

// 合并结果
const merged = Config.merge(baseConfig, overrideConfig);
// {
//   model: { provider: "openai", name: "gpt-4", temperature: 0.5 },
//   tools: ["fs:read", "fs:write", "shell:exec"],
//   permissions: [
//     { type: "allow", pattern: "fs:read:*" },
//     { type: "deny", pattern: "shell:*" },
//   ],
// }
```

### 合并策略

| 数据类型 | 合并策略 | 说明 |
|:---------|:---------|:-----|
| 对象 | 深度合并 | 递归合并属性 |
| 数组 | 去重合并 | 合并并去除重复项 |
| 基本类型 | 覆盖 | 后者覆盖前者 |
| 函数 | 覆盖 | 后者覆盖前者 |
| `null` | 忽略 | 不覆盖已有值 |
| `undefined` | 忽略 | 不覆盖已有值 |

## 插件打包

插件可以打包为独立的配置单元，便于分发和复用：

```typescript
import { Plugin, PluginPackage } from "deepagents";

// 定义插件
const myPlugin: Plugin = {
  name: "my-custom-plugin",
  version: "1.0.0",
  description: "自定义插件",
  
  // 插件配置
  config: {
    model: {
      provider: "openai",
      name: "gpt-4",
    },
    tools: [
      {
        name: "custom-tool",
        execute: async (params) => {
          return { result: "custom" };
        },
      },
    ],
    systemPrompt: "这是插件提供的系统提示",
  },
  
  // 插件初始化
  initialize: async (agent) => {
    console.log("插件已初始化");
  },
  
  // 插件清理
  cleanup: async () => {
    console.log("插件已清理");
  },
};

// 打包插件
const pluginPackage = PluginPackage.create(myPlugin);

// 导出插件
await pluginPackage.export("./my-plugin.deepagents-plugin");

// 导入插件
const importedPlugin = await PluginPackage.import("./my-plugin.deepagents-plugin");

// 注册插件
const agent = new Agent({
  model: "gpt-4",
  plugins: [importedPlugin],
});
```

### 插件目录结构

```
my-plugin/
├── manifest.json          # 插件清单
├── config.yaml            # 插件配置
├── tools/                 # 工具定义
│   ├── tool1.ts
│   └── tool2.ts
├── prompts/               # 提示模板
│   └── system.txt
└── README.md              # 插件说明
```

### 插件清单文件

```json
{
  "name": "my-custom-plugin",
  "version": "1.0.0",
  "description": "自定义插件",
  "author": "Your Name",
  "license": "MIT",
  "dependencies": {
    "deepagents": ">=1.0.0"
  },
  "main": "index.js",
  "config": "config.yaml",
  "tools": [
    "tools/tool1.js",
    "tools/tool2.js"
  ]
}
```

## 配置加载

```typescript
import { ConfigLoader } from "deepagents";

// 从文件加载配置
const config = await ConfigLoader.load("./deepagents.yaml");

// 从环境变量加载
const envConfig = await ConfigLoader.fromEnv();

// 合并多个配置源
const finalConfig = ConfigLoader.merge([
  config,
  envConfig,
  // 命令行参数
  ConfigLoader.fromArgs(process.argv),
]);

// 验证配置
const validated = ConfigLoader.validate(finalConfig);
if (!validated.success) {
  console.error("配置验证失败:", validated.errors);
}
```

> **建议**：生产环境中，建议将敏感信息（API 密钥等）存储在环境变量中，而非配置文件中。
