# 第1章 Deep Agents 概述

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

## 🎯 什么是 Deep Agents

Deep Agents 是一个用于构建 **AI 代理 (Agent)** 的框架。它提供了一套完整的工具和机制，让开发者能够快速创建具有复杂推理和执行能力的 AI 代理。

与其他 Agent 框架不同，Deep Agents 专注于：

- **深度推理**：支持复杂的多步骤任务执行
- **虚拟文件系统**：内置文件读写能力
- **工具生态**：丰富的内置工具和 MCP 工具支持
- **记忆管理**：持久化的上下文记忆
- **子代理系统**：支持任务委托和并行执行

---

## 🌟 核心能力

### 执行环境 (Execution Environment)

Deep Agents 提供了一个安全的执行环境，代理可以在其中：

- 执行代码片段
- 读写文件
- 调用外部工具
- 管理运行时状态

```typescript
import { DeepAgent } from "@langchain/deep-agents";

// 创建具有执行环境的代理
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: ["execute", "read_file", "write_file"],
});
```

### 上下文管理 (Context Management)

框架自动管理代理的上下文，包括：

| 上下文类型 | 说明 | 持久化 |
|-----------|------|:------:|
| 对话历史 | 当前会话的消息记录 | ❌ |
| 工作记忆 | 任务执行中的临时状态 | ❌ |
| 长期记忆 | 跨会话的持久化知识 | ✅ |
| 文件上下文 | 虚拟文件系统中的文件 | ✅ |

### 委托机制 (Delegation)

主代理可以将子任务委托给子代理执行：

```typescript
// 主代理可以委托任务
const result = await agent.delegate({
  task: "分析这段代码的性能问题",
  target: "code-reviewer",
  context: { code: sourceCode },
});
```

### 引导机制 (Steering)

通过系统提示和规则引导代理行为：

```typescript
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  steering: {
    rules: [
      "执行代码前先进行静态分析",
      "修改文件前先创建备份",
      "遇到错误时尝试自行修复",
    ],
  },
});
```

## 📁 虚拟文件系统 (Virtual File System)

Deep Agents 内置了虚拟文件系统 (VFS)，代理可以：

- 读取和写入文件
- 创建和删除目录
- 搜索文件内容
- 管理文件权限

```typescript
// 代理可以操作虚拟文件系统
const files = await agent.vfs.listFiles("./src");
const content = await agent.vfs.readFile("./src/index.ts");
await agent.vfs.writeFile("./output/result.txt", analysisResult);
```

> **注意**：虚拟文件系统默认是临时的，会话结束后文件会丢失。如需持久化，请配置存储后端。

## 🔧 工具 (Tools)

工具是代理执行任务的基本单元。Deep Agents 支持三类工具：

| 工具类型 | 说明 | 示例 |
|---------|------|------|
| 内置工具 | 框架预置的基础工具 | `ls`, `read_file`, `write_file` |
| 自定义工具 | 开发者创建的工具 | API 调用、数据处理 |
| MCP 工具 | 通过 MCP 协议接入的工具 | GitHub、数据库 |

```typescript
// 创建自定义工具
import { createTool } from "@langchain/deep-agents";

const searchTool = createTool({
  name: "web_search",
  description: "搜索互联网获取信息",
  parameters: {
    query: { type: "string", description: "搜索关键词" },
  },
  execute: async ({ query }) => {
    // 实现搜索逻辑
    return await fetchSearchResults(query);
  },
});
```

## 🎯 技能 (Skills)

技能是组合多个工具的高级能力：

```typescript
// 定义技能
const codeReviewSkill = {
  name: "code-review",
  description: "代码审查技能",
  tools: ["read_file", "glob", "grep", "execute"],
  steps: [
    "1. 使用 glob 查找相关文件",
    "2. 使用 read_file 读取代码",
    "3. 使用 grep 搜索潜在问题",
    "4. 使用 execute 运行测试",
  ],
};
```

---

## 🧠 记忆 (Memory)

记忆系统让代理能够跨会话保持上下文：

```typescript
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  memory: {
    type: "long-term",
    storage: "sqlite", // 或 "postgres", "redis"
    maxItems: 1000,
  },
});

// 代理会自动记住重要信息
await agent.run("记住：用户的项目使用 TypeScript + React");
// 下次会话中，代理会记得这个信息
```

## 👥 子代理 (Sub-agents)

子代理系统支持任务分解和并行执行：

```
主代理 (Main Agent)
├── 代码分析子代理 (Code Analyzer)
├── 文档生成子代理 (Doc Generator)
└── 测试执行子代理 (Test Runner)
```

```typescript
// 创建子代理配置
const subAgents = {
  "code-analyzer": {
    model: "anthropic/claude-sonnet-4-20250514",
    tools: ["read_file", "glob", "grep"],
    systemPrompt: "你是代码分析专家",
  },
  "doc-generator": {
    model: "anthropic/claude-sonnet-4-20250514",
    tools: ["read_file", "write_file"],
    systemPrompt: "你是文档生成专家",
  },
};
```

---

## 🤝 人在回路 (Human-in-the-Loop)

支持人工介入关键决策点：

```typescript
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  hitl: {
    // 需要人工确认的操作
    confirmBefore: ["write_file", "execute", "delete_file"],
    // 回调函数
    onConfirm: async (action) => {
      console.log(`等待确认: ${action.description}`);
      return await askUserForConfirmation();
    },
  },
});
```

> **建议**：在生产环境中，建议对所有破坏性操作（删除、覆盖文件等）启用人在回路机制。
