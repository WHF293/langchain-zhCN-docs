# 第5章 工具 (Tools)

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

## 📖 概述

工具 (Tools) 是 Deep Agents 代理执行任务的基本单元。代理通过调用工具来完成各种操作，如文件读写、代码执行、网络请求等。

Deep Agents 支持三类工具：

| 工具类型 | 说明 | 示例 |
|---------|------|------|
| **自定义工具** | 开发者创建的工具 | API 调用、数据处理 |
| **MCP 工具** | 通过 MCP 协议接入的工具 | GitHub、数据库 |
| **内置工具** | 框架预置的基础工具 | `ls`, `read_file`, `write_file` |

---

## 🔧 自定义工具 (Custom Tools)

### 创建基本工具

使用 `createTool` 函数创建自定义工具：

```typescript
import { createTool } from "@langchain/deep-agents";

// 创建天气查询工具
const weatherTool = createTool({
  // 工具名称
  name: "get_weather",

  // 工具描述（代理会根据描述决定何时使用）
  description: "获取指定城市的天气信息",

  // 参数定义
  parameters: {
    city: {
      type: "string",
      description: "城市名称，如：北京、上海",
    },
    unit: {
      type: "string",
      enum: ["celsius", "fahrenheit"],
      description: "温度单位",
      default: "celsius",
    },
  },

  // 执行函数
  execute: async ({ city, unit }) => {
    // 调用天气 API
    const response = await fetch(
      `https://api.weather.com/v1/${city}?unit=${unit}`
    );
    const data = await response.json();

    return {
      city: data.city,
      temperature: data.temperature,
      condition: data.condition,
      humidity: data.humidity,
    };
  },
});

export { weatherTool };
```

### 工具参数类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 字符串 | `"hello"` |
| `number` | 数字 | `42`, `3.14` |
| `boolean` | 布尔值 | `true`, `false` |
| `array` | 数组 | `["a", "b"]` |
| `object` | 对象 | `{ key: "value" }` |
| `enum` | 枚举 | 限定可选值 |

### 复杂参数示例

```typescript
const complexTool = createTool({
  name: "analyze_code",
  description: "分析代码质量和潜在问题",
  parameters: {
    code: {
      type: "string",
      description: "要分析的代码内容",
    },
    language: {
      type: "string",
      enum: ["typescript", "javascript", "python"],
      description: "代码语言",
    },
    options: {
      type: "object",
      properties: {
        checkSecurity: {
          type: "boolean",
          description: "是否检查安全问题",
          default: true,
        },
        maxIssues: {
          type: "number",
          description: "最大返回问题数",
          default: 10,
        },
      },
      description: "分析选项",
    },
  },
  execute: async ({ code, language, options }) => {
    // 实现代码分析逻辑
    return { issues: [], summary: "代码质量良好" };
  },
});
```

### 错误处理

```typescript
const safeTool = createTool({
  name: "risky_operation",
  description: "执行可能失败的操作",
  parameters: {
    input: { type: "string" },
  },
  execute: async ({ input }) => {
    try {
      // 可能失败的操作
      const result = await someRiskyOperation(input);
      return { success: true, data: result };
    } catch (error) {
      // 返回错误信息，让代理决定如何处理
      return {
        success: false,
        error: error.message,
        suggestion: "请检查输入参数是否正确",
      };
    }
  },
});
```

## 🔌 MCP 工具 (MCP Tools)

### 什么是 MCP

MCP (Model Context Protocol) 是一种标准化的工具互操作协议。通过 MCP，Deep Agents 可以接入各种外部工具和服务。

### 配置 MCP 工具

```typescript
import { DeepAgent, MCPToolProvider } from "@langchain/deep-agents";

// 创建 MCP 工具提供者
const mcpProvider = new MCPToolProvider({
  // MCP 服务器配置
  servers: [
    {
      name: "github",
      transport: "stdio",
      command: "npx",
      args: ["@modelcontextprotocol/server-github"],
      env: {
        GITHUB_TOKEN: process.env.GITHUB_TOKEN,
      },
    },
    {
      name: "filesystem",
      transport: "stdio",
      command: "npx",
      args: ["@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"],
    },
  ],
});

// 创建代理并注入 MCP 工具
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: [mcpProvider],
});
```

### 常用 MCP 服务器

| 服务器 | 说明 | 安装 |
|--------|------|------|
| `@modelcontextprotocol/server-github` | GitHub 操作 | `npx @modelcontextprotocol/server-github` |
| `@modelcontextprotocol/server-filesystem` | 文件系统访问 | `npx @modelcontextprotocol/server-filesystem` |
| `@modelcontextprotocol/server-sqlite` | SQLite 数据库 | `npx @modelcontextprotocol/server-sqlite` |
| `@modelcontextprotocol/server-postgres` | PostgreSQL | `npx @modelcontextprotocol/server-postgres` |

### MCP 工具示例：GitHub

```typescript
// 使用 MCP GitHub 工具
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: [mcpProvider],
});

// 代理可以调用 GitHub 工具
const response = await agent.run({
  messages: [
    {
      role: "user",
      content: "查看仓库 user/repo 的最近 5 个 commit",
    },
  ],
});
// 代理会自动调用 MCP 的 GitHub 工具
```

## 🛠️ 内置工具 (Built-in Tools)

Deep Agents 提供了丰富的内置工具，覆盖文件操作、搜索、执行等常见场景。

### ls - 列出目录内容

列出指定目录下的文件和子目录。

```typescript
// 使用方式
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: ["ls"],
});

// 代理可以列出目录内容
const response = await agent.run({
  messages: [
    { role: "user", content: "列出当前目录的文件" },
  ],
});
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `path` | string | 否 | 目录路径，默认为当前目录 |
| `recursive` | boolean | 否 | 是否递归列出子目录 |

**返回示例：**

```
./src/
├── index.ts
├── utils/
│   ├── helper.ts
│   └── config.ts
└── types.ts
```

### glob - 文件模式匹配

使用 glob 模式搜索文件。

```typescript
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: ["glob"],
});
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `pattern` | string | 是 | glob 模式，如 `**/*.ts` |
| `cwd` | string | 否 | 搜索根目录 |

**常用模式：**

| 模式 | 说明 |
|------|------|
| `*.ts` | 当前目录所有 .ts 文件 |
| `**/*.ts` | 递归查找所有 .ts 文件 |
| `src/**/*.tsx` | src 目录下所有 .tsx 文件 |
| `!*.test.ts` | 排除测试文件 |

**返回示例：**

```typescript
{
  files: [
    "./src/index.ts",
    "./src/utils/helper.ts",
    "./src/types.ts"
  ],
  count: 3
}
```

### execute - 执行命令

在沙箱中执行 shell 命令或代码。

```typescript
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: ["execute"],
});
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `command` | string | 是 | 要执行的命令 |
| `language` | string | 否 | 代码语言（执行代码时） |
| `timeout` | number | 否 | 超时时间（毫秒） |

**示例：**

```typescript
// 执行 shell 命令
await agent.run({
  messages: [
    { role: "user", content: "运行 npm test" },
  ],
});

// 执行代码片段
await agent.run({
  messages: [
    {
      role: "user",
      content: "执行这段 TypeScript 代码：console.log('Hello')",
    },
  ],
});
```

> **注意**：execute 工具在沙箱环境中运行，有时间和资源限制。

### task - 创建子任务

```typescript
const agent = new DeepAgent({
```

**参数说明：**

## 🔨 组合使用示例

```typescript
// 创建一个功能完整的代理
const agent = new DeepAgent({
  model: "anthropic/claude-sonnet-4-20250514",
  tools: [
    // 内置工具
    "ls", "read_file", "write_file", "edit_file",
    "glob", "grep", "execute", "task",

    // 自定义工具
    weatherTool,
    searchTool,

    // MCP 工具
    mcpProvider,
  ],
});

// 代理可以组合使用这些工具
const response = await agent.run({
  messages: [
    {
      role: "user",
      content: "查看项目结构，找到所有 TypeScript 文件，分析代码质量，生成报告",
    },
  ],
});
// 代理会依次调用 ls -> glob -> grep -> read_file -> write_file
```

## 💡 最佳实践

1. **工具描述**：写清晰的 description，帮助代理正确选择工具
2. **参数验证**：在 execute 函数中验证参数，返回有用的错误信息
3. **错误处理**：捕获异常并返回结构化错误，而非抛出异常
4. **幂等性**：尽量让工具操作是幂等的，避免副作用
5. **最小权限**：只暴露必要的工具，遵循最小权限原则

---

> **建议**：创建自定义工具时，description 是最重要的字段。代理会根据 description 判断何时使用该工具，因此要写得清晰、准确。
