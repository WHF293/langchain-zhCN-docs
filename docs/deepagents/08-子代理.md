# 第八章 子代理（Sub-Agents）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

子代理是 Deep Agents 架构中的重要概念，它允许将复杂任务分解给专门的代理处理。本章介绍子代理的创建、配置和最佳实践。

## 默认/通用子代理（Default/General Sub-Agents）

Deep Agents 提供开箱即用的通用子代理：

```typescript
// 使用默认子代理
import { deepAgents, DefaultSubAgents } from "@langchain/deep-agents";

const agent = deepAgents.create({
  subAgents: {
    // 内置通用子代理
    researcher: DefaultSubAgents.researcher({
      // 研究深度
      depth: "comprehensive",
      // 信息来源
      sources: ["web", "academic", "news"]
    }),

    coder: DefaultSubAgents.coder({
      // 支持的编程语言
      languages: ["typescript", "python", "rust"],
      // 代码风格
      style: "clean"
    }),

    writer: DefaultSubAgents.writer({
      // 写作风格
      tone: "professional",
      // 输出格式
      formats: ["markdown", "html", "plain"]
    }),

    analyst: DefaultSubAgents.analyst({
      // 分析类型
      capabilities: ["data", "statistical", "trend"],
      // 可视化
      visualization: true
    })
  }
});

// 代理会自动选择合适的子代理
await agent.invoke("帮我研究一下最新的AI趋势并写一份报告");
// 自动调用: researcher -> analyst -> writer
```

## 自定义子代理（Custom Sub-Agents）

### 字典方式定义

最简单的自定义子代理方式：

```typescript
// 使用字典方式定义子代理
const customSubAgent = {
  // 基本信息
  name: "customer-support",
  description: "处理客户咨询和投诉",

  // 能力定义
  capabilities: [
    "answer-questions",
    "handle-complaints",
    "process-returns"
  ],

  // 触发条件
  triggers: {
    // 关键词触发
    keywords: ["客服", "投诉", "退货", "问题"],

    // 意图触发
    intents: ["customer_inquiry", "complaint", "return_request"],

    // 自定义条件
    condition: (input, context) => {
      return input.metadata?.channel === "customer-service";
    }
  },

  // 处理逻辑
  handler: async (input, context) => {
    // 分析客户意图
    const intent = await analyzeIntent(input.text);

    // 根据意图处理
    switch (intent.type) {
      case "question":
        return await handleQuestion(intent, context);
      case "complaint":
        return await handleComplaint(intent, context);
      case "return":
        return await processReturn(intent, context);
      default:
        return await escalateToHuman(input, context);
    }
  },

  // 配置
  config: {
    // 温度设置
    temperature: 0.7,
    // 最大token
    maxTokens: 2000,
    // 超时时间
    timeout: 30000
  }
};

const agent = deepAgents.create({
  subAgents: {
    "customer-support": customSubAgent
  }
});
```

### CompiledSubAgent 方式

对于更复杂的子代理，使用 CompiledSubAgent 进行编译：

```typescript
// 使用 CompiledSubAgent 创建高级子代理
import { CompiledSubAgent, StateGraph } from "@langchain/deep-agents";

// 定义子代理状态
interface ResearchState {
  query: string;
  sources: Source[];
  findings: Finding[];
  summary: string;
  confidence: number;
}

// 创建状态图
const researchGraph = new StateGraph<ResearchState>({
  // 状态模式
  schema: {
    query: { type: "string" },
    sources: { type: "array" },
    findings: { type: "array" },
    summary: { type: "string", default: "" },
    confidence: { type: "number", default: 0 }
  }
});

// 添加节点
researchGraph.addNode("search", async (state) => {
  const results = await searchWeb(state.query);
  return { sources: results };
});

researchGraph.addNode("analyze", async (state) => {
  const findings = await analyzeSources(state.sources);
  return { findings };
});

researchGraph.addNode("synthesize", async (state) => {
  const summary = await synthesizeFindings(state.findings);
  const confidence = calculateConfidence(state.findings);
  return { summary, confidence };
});

// 添加边
researchGraph.addEdge("search", "analyze");
researchGraph.addEdge("analyze", "synthesize");

// 条件边
researchGraph.addConditionalEdge("synthesize", (state) => {
  if (state.confidence < 0.7) {
    return "search";  // 信心不足，继续搜索
  }
  return "__end__";
});

// 编译子代理
const researchSubAgent = new CompiledSubAgent({
  name: "deep-researcher",
  description: "深度研究代理，支持多轮搜索和分析",

  // 编译状态图
  graph: researchGraph.compile(),

  // 输入映射
  inputMapping: (parentInput) => ({
    query: parentInput.text
  }),

  // 输出映射
  outputMapping: (agentOutput) => ({
    result: agentOutput.summary,
    sources: agentOutput.sources,
    confidence: agentOutput.confidence
  }),

  // 配置
  config: {
    maxIterations: 5,
    checkpointing: true,
    humanInTheLoop: false
  }
});

// 使用编译后的子代理
const agent = deepAgents.create({
  subAgents: {
    "researcher": researchSubAgent
  }
});
```

## 流式输出（Streaming Output）

子代理支持流式输出，实时返回处理进度：

```typescript
// 子代理流式输出配置
const streamingConfig = {
  // 启用流式
  enabled: true,

  // 流式事件类型
  events: {
    // 节点开始
    nodeStart: true,
    // 节点完成
    nodeComplete: true,
    // 生成文本
    textGeneration: true,
    // 工具调用
    toolCalls: true,
    // 状态更新
    stateUpdates: true
  },

  // 回调函数
  callbacks: {
    onToken: (token) => {
      process.stdout.write(token);
    },

    onNodeStart: (nodeName) => {
      console.log(`\n[开始] ${nodeName}`);
    },

    onNodeComplete: (nodeName, output) => {
      console.log(`[完成] ${nodeName}`);
    },

    onToolCall: (toolName, args) => {
      console.log(`[调用工具] ${toolName}`, args);
    }
  }
};

// 使用流式子代理
const agent = deepAgents.create({
  subAgents: {
    "writer": {
      stream: streamingConfig,
      handler: async function* (input) {
        // 使用生成器实现流式输出
        yield { type: "thinking", content: "正在分析需求..." };

        const outline = await generateOutline(input);
        yield { type: "outline", content: outline };

        for (const section of outline.sections) {
          yield { type: "writing", content: `正在撰写: ${section.title}` };
          const content = await writeSection(section);
          yield { type: "section-complete", content };
        }

        yield { type: "complete", content: "文章撰写完成" };
      }
    }
  }
});

// 流式调用
const stream = agent.stream({
  input: "写一篇关于AI的文章",
  subAgent: "writer"
});

for await (const event of stream) {
  switch (event.type) {
    case "thinking":
      console.log(`💭 ${event.content}`);
      break;
    case "writing":
      console.log(`✍️ ${event.content}`);
      break;
    case "section-complete":
      console.log(`✅ ${event.content.title}`);
      break;
  }
}
```

## 结构化输出（Structured Output）

子代理可以返回结构化数据：

```typescript
// 定义输出模式
const outputSchema = {
  type: "object",
  properties: {
    title: { type: "string" },
    summary: { type: "string" },
    keyPoints: {
      type: "array",
      items: { type: "string" }
    },
    sentiment: {
      type: "string",
      enum: ["positive", "negative", "neutral"]
    },
    confidence: {
      type: "number",
      minimum: 0,
      maximum: 1
    },
    metadata: {
      type: "object",
      properties: {
        wordCount: { type: "number" },
        readingTime: { type: "string" },
        tags: { type: "array", items: { type: "string" } }
      }
    }
  },
  required: ["title", "summary", "keyPoints", "sentiment"]
};

// 创建结构化输出子代理
const analyzerSubAgent = {
  name: "content-analyzer",
  description: "分析内容并返回结构化结果",

  outputSchema: outputSchema,

  handler: async (input) => {
    // 使用 LLM 生成结构化输出
    const result = await llm.invoke({
      prompt: `分析以下内容并返回JSON格式结果: ${input.text}`,
      responseFormat: { type: "json_object", schema: outputSchema }
    });

    // 验证输出
    const validated = validateSchema(result, outputSchema);

    return validated;
  }
};

// 调用并获取结构化结果
const result = await agent.invoke({
  input: "分析这篇文章的情感和关键点",
  subAgent: "content-analyzer",
  outputFormat: "structured"
});

console.log(result.title);      // "文章标题"
console.log(result.sentiment);  // "positive"
console.log(result.keyPoints);  // ["要点1", "要点2", ...]
```

## 技能继承（Skill Inheritance）

子代理可以继承父代理或其他代理的技能：

```typescript
// 技能继承配置
const inheritanceConfig = {
  // 继承模式
  mode: "selective",  // 'all' | 'selective' | 'none'

  // 继承的技能
  inheritSkills: [
    "web-search",
    "file-read",
    "calculator"
  ],

  // 排除的技能
  excludeSkills: [
    "admin-functions",
    "sensitive-operations"
  ],

  // 技能覆盖
  overrides: {
    // 覆盖继承的技能配置
    "web-search": {
      config: {
        maxResults: 5,  // 子代理限制为5个结果
        safeSearch: true
      }
    }
  },

  // 独有技能
  exclusiveSkills: {
    "special-analysis": {
      description: "子代理独有的分析技能",
      handler: async (input) => {
        // 特殊处理逻辑
      }
    }
  }
};

// 定义带继承的子代理
const childAgent = {
  name: "junior-researcher",
  description: "初级研究代理",

  // 继承配置
  inheritance: inheritanceConfig,

  // 父代理
  parent: "senior-researcher",

  handler: async (input, context) => {
    // 可以使用继承的技能
    const searchResults = await context.skills.invoke("web-search", {
      query: input.query
    });

    // 也可以使用独有技能
    const analysis = await context.skills.invoke("special-analysis", {
      data: searchResults
    });

    return analysis;
  }
};

// 创建代理层级
const agentSystem = deepAgents.createHierarchy({
  root: "senior-researcher",
  children: {
    "junior-researcher": childAgent,
    "data-analyst": dataAnalystAgent
  }
});
```

## 上下文管理（Context Management）

管理子代理之间的上下文传递：

```typescript
// 上下文管理配置
const contextManagement = {
  // 上下文传递规则
  passThrough: {
    // 始终传递的字段
    always: ["sessionId", "userId", "requestId"],

    // 条件传递
    conditional: [
      {
        field: "userPreferences",
        condition: (ctx) => ctx.user?.consent === true
      }
    ],

    // 从不传递
    never: ["password", "apiKey", "internalNotes"]
  },

  // 上下文转换
  transformations: {
    // 字段重命名
    rename: {
      "userInput": "query"
    },

    // 字段转换
    transform: {
      "timestamp": (value) => new Date(value).toISOString(),
      "userData": (value) => sanitize(value)
    },

    // 聚合
    aggregate: {
      "history": (entries) => entries.slice(-10)  // 只保留最近10条
    }
  },

  // 上下文注入
  injection: {
    // 自动注入的上下文
    auto: {
      "systemTime": () => new Date().toISOString(),
      "environment": () => process.env.NODE_ENV
    },

    // 按需注入
    onDemand: {
      "detailedLogs": async (context) => {
        return await getLogs(context.sessionId);
      }
    }
  },

  // 上下文隔离
  isolation: {
    // 子代理间隔离
    betweenSubAgents: true,

    // 隔离级别
    level: "strict",  // 'strict' | 'relaxed' | 'none'

    // 共享白名单
    sharedWhitelist: ["publicData", "commonKnowledge"]
  }
};

// 创建带上下文管理的子代理
const agent = deepAgents.create({
  contextManagement: contextManagement,

  subAgents: {
    "worker-a": {
      handler: async (input, context) => {
        // 接收转换后的上下文
        console.log(context.query);  // 原始 userInput
        console.log(context.systemTime);  // 自动注入的时间

        // 子代理间隔离，无法访问 worker-b 的上下文
      }
    },

    "worker-b": {
      handler: async (input, context) => {
        // 只能访问白名单中的共享数据
        console.log(context.publicData);
      }
    }
  }
});
```

## 最佳实践（Best Practices）

### 子代理设计原则

```typescript
// 1. 单一职责原则
const goodSubAgent = {
  name: "email-formatter",  // 明确的职责
  description: "将内容格式化为电子邮件格式",
  // 只做一件事：格式化邮件
  handler: async (input) => {
    return formatAsEmail(input.content);
  }
};

// 避免：职责不清的子代理
const badSubAgent = {
  name: "do-everything",
  description: "处理各种任务",  // 过于宽泛
  handler: async (input) => {
    // 试图做太多事情
  }
};

// 2. 松耦合
const looselyCoupled = {
  // 通过接口通信，不依赖实现细节
  input: {
    type: "object",
    properties: {
      text: { type: "string" }
    }
  },
  output: {
    type: "object",
    properties: {
      result: { type: "string" }
    }
  }
};

// 3. 错误处理
const robustSubAgent = {
  name: "robust-worker",

  // 超时配置
  timeout: 30000,

  // 重试策略
  retry: {
    maxAttempts: 3,
    backoff: "exponential"
  },

  // 降级策略
  fallback: async (input, error) => {
    console.warn("主处理失败，使用降级方案");
    return { result: "降级处理结果", degraded: true };
  },

  handler: async (input) => {
    try {
      return await processInput(input);
    } catch (error) {
      // 记录错误
      await logError(error);
      throw error;  // 让重试机制处理
    }
  }
};
```

## 故障排除（Troubleshooting）

### 常见问题及解决方案

```typescript
// 问题1: 子代理超时
// 解决方案: 增加超时时间或优化处理逻辑
const timeoutSolution = {
  config: {
    timeout: 60000,  // 增加到60秒
    // 或使用分块处理
    chunking: {
      enabled: true,
      chunkSize: 1000
    }
  }
};

// 问题2: 内存溢出
// 解决方案: 限制输入大小和处理数据量
const memorySolution = {
  config: {
    maxInputSize: 10000,  // 限制输入大小
    streaming: true,       // 使用流式处理
    // 分页处理
    pagination: {
      pageSize: 100
    }
  }
};

// 问题3: 子代理返回错误结果
// 解决方案: 添加验证和重试
const validationSolution = {
  // 输出验证
  outputValidator: (output) => {
    if (!output || typeof output.result !== "string") {
      return { valid: false, error: "输出格式无效" };
    }
    return { valid: true };
  },

  // 重试策略
  retry: {
    maxAttempts: 3,
    // 只在特定错误时重试
    retryableErrors: ["ValidationError", "TimeoutError"]
  }
};

// 问题4: 子代理间通信失败
// 解决方案: 使用消息队列和重试机制
const communicationSolution = {
  // 消息队列
  messageQueue: {
    type: "redis",
    connection: process.env.REDIS_URL,
    // 重试策略
    retry: {
      attempts: 5,
      backoff: "exponential"
    }
  },

  // 超时处理
  timeout: {
    send: 5000,
    receive: 10000
  }
};
```

> **注意**：在生产环境中，建议关闭详细调试日志，只保留错误级别日志，以避免性能影响和敏感信息泄露。

> **提示**：使用 `agent.subAgent("name").health()` 定期检查子代理健康状态，及时发现和处理问题。

> **建议**：对于关键业务子代理，实现熔断器（Circuit Breaker）模式，当错误率超过阈值时自动停止调用，避免级联故障。
