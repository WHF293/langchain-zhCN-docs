# 第十三章 后端（Backends）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

后端系统是 Deep Agents 的基础设施层，提供可插拔的文件系统、状态管理、存储和执行环境。每个后端负责特定类型的操作，可以通过 `CompositeBackend` 组合使用。

## 后端类型概览

| 后端 | 功能 | 典型用途 |
|:-----|:-----|:---------|
| `StateBackend` | 管理代理运行状态 | 会话状态、任务进度 |
| `FilesystemBackend` | 文件系统操作 | 读写文件、目录管理 |
| `LocalShellBackend` | 本地 Shell 执行 | 命令行操作、脚本运行 |
| `StoreBackend` | 键值存储 | 配置存储、数据缓存 |
| `ContextHubBackend` | 上下文管理 | 对话历史、上下文注入 |
| `CompositeBackend` | 组合多个后端 | 统一接口访问多种能力 |

## StateBackend（状态后端）

管理代理的运行时状态，包括会话信息和任务进度：

```typescript
import { StateBackend } from "deepagents";

const stateBackend = new StateBackend({
  // 状态存储方式：内存、文件或数据库
  storage: "memory", // 或 "file"、"redis"
  // 状态过期时间（秒）
  ttl: 3600,
  // 是否持久化
  persistent: false,
});

// 读取状态
const state = await stateBackend.get("session_123");

// 更新状态
await stateBackend.set("session_123", {
  currentTask: "analyzing_data",
  progress: 0.75,
});
```

## FilesystemBackend（文件系统后端）

提供安全的文件系统访问，支持路径限制和权限控制：

```typescript
import { FilesystemBackend } from "deepagents";

const fsBackend = new FilesystemBackend({
  // 根目录限制（沙箱化）
  root: "/workspace",
  // 是否允许符号链接
  allowSymlinks: false,
  // 最大文件大小（字节）
  maxFileSize: 10 * 1024 * 1024, // 10MB
  // 忽略的文件模式
  ignore: ["node_modules/**", ".git/**"],
});

// 读取文件
const content = await fsBackend.read("src/index.ts");

// 写入文件
await fsBackend.write("output/result.json", JSON.stringify(data));

// 列出目录
const files = await fsBackend.list("src/");

// 检查文件是否存在
const exists = await fsBackend.exists("config.yaml");
```

## LocalShellBackend（本地 Shell 后端）

在本地环境中执行 Shell 命令：

```typescript
import { LocalShellBackend } from "deepagents";

const shellBackend = new LocalShellBackend({
  // 默认工作目录
  cwd: "/workspace",
  // 环境变量
  env: { NODE_ENV: "development" },
  // 命令超时时间（毫秒）
  timeout: 30000,
  // 允许的命令白名单（可选）
  allowedCommands: ["ls", "cat", "grep", "node", "npm"],
});

// 执行命令
const result = await shellBackend.exec("ls -la");
console.log(result.stdout);

// 执行带管道的命令
const grepResult = await shellBackend.exec("cat data.txt | grep 'error'");
```

## StoreBackend（存储后端）

提供命名空间隔离的键值存储：

```typescript
import { StoreBackend } from "deepagents";

const storeBackend = new StoreBackend({
  // 命名空间
  namespace: "myapp",
  // 存储后端类型
  driver: "memory", // 或 "file"、"redis"、"sqlite"
  // 序列化方式
  serialization: "json",
});

// 存储数据
await storeBackend.set("user_settings", { theme: "dark", lang: "zh" });

// 读取数据
const settings = await storeBackend.get("user_settings");

// 列出所有键
const keys = await storeBackend.keys();

// 删除数据
await storeBackend.delete("temp_data");

// 命名空间隔离：不同命名空间的键互不影响
const cacheBackend = new StoreBackend({ namespace: "cache" });
await cacheBackend.set("user_settings", { cached: true }); // 不会影响 myapp 命名空间
```

## ContextHubBackend（上下文中心后端）

管理代理的对话上下文和历史记录：

```typescript
import { ContextHubBackend } from "deepagents";

const contextBackend = new ContextHubBackend({
  // 最大上下文长度（token 数）
  maxTokens: 8000,
  // 上下文压缩策略
  compression: "sliding_window", // 或 "summary"、"smart"
  // 是否包含系统提示
  includeSystemPrompt: true,
});

// 添加消息到上下文
await contextBackend.addMessage({
  role: "user",
  content: "请帮我分析这段代码",
});

// 获取当前上下文
const context = await contextBackend.getContext();

// 清除上下文
await contextBackend.clear();

// 注入系统上下文
await contextBackend.injectContext({
  type: "system",
  content: "当前项目使用 TypeScript + React",
});
```

## CompositeBackend（组合后端）

将多个后端组合为统一接口，是生产环境中最常用的配置方式：

```typescript
import {
  CompositeBackend,
  FilesystemBackend,
  LocalShellBackend,
  StoreBackend,
  ContextHubBackend,
} from "deepagents";

const backend = new CompositeBackend({
  backends: {
    // 文件系统操作
    fs: new FilesystemBackend({
      root: "/workspace",
      ignore: ["node_modules/**"],
    }),
    // Shell 命令执行
    shell: new LocalShellBackend({
      cwd: "/workspace",
      timeout: 30000,
    }),
    // 键值存储
    store: new StoreBackend({
      namespace: "app",
      driver: "memory",
    }),
    // 上下文管理
    context: new ContextHubBackend({
      maxTokens: 8000,
      compression: "sliding_window",
    }),
  },
});

// 通过组合后端访问各个子后端
await backend.fs.read("config.yaml");
await backend.shell.exec("npm test");
await backend.store.set("key", "value");
await backend.context.addMessage({ role: "user", content: "hello" });
```

## 后端路由

`CompositeBackend` 支持基于操作类型的路由，可以将不同操作路由到不同的后端：

```typescript
const backend = new CompositeBackend({
  backends: {
    localFs: new FilesystemBackend({ root: "/workspace" }),
    remoteFs: new FilesystemBackend({
      root: "/remote",
      driver: "s3",
      bucket: "my-bucket",
    }),
    shell: new LocalShellBackend(),
  },
  routes: {
    // 本地文件操作路由到 localFs
    "fs:read:/workspace/**": "localFs",
    "fs:write:/workspace/**": "localFs",
    // 远程文件操作路由到 remoteFs
    "fs:read:/remote/**": "remoteFs",
    "fs:write:/remote/**": "remoteFs",
    // Shell 操作路由到 shell
    "shell:*": "shell",
  },
});
```

## 自定义后端

通过实现 `Backend` 接口可以创建自定义后端：

```typescript
import { Backend, BackendContext } from "deepagents";

class DatabaseBackend implements Backend {
  name = "database";
  private connection: any;

  constructor(config: { connectionString: string }) {
    this.connection = createConnection(config.connectionString);
  }

  // 实现后端接口方法
  async execute(operation: string, params: any, ctx: BackendContext): Promise<any> {
    switch (operation) {
      case "query":
        return this.connection.query(params.sql, params.values);
      case "insert":
        return this.connection.insert(params.table, params.data);
      case "update":
        return this.connection.update(params.table, params.where, params.data);
      default:
        throw new Error(`Unknown operation: ${operation}`);
    }
  }

  // 清理资源
  async dispose(): Promise<void> {
    await this.connection.close();
  }
}

// 使用自定义后端
const backend = new CompositeBackend({
  backends: {
    fs: new FilesystemBackend({ root: "/workspace" }),
    db: new DatabaseBackend({ connectionString: "postgresql://..." }),
  },
});
```

## 权限与后端

后端与权限系统紧密集成，每个后端操作都会经过权限检查：

```typescript
const backend = new CompositeBackend({
  backends: {
    fs: new FilesystemBackend({ root: "/workspace" }),
    shell: new LocalShellBackend(),
  },
  permissions: [
    // 文件系统：允许读取，写入需要确认
    { type: "allow", pattern: "fs:read:*" },
    { type: "interrupt", pattern: "fs:write:*" },
    // Shell：完全禁止
    { type: "deny", pattern: "shell:*" },
  ],
});
```

> **注意**：后端级别的权限配置会与代理级别的权限配置合并，两者取交集（更严格的权限生效）。

## 策略钩子（Policy Hooks）

策略钩子允许在后端操作前后注入自定义逻辑：

```typescript
const backend = new CompositeBackend({
  backends: {
    fs: new FilesystemBackend({ root: "/workspace" }),
  },
  hooks: {
    // 操作执行前的钩子
    beforeExecute: async (operation, params, ctx) => {
      console.log(`即将执行: ${operation}`, params);
      // 可以修改参数或阻止操作
      if (params.path?.includes("..")) {
        throw new Error("不允许路径遍历");
      }
      return params; // 返回修改后的参数
    },
    // 操作执行后的钩子
    afterExecute: async (operation, params, result, ctx) => {
      console.log(`操作完成: ${operation}`, result);
      // 可以修改返回结果
      return result;
    },
    // 错误处理钩子
    onError: async (operation, params, error, ctx) => {
      console.error(`操作失败: ${operation}`, error);
      // 可以决定是否重试
      if (error.code === "TIMEOUT") {
        throw error; // 重试
      }
      return null; // 忽略错误
    },
  },
});
```

## 命名空间工厂（Namespace Factory）

命名空间工厂用于动态创建隔离的后端实例，常用于多租户场景：

```typescript
import { NamespaceFactory, StoreBackend } from "deepagents";

const factory = new NamespaceFactory({
  // 基础配置
  baseBackend: StoreBackend,
  baseConfig: { driver: "memory" },
  // 命名空间分隔符
  separator: ":",
  // 是否缓存实例
  cacheInstances: true,
});

// 为不同租户创建隔离的存储后端
const tenantAStore = factory.create("tenant_a");
const tenantBStore = factory.create("tenant_b");

// 数据完全隔离
await tenantAStore.set("config", { theme: "dark" });
await tenantBStore.set("config", { theme: "light" });

// tenantA 的 config 不会影响 tenantB
const configA = await tenantAStore.get("config"); // { theme: "dark" }
const configB = await tenantBStore.get("config"); // { theme: "light" }
```

## 从废弃 API 迁移

如果您正在从旧版本的 API 迁移，请参考以下对照表：

| 旧 API | 新 API | 说明 |
|:-------|:-------|:-----|
| `createFileSystem()` | `new FilesystemBackend()` | 文件系统后端改为类实例化 |
| `executeShell(cmd)` | `shellBackend.exec(cmd)` | Shell 执行改为后端方法调用 |
| `getStorage(ns)` | `new StoreBackend({ namespace: ns })` | 存储改为命名空间后端 |
| `CompositeBackend.create()` | `new CompositeBackend()` | 构造函数统一为 `new` |
| `backend.use(middleware)` | `backend.hooks` | 中间件改为策略钩子 |

### 迁移示例

**旧代码：**
```typescript
import { createFileSystem, executeShell, CompositeBackend } from "deepagents-old";

const fs = createFileSystem("/workspace");
const backend = CompositeBackend.create({
  fs,
  shell: { execute: executeShell },
});
```

**新代码：**
```typescript
import { FilesystemBackend, LocalShellBackend, CompositeBackend } from "deepagents";

const backend = new CompositeBackend({
  backends: {
    fs: new FilesystemBackend({ root: "/workspace" }),
    shell: new LocalShellBackend({ cwd: "/workspace" }),
  },
});
```

> **建议**：迁移时建议逐步替换，先确保单个后端正常工作，再组合使用。可以利用 `CompositeBackend` 的兼容模式平滑过渡。
