# 第二十二章 编程式子代理

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

编程式子代理（Programmatic Sub-Agent）允许开发者从解释器代码中直接派发和管理子代理。与声明式配置不同，编程式子代理提供了更精细的控制能力，支持复杂的编排模式。本章介绍编程式子代理的核心概念、实现方式以及常见的编排模式。

## 概述

编程式子代理模式将子代理的派发和编排逻辑嵌入到宿主语言（如 TypeScript）的控制流中，使开发者能够利用语言本身的条件、循环、错误处理等能力来构建复杂的代理工作流。

### 从解释器代码派发子代理

```typescript
// 编程式子代理接口
interface ProgrammaticSubAgent {
  // 子代理标识
  id: string;
  // 子代理类型
  type: string;
  // 执行方法
  execute(input: any): Promise<any>;
  // 取消执行
  cancel(): Promise<void>;
  // 获取状态
  getStatus(): SubAgentStatus;
}

// 子代理派发器
class SubAgentDispatcher {
  private registry: SubAgentRegistry;
  private executor: SubAgentExecutor;

  constructor(registry: SubAgentRegistry) {
    this.registry = registry;
    this.executor = new SubAgentExecutor();
  }

  // 派发子代理
  async dispatch<T>(
    type: string,
    input: any,
    options?: DispatchOptions
  ): Promise<T> {
    // 从注册表获取子代理定义
    const definition = this.registry.get(type);
    if (!definition) {
      throw new Error(`未注册的子代理类型: ${type}`);
    }

    // 创建子代理实例
    const instance = this.createInstance(definition, input, options);

    // 执行子代理
    return this.executor.execute<T>(instance);
  }

  // 并行派发多个子代理
  async dispatchParallel<T>(
    tasks: Array<{ type: string; input: any }>
  ): Promise<T[]> {
    const promises = tasks.map((task) =>
      this.dispatch<T>(task.type, task.input)
    );

    return Promise.all(promises);
  }

  // 串行派发子代理
  async dispatchSequential<T>(
    tasks: Array<{ type: string; input: any }>,
    chainResults: boolean = false
  ): Promise<T[]> {
    const results: T[] = [];
    let previousResult: any = null;

    for (const task of tasks) {
      const input = chainResults
        ? { ...task.input, previousResult }
        : task.input;

      const result = await this.dispatch<T>(task.type, input);
      results.push(result);
      previousResult = result;
    }

    return results;
  }
}

// 使用示例：从解释器代码派发子代理
async function processUserRequest(request: UserRequest): Promise<Response> {
  const dispatcher = new SubAgentDispatcher(subAgentRegistry);

  // 1. 分析用户意图
  const intent = await dispatcher.dispatch<Intent>(
    "intent_analyzer",
    { query: request.query }
  );

  // 2. 根据意图选择处理策略
  if (intent.type === "code_generation") {
    // 代码生成流程
    const code = await dispatcher.dispatch<GeneratedCode>(
      "code_generator",
      { intent, context: request.context }
    );

    const review = await dispatcher.dispatch<CodeReview>(
      "code_reviewer",
      { code, requirements: intent.requirements }
    );

    return { code, review };
  } else if (intent.type === "research") {
    // 研究流程：并行搜索多个来源
    const results = await dispatcher.dispatchParallel<SearchResult>([
      { type: "web_search", input: { query: intent.query } },
      { type: "knowledge_base", input: { query: intent.query } },
      { type: "academic_search", input: { query: intent.query } },
    ]);

    const synthesis = await dispatcher.dispatch<Synthesis>(
      "result_synthesizer",
      { results }
    );

    return { synthesis };
  }

  throw new Error(`不支持的意图类型: ${intent.type}`);
}
```

> **注意**：编程式子代理提供了极大的灵活性，但也增加了代码复杂度。建议在简单场景中使用声明式配置，仅在需要复杂控制流时使用编程式方法。

## 编排模式

编程式子代理支持多种编排模式，每种模式适用于不同的场景。

### 分类并执行

分类并执行（Classify and Execute）模式首先对输入进行分类，然后根据分类结果执行相应的处理逻辑。

```typescript
// 分类并执行模式
class ClassifyAndExecutePattern {
  private classifier: SubAgentDispatcher;
  private executors: Map<string, SubAgentDispatcher>;

  // 执行分类并处理
  async execute(input: any): Promise<any> {
    // 1. 分类输入
    const classification = await this.classifier.dispatch<Classification>(
      "classifier",
      { input }
    );

    // 2. 根据分类选择执行器
    const executor = this.executors.get(classification.category);
    if (!executor) {
      throw new Error(`未找到分类 ${classification.category} 的执行器`);
    }

    // 3. 执行对应的处理逻辑
    const result = await executor.dispatch(classification.category, {
      input,
      classification,
    });

    return {
      classification,
      result,
    };
  }
}

// 使用示例：客户服务请求分类处理
const customerServicePattern = new ClassifyAndExecutePattern({
  classifier: "request_classifier",
  executors: new Map([
    ["technical_support", "tech_support_agent"],
    ["billing_inquiry", "billing_agent"],
    ["product_question", "product_agent"],
    ["complaint", "complaint_handler"],
    ["general", "general_assistant"],
  ]),
});

// 处理客户请求
const response = await customerServicePattern.execute({
  message: "我的订单 #12345 还没有发货，已经等了一周了",
});
// 分类结果: complaint
// 执行器: complaint_handler
// 响应: 道歉 + 查询订单状态 + 提供补偿方案
```

### 扇出并综合

扇出并综合（Fan-Out and Synthesize）模式将任务分解为多个并行子任务，然后综合所有结果。

```typescript
// 扇出并综合模式
class FanOutAndSynthesizePattern {
  private dispatcher: SubAgentDispatcher;

  // 执行扇出并综合
  async execute<T>(
    input: any,
    fanOutConfig: FanOutConfig,
    synthesizerType: string
  ): Promise<T> {
    // 1. 扇出：并行执行多个子任务
    const fanOutTasks = this.createFanOutTasks(input, fanOutConfig);
    const results = await this.dispatcher.dispatchParallel(fanOutTasks);

    // 2. 综合：合并所有结果
    const synthesized = await this.dispatcher.dispatch<T>(
      synthesizerType,
      {
        originalInput: input,
        results,
      }
    );

    return synthesized;
  }

  // 创建扇出任务
  private createFanOutTasks(
    input: any,
    config: FanOutConfig
  ): Array<{ type: string; input: any }> {
    switch (config.strategy) {
      case "split_input":
        // 将输入拆分为多个部分
        return this.splitInput(input, config.splits);

      case "vary_parameters":
        // 对同一输入使用不同参数
        return this.varyParameters(input, config.parameters);

      case "different_sources":
        // 从不同来源处理同一查询
        return config.sources.map((source) => ({
          type: source.type,
          input: { ...input, source: source.config },
        }));

      default:
        throw new Error(`未知的扇出策略: ${config.strategy}`);
    }
  }
}

// 使用示例：多角度研究
const researchPattern = new FanOutAndSynthesizePattern(dispatcher);

const researchResult = await researchPattern.execute<ResearchReport>(
  { topic: "量子计算在密码学中的应用" },
  {
    strategy: "different_sources",
    sources: [
      { type: "academic_search", config: { databases: ["arxiv", "ieee"] } },
      { type: "news_search", config: { timeRange: "last_year" } },
      { type: "expert_network", config: { domain: "quantum_computing" } },
      { type: "patent_search", config: { jurisdictions: ["us", "eu", "cn"] } },
    ],
  },
  "research_synthesizer"
);

// 使用示例：多模型投票
const votingPattern = new FanOutAndSynthesizePattern(dispatcher);

const decision = await votingPattern.execute<VotingResult>(
  { proposal: "采用微服务架构重构系统" },
  {
    strategy: "vary_parameters",
    parameters: [
      { model: "gpt-4", perspective: "technical" },
      { model: "claude-3", perspective: "business" },
      { model: "gemini-pro", perspective: "risk" },
    ],
  },
  "voting_aggregator"
);
```

| 扇出策略 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| split_input | 大数据处理 | 并行加速 | 需要拆分逻辑 |
| vary_parameters | 多视角分析 | 结果多样 | 综合复杂 |
| different_sources | 信息收集 | 全面覆盖 | 数据格式不一 |

### 对抗验证

对抗验证（Adversarial Validation）模式使用一个子代理生成内容，另一个子代理进行批判和验证。

```typescript
// 对抗验证模式
class AdversarialValidationPattern {
  private dispatcher: SubAgentDispatcher;
  private config: AdversarialConfig;

  // 执行对抗验证
  async execute<T>(input: any): Promise<AdversarialResult<T>> {
    let currentVersion: T | null = null;
    let iteration = 0;
    const history: AdversarialIteration<T>[] = [];

    while (iteration < this.config.maxIterations) {
      iteration++;

      // 1. 生成器：生成或改进内容
      const generated = await this.dispatcher.dispatch<T>(
        "generator",
        {
          input,
          previousVersion: currentVersion,
          feedback: history.length > 0
            ? history[history.length - 1].critique
            : null,
        }
      );

      // 2. 批判器：评估生成的内容
      const critique = await this.dispatcher.dispatch<Critique>(
        "critic",
        {
          input,
          generated,
          criteria: this.config.validationCriteria,
        }
      );

      // 记录历史
      history.push({
        iteration,
        generated,
        critique,
        timestamp: new Date(),
      });

      // 3. 检查是否通过验证
      if (this.isAccepted(critique)) {
        return {
          result: generated,
          iterations: iteration,
          history,
          accepted: true,
        };
      }

      currentVersion = generated;
    }

    // 达到最大迭代次数，返回最佳版本
    const bestIteration = this.selectBest(history);
    return {
      result: bestIteration.generated,
      iterations: iteration,
      history,
      accepted: false,
    };
  }

  // 判断是否通过验证
  private isAccepted(critique: Critique): boolean {
    return (
      critique.score >= this.config.acceptanceThreshold &&
      critique.criticalIssues.length === 0
    );
  }

  // 选择最佳迭代
  private selectBest(history: AdversarialIteration<T>[]): AdversarialIteration<T> {
    return history.reduce((best, current) =>
      current.critique.score > best.critique.score ? current : best
    );
  }
}

// 使用示例：代码生成与验证
const codeGenerationPattern = new AdversarialValidationPattern(dispatcher, {
  maxIterations: 3,
  acceptanceThreshold: 4.0,
  validationCriteria: [
    "correctness",
    "efficiency",
    "readability",
    "security",
  ],
});

const codeResult = await codeGenerationPattern.execute<GeneratedCode>({
  requirements: "实现一个高性能的 LRU 缓存",
  language: "typescript",
  constraints: {
    maxSize: 1000,
    threadSafe: true,
  },
});

console.log(`代码生成完成: ${codeResult.iterations} 次迭代`);
console.log(`验证通过: ${codeResult.accepted}`);
console.log(`最终评分: ${codeResult.history[codeResult.history.length - 1].critique.score}`);
```

### 生成并过滤

生成并过滤（Generate and Filter）模式先生成多个候选结果，然后通过过滤器选择最佳结果。

```typescript
// 生成并过滤模式
class GenerateAndFilterPattern {
  private dispatcher: SubAgentDispatcher;
  private config: GenerateFilterConfig;

  // 执行生成并过滤
  async execute<T>(input: any): Promise<FilteredResult<T>> {
    // 1. 生成多个候选结果
    const candidates = await this.generateCandidates<T>(input);

    // 2. 评分所有候选
    const scoredCandidates = await this.scoreCandidates<T>(
      input,
      candidates
    );

    // 3. 过滤并选择最佳结果
    const filtered = this.filterCandidates(scoredCandidates);

    // 4. 可选：对最佳结果进行精炼
    let finalResult = filtered[0].candidate;
    if (this.config.refinementEnabled) {
      finalResult = await this.refineResult<T>(input, finalResult);
    }

    return {
      result: finalResult,
      candidates: scoredCandidates,
      selected: filtered,
    };
  }

  // 生成候选结果
  private async generateCandidates<T>(input: any): Promise<T[]> {
    const generationTasks = Array.from(
      { length: this.config.candidateCount },
      (_, i) => ({
        type: "generator",
        input: {
          ...input,
          variation: i,
          temperature: this.config.temperatureVariations?.[i] || 0.7,
        },
      })
    );

    return this.dispatcher.dispatchParallel<T>(generationTasks);
  }

  // 评分候选结果
  private async scoreCandidates<T>(
    input: any,
    candidates: T[]
  ): Promise<Array<ScoredCandidate<T>>> {
    const scoringTasks = candidates.map((candidate, index) => ({
      type: "scorer",
      input: {
        originalInput: input,
        candidate,
        criteria: this.config.scoringCriteria,
      },
    }));

    const scores = await this.dispatcher.dispatchParallel<Score>(
      scoringTasks
    );

    return candidates.map((candidate, index) => ({
      candidate,
      score: scores[index],
      index,
    }));
  }

  // 过滤候选结果
  private filterCandidates<T>(
    scored: Array<ScoredCandidate<T>>
  ): Array<ScoredCandidate<T>> {
    // 按分数排序
    const sorted = scored.sort((a, b) => b.score.total - a.score.total);

    // 应用过滤条件
    const filtered = sorted.filter((item) => {
      // 最低分数阈值
      if (item.score.total < this.config.minScore) return false;

      // 必须满足的条件
      for (const criterion of this.config.requiredCriteria) {
        if (item.score.dimensions[criterion] < this.config.minDimensionScore) {
          return false;
        }
      }

      return true;
    });

    // 返回前 N 个
    return filtered.slice(0, this.config.topN || 1);
  }
}

// 使用示例：创意文案生成
const copywritingPattern = new GenerateAndFilterPattern(dispatcher, {
  candidateCount: 5,
  temperatureVariations: [0.5, 0.7, 0.9, 1.0, 1.2],
  scoringCriteria: ["creativity", "clarity", "persuasiveness", "brand_alignment"],
  minScore: 3.5,
  requiredCriteria: ["brand_alignment"],
  topN: 3,
  refinementEnabled: true,
});

const copyResult = await copywritingPattern.execute<MarketingCopy>({
  product: "AI 编程助手",
  targetAudience: "专业开发者",
  tone: "专业但友好",
  maxLength: 200,
});

console.log("生成的文案:");
copyResult.selected.forEach((item, i) => {
  console.log(`\n--- 选项 ${i + 1} (评分: ${item.score.total}) ---`);
  console.log(item.candidate);
});
```

### 锦标赛

锦标赛（Tournament）模式通过多轮淘汰赛选出最佳结果。

```typescript
// 锦标赛模式
class TournamentPattern {
  private dispatcher: SubAgentDispatcher;
  private config: TournamentConfig;

  // 执行锦标赛
  async execute<T>(input: any): Promise<TournamentResult<T>> {
    // 1. 生成初始候选池
    let pool = await this.generateInitialPool<T>(input);
    const rounds: TournamentRound<T>[] = [];

    // 2. 多轮淘汰
    let roundNumber = 0;
    while (pool.length > 1 && roundNumber < this.config.maxRounds) {
      roundNumber++;

      // 分组对决
      const matches = this.createMatches(pool);
      const matchResults = await this.executeMatches<T>(
        input,
        matches
      );

      // 选择胜者
      pool = this.selectWinners(matchResults);

      rounds.push({
        round: roundNumber,
        matches: matchResults,
        survivors: pool,
      });
    }

    // 3. 返回最终胜者
    return {
      winner: pool[0],
      rounds,
      totalRounds: roundNumber,
    };
  }

  // 生成初始候选池
  private async generateInitialPool<T>(input: any): Promise<T[]> {
    const tasks = Array.from(
      { length: this.config.poolSize },
      (_, i) => ({
        type: this.config.generatorType,
        input: {
          ...input,
          strategy: this.config.generationStrategies[i % this.config.generationStrategies.length],
        },
      })
    );

    return this.dispatcher.dispatchParallel<T>(tasks);
  }

  // 创建对决配对
  private createMatches<T>(pool: T[]): Array<[T, T]> {
    const matches: Array<[T, T]> = [];

    // 随机配对
    const shuffled = [...pool].sort(() => Math.random() - 0.5);
    for (let i = 0; i < shuffled.length - 1; i += 2) {
      matches.push([shuffled[i], shuffled[i + 1]]);
    }

    // 如果是奇数，最后一个直接晋级
    if (shuffled.length % 2 !== 0) {
      matches.push([shuffled[shuffled.length - 1], null as any]);
    }

    return matches;
  }

  // 执行对决
  private async executeMatches<T>(
    input: any,
    matches: Array<[T, T]>
  ): Promise<Array<MatchResult<T>>> {
    const matchTasks = matches.map(([a, b]) => ({
      type: "judge",
      input: {
        originalInput: input,
        candidateA: a,
        candidateB: b,
        criteria: this.config.comparisonCriteria,
      },
    }));

    const judgments = await this.dispatcher.dispatchParallel<Judgment>(
      matchTasks
    );

    return matches.map(([a, b], index) => ({
      candidateA: a,
      candidateB: b,
      judgment: judgments[index],
      winner: judgments[index].winner === "A" ? a : b,
    }));
  }

  // 选择胜者
  private selectWinners<T>(matchResults: Array<MatchResult<T>>): T[] {
    return matchResults.map((result) => result.winner);
  }
}

// 使用示例：最佳算法选择
const algorithmTournament = new TournamentPattern(dispatcher, {
  poolSize: 8,
  maxRounds: 3,
  generatorType: "algorithm_generator",
  generationStrategies: [
    "dynamic_programming",
    "greedy",
    "divide_and_conquer",
    "backtracking",
    "graph_based",
    "mathematical",
    "heuristic",
    "hybrid",
  ],
  comparisonCriteria: [
    "correctness",
    "time_complexity",
    "space_complexity",
    "readability",
  ],
});

const tournamentResult = await algorithmTournament.execute<Algorithm>({
  problem: "旅行商问题",
  constraints: {
    maxNodes: 100,
    timeLimit: "1s",
  },
});

console.log(`锦标赛完成: ${tournamentResult.totalRounds} 轮`);
console.log(`获胜算法: ${tournamentResult.winner.name}`);
console.log(`复杂度: ${tournamentResult.winner.timeComplexity}`);
```

### 循环直到完成

循环直到完成（Loop Until Done）模式持续执行直到满足终止条件。

```typescript
// 循环直到完成模式
class LoopUntilDonePattern {
  private dispatcher: SubAgentDispatcher;
  private config: LoopConfig;

  // 执行循环
  async execute<T>(input: any): Promise<LoopResult<T>> {
    let currentResult: T | null = null;
    let iteration = 0;
    const history: LoopIteration<T>[] = [];

    while (true) {
      iteration++;

      // 1. 执行主体逻辑
      const result = await this.dispatcher.dispatch<T>(
        this.config.bodyType,
        {
          input,
          previousResult: currentResult,
          iteration,
        }
      );

      // 2. 检查终止条件
      const check = await this.dispatcher.dispatch<TerminationCheck>(
        "termination_checker",
        {
          input,
          result,
          iteration,
          history,
          conditions: this.config.terminationConditions,
        }
      );

      // 记录历史
      history.push({
        iteration,
        result,
        check,
        timestamp: new Date(),
      });

      // 3. 判断是否终止
      if (check.shouldTerminate) {
        return {
          result,
          iterations: iteration,
          history,
          terminationReason: check.reason,
        };
      }

      // 4. 检查是否超过最大迭代次数
      if (iteration >= this.config.maxIterations) {
        return {
          result,
          iterations: iteration,
          history,
          terminationReason: "达到最大迭代次数",
        };
      }

      currentResult = result;
    }
  }
}

// 使用示例：迭代优化
const optimizationLoop = new LoopUntilDonePattern(dispatcher, {
  bodyType: "optimizer",
  maxIterations: 10,
  terminationConditions: [
    {
      type: "convergence",
      threshold: 0.01, // 改进幅度小于 1% 时停止
    },
    {
      type: "target_reached",
      target: 0.95, // 达到 95% 准确率时停止
    },
  ],
});

const optimizationResult = await optimizationLoop.execute<ModelConfig>({
  dataset: "training_data",
  baseline_accuracy: 0.75,
});

console.log(`优化完成: ${optimizationResult.iterations} 次迭代`);
console.log(`终止原因: ${optimizationResult.terminationReason}`);
console.log(`最终准确率: ${optimizationResult.result.accuracy}`);

// 使用示例：自动修复循环
const autoFixLoop = new LoopUntilDonePattern(dispatcher, {
  bodyType: "code_fixer",
  maxIterations: 5,
  terminationConditions: [
    {
      type: "all_tests_passing",
    },
    {
      type: "no_errors",
    },
  ],
});

const fixResult = await autoFixLoop.execute<FixedCode>({
  code: buggyCode,
  tests: testSuite,
  errors: initialErrors,
});

console.log(`自动修复完成: ${fixResult.iterations} 次迭代`);
console.log(`终止原因: ${fixResult.terminationReason}`);
console.log(`修复后的代码:\n${fixResult.result.code}`);
```

> **提示**：循环直到完成模式应始终设置最大迭代次数限制，防止无限循环。

## 模式组合

在实际应用中，这些模式可以组合使用，构建更复杂的代理工作流。

```typescript
// 组合模式示例：智能内容创作系统
class ContentCreationPipeline {
  private dispatcher: SubAgentDispatcher;

  async createContent(request: ContentRequest): Promise<Content> {
    // 阶段 1：分类并执行 - 确定内容类型
    const classification = await this.dispatcher.dispatch<Classification>(
      "content_classifier",
      { request }
    );

    // 阶段 2：扇出并综合 - 多角度研究
    const research = await this.fanOutResearch(classification);

    // 阶段 3：生成并过滤 - 生成多个草稿
    const drafts = await this.generateAndFilterDrafts(research);

    // 阶段 4：对抗验证 - 审查和改进
    const validated = await this.adversarialValidation(drafts[0]);

    // 阶段 5：循环直到完成 - 最终精炼
    const final = await this.refineUntilDone(validated);

    return final;
  }

  private async fanOutResearch(classification: Classification): Promise<Research> {
    // 并行从多个来源收集信息
    const sources = ["web", "academic", "internal_kb", "expert_opinions"];
    const tasks = sources.map((source) => ({
      type: `${source}_researcher`,
      input: { topic: classification.topic },
    }));

    const results = await this.dispatcher.dispatchParallel(tasks);
    return this.dispatcher.dispatch("research_synthesizer", { results });
  }

  private async generateAndFilterDrafts(research: Research): Promise<Draft[]> {
    // 生成多个草稿并选择最佳的
    const pattern = new GenerateAndFilterPattern(this.dispatcher, {
      candidateCount: 5,
      scoringCriteria: ["relevance", "quality", "originality"],
      minScore: 3.0,
      topN: 3,
    });

    const result = await pattern.execute<Draft>({ research });
    return result.selected.map((s) => s.candidate);
  }

  private async adversarialValidation(draft: Draft): Promise<Draft> {
    // 使用对抗验证改进草稿
    const pattern = new AdversarialValidationPattern(this.dispatcher, {
      maxIterations: 3,
      acceptanceThreshold: 4.0,
    });

    const result = await pattern.execute<Draft>({ draft });
    return result.result;
  }

  private async refineUntilDone(draft: Draft): Promise<Content> {
    // 迭代精炼直到满足质量标准
    const pattern = new LoopUntilDonePattern(this.dispatcher, {
      bodyType: "content_refiner",
      maxIterations: 5,
      terminationConditions: [
        { type: "quality_score_above", threshold: 4.5 },
        { type: "no_suggestions_remaining" },
      ],
    });

    const result = await pattern.execute<Content>({ draft });
    return result.result;
  }
}
```
