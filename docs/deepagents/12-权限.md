# 第十二章 权限（Permissions）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

权限系统通过声明式的 allow/deny/interrupt 规则，控制代理（Agent）在运行时可以执行哪些操作。这套机制是 Deep Agents 安全模型的核心，确保代理在受控范围内运行。

## 声明式规则概述

Deep Agents 采用三层权限模型：

| 规则类型 | 作用 | 优先级 |
|:---------|:-----|:-------|
| `allow` | 允许代理执行特定操作 | 最低 |
| `deny`  | 禁止代理执行特定操作 | 中等 |
| `interrupt` | 执行前需要人工确认 | 最高 |

> **注意**：规则按声明顺序评估，`interrupt` 优先级最高，会覆盖 `allow` 和 `deny` 的设定。

## 规则结构

每条权限规则由以下字段组成：

```typescript
interface PermissionRule {
  // 规则类型
  type: "allow" | "deny" | "interrupt";
  // 匹配的操作模式（支持通配符）
  pattern: string;
  // 可选：适用的后端类型
  backend?: string;
  // 可选：适用的工具名称
  tool?: string;
  // 可选：规则描述
  description?: string;
}
```

### 规则匹配模式

规则使用 glob 模式匹配操作路径：

| 模式示例 | 含义 |
|:---------|:-----|
| `fs:read:*` | 允许读取所有文件 |
| `fs:write:/data/**` | 允许写入 `/data` 目录下的所有文件 |
| `shell:exec:*` | 允许执行所有 Shell 命令 |
| `shell:exec:rm *` | 匹配所有 `rm` 命令 |
| `store:get:secrets/*` | 匹配 secrets 命名空间下的所有读取操作 |

## 基本示例

### 配置文件方式

```yaml
# permissions.yaml
permissions:
  - type: allow
    pattern: "fs:read:*"
    description: "允许读取所有文件"

  - type: allow
    pattern: "fs:write:/workspace/**"
    description: "允许写入工作区目录"

  - type: deny
    pattern: "fs:write:/etc/**"
    description: "禁止修改系统配置文件"

  - type: interrupt
    pattern: "shell:exec:*"
    description: "执行 Shell 命令前需要人工确认"
```

### 编程式配置

```typescript
import { Agent } from "deepagents";

const agent = new Agent({
  model: "gpt-4",
  permissions: [
    // 允许读取所有文件
    { type: "allow", pattern: "fs:read:*" },
    // 允许写入工作区
    { type: "allow", pattern: "fs:write:/workspace/**" },
    // 禁止修改系统文件
    { type: "deny", pattern: "fs:write:/etc/**" },
    // Shell 命令需要确认
    { type: "interrupt", pattern: "shell:exec:*" },
  ],
});
```

## 子代理权限

子代理（Sub-agent）继承父代理的权限规则，但可以追加更严格的限制：

```typescript
import { Agent, SubAgent } from "deepagents";

const parentAgent = new Agent({
  model: "gpt-4",
  permissions: [
    { type: "allow", pattern: "fs:read:*" },
    { type: "allow", pattern: "fs:write:/workspace/**" },
  ],
});

// 子代理继承父代理权限，并追加更严格的限制
const childAgent = new SubAgent({
  parent: parentAgent,
  // 追加的权限限制（仅在子代理范围内生效）
  additionalPermissions: [
    { type: "deny", pattern: "fs:write:/workspace/secrets/**" },
    { type: "interrupt", pattern: "fs:write:*" },
  ],
});
```

> **注意**：子代理的 `additionalPermissions` 只能进一步限制权限，不能扩大父代理授予的权限范围。

### 权限继承链示例

```
Root Agent (allow: fs:read:*, fs:write:/workspace/**)
  └── SubAgent A (继承 + deny: fs:write:/workspace/secrets/**)
       └── SubAgent B (继承 + interrupt: fs:write:*)
```

SubAgent B 的最终权限效果：
- 可以读取所有文件（继承自 Root）
- 写入操作需要人工确认（interrupt 覆盖了 allow）
- 仍然无法写入 secrets 目录（deny 持续生效）

## 组合后端权限

当使用 `CompositeBackend` 组合多个后端时，权限规则可以按后端类型进行细粒度控制：

```typescript
import { Agent, CompositeBackend } from "deepagents";

const backend = new CompositeBackend({
  backends: {
    fs: new FilesystemBackend({ root: "/workspace" }),
    shell: new LocalShellBackend(),
    store: new StoreBackend({ namespace: "app" }),
  },
});

const agent = new Agent({
  model: "gpt-4",
  backend,
  permissions: [
    // 文件系统：允许读取，写入需要确认
    { type: "allow", pattern: "fs:read:*", backend: "fs" },
    { type: "interrupt", pattern: "fs:write:*", backend: "fs" },

    // Shell：完全禁止
    { type: "deny", pattern: "shell:*", backend: "shell" },

    // Store：允许所有操作
    { type: "allow", pattern: "store:*", backend: "store" },
  ],
});
```

### 后端权限优先级

当多个规则匹配同一操作时，按以下顺序决定优先级：

1. **精确匹配** 优先于通配符匹配
2. **后端特定规则** 优先于全局规则
3. **后声明的规则** 优先于先声明的规则
4. **`interrupt`** 始终覆盖 `allow` 和 `deny`

```typescript
const permissions = [
  // 全局允许读取
  { type: "allow", pattern: "fs:read:*" },
  // 但 fs 后端的特定路径需要确认
  { type: "interrupt", pattern: "fs:read:/workspace/config.yaml", backend: "fs" },
  // 最终效果：读取 config.yaml 需要人工确认，其他读取操作直接允许
];
```

> **建议**：在生产环境中，建议对所有写入操作设置 `interrupt` 规则，确保人工审核关键操作。

## 权限评估流程

```
请求操作
    │
    ▼
┌─────────────────┐
│ 按顺序匹配规则  │
└────────┬────────┘
         │
    ┌────┴────┐
    │ 匹配到？ │
    └────┬────┘
    No   │   Yes
    │    │    │
    ▼    │    ▼
  拒绝   │  ┌─────────┐
  操作   │  │ 规则类型 │
         │  └────┬────┘
         │       │
    ┌────┼───────┼────┐
    │    │       │    │
    ▼    ▼       ▼    ▼
 allow deny interrupt
    │    │       │
    ▼    ▼       ▼
  执行  拒绝   等待人工确认
```

> **提示**：如果没有任何规则匹配操作，默认行为是拒绝（deny），这是为了确保安全的默认策略。
