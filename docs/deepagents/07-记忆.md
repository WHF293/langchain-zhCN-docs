# 第七章 记忆（Memory）

> 来源: [Deep Agents 官方文档](https://docs.langchain.com/deep-agents)

---

记忆系统是 Deep Agents 的核心功能之一，它使代理能够在不同会话间保持上下文、学习用户偏好并持续改进。本章详细介绍记忆系统的各种类型和使用方法。

## 长期记忆（Long-Term Memory）

长期记忆允许代理在多次对话间保留信息，提供连贯的用户体验：

```typescript
// 配置长期记忆
import { LongTermMemory } from "@langchain/deep-agents/memory";

const memory = new LongTermMemory({
  // 存储后端
  storage: {
    type: "postgres",  // 'postgres' | 'redis' | 'custom'
    connection: process.env.DATABASE_URL
  },

  // 记忆保留策略
  retention: {
    // 默认保留时间（天）
    defaultTTL: 90,

    // 重要记忆永久保留
    permanentCategories: ["user_preferences", "key_facts"]
  },

  // 检索配置
  retrieval: {
    // 每次检索的最大记忆数
    maxResults: 10,

    // 相似度阈值
    similarityThreshold: 0.7
  }
});

// 创建带记忆的代理
const agent = deepAgents.create({
  memory: memory
});
```

### 记忆存储结构

```typescript
// 记忆条目结构
interface MemoryEntry {
  // 唯一标识
  id: string;

  // 记忆类型
  type: "fact" | "preference" | "context" | "experience";

  // 记忆内容
  content: string;

  // 元数据
  metadata: {
    // 创建时间
    createdAt: Date;

    // 最后访问时间
    lastAccessedAt: Date;

    // 访问次数
    accessCount: number;

    // 重要性评分（0-1）
    importance: number;

    // 来源
    source: string;
  };

  // 关联实体
  entities: string[];

  // 嵌入向量（用于语义搜索）
  embedding?: number[];
}
```

## Agent 范围记忆（Agent-Scoped Memory）

Agent 范围的记忆与特定代理实例绑定，用于存储代理自身的工作记忆：

```typescript
// Agent 范围记忆配置
const agentMemory = {
  scope: "agent",

  // 工作记忆：当前任务相关
  workingMemory: {
    maxSize: 1000,  // 最大条目数
    ttl: 3600000    // 1小时过期
  },

  // 程序记忆：如何执行任务
  proceduralMemory: {
    enabled: true,
    learnFromInteractions: true
  }
};

const agent = deepAgents.create({
  memory: {
    agentScope: agentMemory
  }
});

// 访问 Agent 记忆
await agent.memory.set("current_task", {
  goal: "分析销售数据",
  progress: 0.6,
  findings: ["Q1增长15%", "产品A表现最佳"]
});

const taskInfo = await agent.memory.get("current_task");
```

## 用户范围记忆（User-Scoped Memory）

用户范围的记忆与特定用户绑定，存储用户偏好和历史：

```typescript
// 用户范围记忆
const userMemoryConfig = {
  scope: "user",

  // 用户偏好
  preferences: {
    // 自动学习用户偏好
    autoLearn: true,

    // 偏好类别
    categories: [
      "communication_style",  // 沟通风格
      "response_length",      // 回复长度
      "technical_level",      // 技术水平
      "language",             // 语言偏好
      "topics"                // 兴趣话题
    ]
  },

  // 交互历史
  interactionHistory: {
    // 保留最近的对话数
    recentConversations: 50,

    // 摘要生成
    autoSummarize: true
  }
};

// 使用用户记忆
const agent = deepAgents.create({
  memory: {
    userScope: userMemoryConfig
  }
});

// 带用户上下文调用
await agent.invoke({
  input: "帮我写个邮件",
  userId: "user-123",
  // 代理会自动使用该用户的记忆和偏好
});
```

## 组织级别记忆（Organization-Level Memory）

组织级别记忆在团队或组织内共享知识：

```typescript
// 组织记忆配置
const orgMemoryConfig = {
  scope: "organization",

  // 共享知识库
  knowledgeBase: {
    // 文档集合
    collections: ["docs", "policies", "procedures"],

    // 访问控制
    accessControl: {
      roles: {
        "admin": ["read", "write", "delete"],
        "member": ["read"],
        "guest": []
      }
    }
  },

  // 团队最佳实践
  bestPractices: {
    enabled: true,
    autoCapture: true
  },

  // 共享上下文
  sharedContext: {
    // 组织特定术语
    terminology: {
      "KPI": "关键绩效指标",
      "OKR": "目标与关键成果"
    },

    // 常用模板
    templates: ["report", "proposal", "meeting-notes"]
  }
};

// 使用组织记忆
const agent = deepAgents.create({
  memory: {
    orgScope: orgMemoryConfig,
    orgId: "org-456"
  }
});
```

## 情景记忆（Episodic Memory）

情景记忆记录具体的交互事件，用于学习和改进：

```typescript
// 情景记忆配置
const episodicMemory = {
  enabled: true,

  // 记录什么类型的事件
  captureEvents: [
    "successful_task",      // 成功完成的任务
    "failed_task",          // 失败的任务
    "user_correction",      // 用户纠正
    "preference_change",    // 偏好变化
    "new_fact_learned"      // 学到的新知识
  ],

  // 情景结构
  episodeStructure: {
    // 触发条件
    trigger: "string",

    // 执行的动作
    actions: ["string"],

    // 结果
    outcome: "success" | "failure" | "partial",

    // 学到的经验
    lessons: ["string"],

    // 情感标记（用于重要性评估）
    emotionalTag: "positive" | "negative" | "neutral"
  }
};

// 从情景中学习
agent.on("episode-recorded", async (episode) => {
  // 分析情景，提取经验
  const insights = await analyzeEpisode(episode);

  // 更新代理策略
  if (insights.shouldAdjustBehavior) {
    await agent.updateStrategy(insights.adjustment);
  }
});

// 查询相关情景
const relevantEpisodes = await agent.memory.episodic.search({
  query: "如何处理客户投诉",
  limit: 5,
  minImportance: 0.6
});
```

## 后台合并（Background Consolidation / Cron）

记忆系统使用后台任务定期合并和优化记忆：

```typescript
// 配置记忆合并
const consolidationConfig = {
  // 定时任务配置
  schedule: {
    // 使用 cron 表达式
    cron: "0 2 * * *",  // 每天凌晨2点

    // 或使用间隔（毫秒）
    interval: 86400000
  },

  // 合并策略
  strategy: {
    // 重复记忆检测
    deduplication: {
      enabled: true,
      similarityThreshold: 0.95
    },

    // 重要性重评估
    importanceReEvaluation: {
      enabled: true,
      decayRate: 0.01  // 每天衰减1%
    },

    // 记忆压缩
    compression: {
      enabled: true,
      // 合并相似记忆
      mergeSimilar: true,
      // 生成摘要
      summarizeLong: true,
      // 摘要阈值（字符数）
      summarizeThreshold: 1000
    },

    // 过期清理
    expiration: {
      enabled: true,
      // 低重要性记忆过期时间（天）
      lowImportanceTTL: 30,
      // 中重要性记忆过期时间（天）
      mediumImportanceTTL: 180
    }
  },

  // 批处理大小
  batchSize: 100,

  // 并发处理数
  concurrency: 5
};

// 启动合并任务
const consolidationJob = await agent.memory.startConsolidation(
  consolidationConfig
);

// 监控合并进度
consolidationJob.on("progress", (progress) => {
  console.log(`合并进度: ${progress.percentage}%`);
  console.log(`已处理: ${progress.processed}/${progress.total}`);
});

// 手动触发合并
await agent.memory.consolidateNow();
```

## 只读 vs 可写记忆（Read-Only vs Writable Memory）

根据使用场景，可以配置记忆的访问权限：

```typescript
// 只读记忆配置
const readOnlyMemory = {
  // 外部知识源，代理只能读取
  sources: [
    {
      name: "company-wiki",
      type: "confluence",
      access: "read-only",
      connection: {
        url: "https://wiki.company.com",
        token: process.env.CONFLUENCE_TOKEN
      }
    },
    {
      name: "product-docs",
      type: "static",
      access: "read-only",
      path: "/docs/products"
    }
  ]
};

// 可写记忆配置
const writableMemory = {
  // 代理可以修改的记忆
  stores: [
    {
      name: "user-notes",
      type: "user-scoped",
      access: "read-write",
      // 写入限制
      writeConstraints: {
        maxEntrySize: 10000,  // 单条最大10KB
        maxTotalSize: 1000000, // 总计最大1MB
        allowedTypes: ["note", "preference", "bookmark"]
      }
    },
    {
      name: "task-logs",
      type: "agent-scoped",
      access: "read-write",
      // 自动清理策略
      autoCleanup: {
        maxAge: "7d",
        maxSize: 1000
      }
    }
  ]
};

// 组合使用
const agent = deepAgents.create({
  memory: {
    readOnly: readOnlyMemory,
    writable: writableMemory
  }
});
```

## 并发写入（Concurrent Writes）

在多代理或高并发场景下，处理记忆写入冲突：

```typescript
// 并发写入配置
const concurrencyConfig = {
  // 锁策略
  locking: {
    type: "optimistic",  // 'optimistic' | 'pessimistic'

    // 乐观锁：版本检查
    optimistic: {
      // 重试次数
      maxRetries: 3,
      // 重试间隔（毫秒）
      retryDelay: 100,
      // 退避策略
      backoff: "exponential"
    },

    // 悲观锁：直接锁定
    pessimistic: {
      // 锁超时（毫秒）
      lockTimeout: 5000,
      // 死锁检测
      deadlockDetection: true
    }
  },

  // 冲突解决策略
  conflictResolution: {
    // 策略：'last-write-wins' | 'merge' | 'manual'
    strategy: "merge",

    // 合并策略
    merge: {
      // 字段级别合并
      fieldLevel: true,
      // 冲突时通知
      notifyOnConflict: true
    }
  },

  // 批量写入优化
  batching: {
    enabled: true,
    // 批量大小
    batchSize: 50,
    // 刷新间隔（毫秒）
    flushInterval: 1000
  }
};

// 使用事务
await agent.memory.transaction(async (tx) => {
  // 在事务中执行多个操作
  const user = await tx.get("user-123");
  user.preferences.theme = "dark";
  await tx.set("user-123", user);

  // 记录操作日志
  await tx.append("audit-log", {
    action: "update-preference",
    userId: "user-123",
    timestamp: new Date()
  });

  // 事务自动提交或回滚
});
```

## 多代理记忆（Multi-Agent Memory）

在多代理系统中共享和协调记忆：

```typescript
// 多代理记忆架构
const multiAgentMemory = {
  // 共享记忆空间
  sharedSpaces: {
    // 全局共享空间
    global: {
      type: "shared",
      access: "all-agents",
      // 同步策略
      sync: {
        mode: "realtime",  // 'realtime' | 'periodic' | 'on-demand'
        interval: 5000
      }
    },

    // 团队共享空间
    team: {
      type: "team",
      access: "team-members",
      // 团队成员列表
      members: ["agent-a", "agent-b", "agent-c"]
    },

    // 私有空间
    private: {
      type: "private",
      access: "single-agent"
    }
  },

  // 记忆同步协议
  synchronization: {
    // 冲突检测
    conflictDetection: true,

    // 一致性级别
    consistency: "eventual",  // 'strong' | 'eventual'

    // 同步过滤器
    filters: {
      // 只同步特定类型的记忆
      types: ["fact", "preference"],
      // 排除敏感信息
      excludePatterns: ["password", "token", "secret"]
    }
  },

  // 代理间通信
  communication: {
    // 记忆变更通知
    notifications: {
      enabled: true,
      // 通知类型
      types: ["memory-created", "memory-updated", "memory-deleted"]
    },

    // 记忆请求协议
    requestProtocol: {
      // 允许代理请求其他代理的记忆
      allowCrossAgentQueries: true,
      // 需要授权
      requireAuthorization: true
    }
  }
};

// 创建多代理系统
const agentSystem = deepAgents.createMultiAgent({
  agents: ["researcher", "writer", "reviewer"],
  memory: multiAgentMemory
});

// 代理 A 创建记忆
await agentSystem.agents.researcher.memory.set("research-findings", {
  topic: "AI趋势",
  findings: ["GPT-5即将发布", "多模态成为主流"]
});

// 代理 B 访问代理 A 的记忆（通过共享空间）
const findings = await agentSystem.agents.writer.memory.getShared(
  "research-findings",
  { source: "researcher" }
);

// 监听记忆变更
agentSystem.on("memory-sync", (event) => {
  console.log(`记忆同步: ${event.source} -> ${event.target}`);
  console.log(`变更类型: ${event.type}`);
  console.log(`记忆键: ${event.key}`);
});
```

> **注意**：在多代理系统中，确保正确配置访问控制，避免敏感信息在代理间不当传播。

> **提示**：对于大规模部署，建议使用分布式记忆存储（如 Redis Cluster）以提高性能和可靠性。

> **建议**：定期审查和清理过期记忆，保持记忆库的高效运行。可以使用 `agent.memory.stats()` 查看记忆使用情况。
