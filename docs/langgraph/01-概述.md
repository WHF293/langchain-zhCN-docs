# 第1章 LangGraph 概述

> 来源: [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)

---

## 什么是 LangGraph

LangGraph 是一个用于构建**有状态、多参与者应用程序**的库，特别适合创建基于 LLM（大语言模型）的 Agent（智能体）。它扩展了 LangChain 的表达式语言，提供了定义循环图的能力，使得复杂的 Agent 工作流变得更加可控和可靠。

### 核心特点

- **有状态管理**：自动管理应用程序状态，支持持久化和恢复
- **循环图支持**：支持定义包含循环的复杂工作流
- **多 Agent 协作**：支持多个 Agent 协同完成任务
- **流式输出**：支持实时流式输出中间结果
- **人类介入**：支持 Human-in-the-loop（人类介入）模式
- **容错机制**：内置错误处理和重试机制

---

## 核心概念

### 图（Graph）

图是 LangGraph 的核心抽象，它定义了 Agent 的工作流程。图由节点（Node）和边（Edge）组成，描述了任务的执行顺序和条件分支。

```typescript
import { StateGraph } from "@langchain/langgraph";

// 创建一个状态图实例
const workflow = new StateGraph(StateAnnotation);
```

### 节点（Node）

节点是图中的执行单元，每个节点代表一个具体的任务或操作。节点是一个函数，接收当前状态作为输入，返回更新后的状态。

```typescript
// 定义一个节点函数
const myNode = async (state: typeof StateAnnotation.State) => {
  // 执行具体任务
  return { messages: [...state.messages, newMessage] };
};

// 将节点添加到图中
workflow.addNode("myNode", myNode);
```

### 边（Edge）

边定义了节点之间的连接关系，决定了执行流程的走向。LangGraph 支持多种边类型：

- **普通边（Normal Edge）**：无条件连接两个节点
- **条件边（Conditional Edge）**：根据条件动态选择下一个节点
- **入口边（Entry Point）**：图的起始点
- **出口边（Exit Point）**：图的结束点

```typescript
// 普通边：从 nodeA 到 nodeB
workflow.addEdge("nodeA", "nodeB");

// 条件边：根据条件选择下一个节点
workflow.addConditionalEdges("nodeA", (state) => {
  if (state.shouldContinue) {
    return "nodeB";
  }
  return "nodeC";
});
```

### 状态（State）

状态是图中传递的数据，它在节点之间流转并被更新。状态定义了图的输入、输出和中间数据结构。

```typescript
import { Annotation } from "@langchain/langgraph";

// 定义状态类型
const StateAnnotation = Annotation.Root({
  messages: Annotation({
    reducer: (current, update) => current.concat(update),
    default: () => [],
  }),
  currentStep: Annotation<string>,
});
```

## 与 LangChain 的关系

LangGraph 是 LangChain 生态系统的一部分，它与 LangChain 紧密集成：

| 组件 | 定位 | 关系 |
|:---:|:---:|:---|
| LangChain | 基础框架 | 提供 LLM 抽象、Prompt 模板、Chain 等基础组件 |
| LangGraph | 编排框架 | 基于 LangChain 构建，提供图编排能力 |
| LangServe | 部署框架 | 用于将 LangChain/LangGraph 应用部署为 API |
| LangSmith | 监控平台 | 提供调试、监控和评估能力 |

```typescript
// LangGraph 与 LangChain 集成示例
import { ChatOpenAI } from "@langchain/openai";
import { StateGraph } from "@langchain/langgraph";

// 使用 LangChain 的模型
const model = new ChatOpenAI({ modelName: "gpt-4" });

// 在 LangGraph 节点中使用
const callModel = async (state) => {
  const response = await model.invoke(state.messages);
  return { messages: [response] };
};
```

## 适用场景

### ✅ 推荐使用 LangGraph 的场景

| 场景 | 说明 | 示例 |
|:---:|:---|:---|
| 多步骤 Agent | 需要多轮推理和工具调用的 Agent | 研究助手、代码生成器 |
| 多 Agent 协作 | 多个 Agent 协同完成复杂任务 | 辩论系统、团队协作 |
| 有状态工作流 | 需要维护会话状态的工作流 | 客服系统、对话机器人 |
| 人类介入 | 需要人类审核或决策的流程 | 内容审核、审批流程 |
| 复杂分支逻辑 | 包含条件判断和循环的流程 | 决策树、状态机 |

### ❌ 不推荐使用 LangGraph 的场景

| 场景 | 说明 | 替代方案 |
|:---:|:---|:---|
| 简单线性链 | 只需要顺序执行的简单任务 | LangChain Chain |
| 无状态请求 | 不需要维护状态的单次请求 | 直接调用 LLM API |
| 实时流处理 | 需要高吞吐量的实时数据处理 | 专用流处理框架 |

## LangGraph 的优势

### 1. 可控性

与传统的 ReAct Agent 相比，LangGraph 提供了更精细的控制能力：

```typescript
// 传统 ReAct Agent：黑盒执行
const agent = createReactAgent({ model, tools });

// LangGraph：白盒控制每个步骤
const workflow = new StateGraph(StateAnnotation)
  .addNode("analyze", analyzeStep)
  .addNode("decide", decideStep)
  .addNode("execute", executeStep)
  .addEdge("analyze", "decide")
  .addConditionalEdges("decide", routeDecision);
```

### 2. 可观测性

LangGraph 提供了完整的执行追踪能力：

```typescript
// 获取执行状态
const result = await graph.invoke(input);
console.log(result); // 包含完整的执行历史

// 流式获取中间状态
for await (const event of graph.stream(input)) {
  console.log(event); // 实时查看每个节点的执行结果
}
```

### 3. 持久化

支持状态的持久化和恢复：

```typescript
import { MemorySaver } from "@langchain/langgraph";

// 创建持久化存储
const checkpointer = new MemorySaver();

// 编译图时启用持久化
const graph = workflow.compile({ checkpointer });

// 执行时指定线程 ID
const result = await graph.invoke(input, { thread_id: "user-123" });
```
