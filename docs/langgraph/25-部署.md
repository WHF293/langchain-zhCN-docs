# 第25章 部署

> 来源: [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)

---

## 部署选项

LangGraph 提供了多种部署方式，可以根据需求选择合适的方案。

### 部署方式对比

| 部署方式 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| LangServe | 快速部署 API | 简单易用、自动文档 | 功能相对有限 |
| Docker | 容器化部署 | 环境一致性、易扩展 | 需要容器知识 |
| 云平台 | 生产环境 | 高可用、自动扩展 | 成本较高 |
| 本地服务器 | 开发调试 | 快速迭代 | 不适合生产 |

## LangServe

LangServe 是 LangChain 官方的部署工具，可以快速将 LangGraph 应用部署为 REST API。

### 安装依赖

```bash
# 安装 LangServe
pip install langserve[all]

# 或使用 npm
npm install @langchain/langserve
```

### 创建 LangServe 应用

```typescript
import { StateGraph, END } from "@langchain/langgraph";
import { Server } from "@langchain/langserve";

interface State {
  input: string;
  output: string;
}

// 创建工作流
const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const processNode = async (state: State) => {
  return {
    output: `处理结果: ${state.input}`,
  };
};

workflow.addNode("process", processNode);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();

// 创建 LangServe 服务器
const server = new Server({
  runnable: app,
  path: "/my-workflow",
  port: 8000,
});

// 启动服务器
server.listen(() => {
  console.log("LangServe 服务器已启动: http://localhost:8000");
  console.log("API 文档: http://localhost:8000/docs");
});
```

### API 端点

LangServe 自动创建以下端点：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/my-workflow/invoke` | POST | 调用一次执行 |
| `/my-workflow/batch` | POST | 批量执行 |
| `/my-workflow/stream` | POST | 流式执行 |
| `/my-workflow/playground` | GET | 交互式测试 UI |

## Docker

使用 Docker 可以确保应用在不同环境中的一致性。

### Dockerfile

```dockerfile
# 使用官方 Node.js 镜像
FROM node:18-slim

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production

# 复制源代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["node", "dist/server.js"]
```

### docker-compose.yml

```yaml
version: '3.8'

services:
  langgraph-app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - NODE_ENV=production
      - LANGCHAIN_API_KEY=${LANGCHAIN_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

### 构建和运行

```bash
# 构建镜像
docker build -t langgraph-app .

# 运行容器
docker run -p 8000:8000 \
  -e LANGCHAIN_API_KEY=your-key \
  -e OPENAI_API_KEY=your-key \
  langgraph-app

# 使用 docker-compose
docker-compose up -d
```

## 云平台部署

### AWS 部署

```typescript
// AWS Lambda 部署示例
import { StateGraph, END } from "@langchain/langgraph";

interface State {
  query: string;
  response: string;
}

const workflow = new StateGraph<State>({
  channels: {
    query: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    response: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const processNode = async (state: State) => {
  return {
    response: `处理: ${state.query}`,
  };
};

workflow.addNode("process", processNode);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();

// Lambda 处理函数
export const handler = async (event: any) => {
  const body = JSON.parse(event.body);
  const result = await app.invoke({
    query: body.query,
    response: "",
  });

  return {
    statusCode: 200,
    body: JSON.stringify(result),
  };
};
```

### Vercel 部署

```typescript
// api/workflow.ts
import { StateGraph, END } from "@langchain/langgraph";
import type { VercelRequest, VercelResponse } from "@vercel/node";

interface State {
  input: string;
  output: string;
}

const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const processNode = async (state: State) => {
  return {
    output: `处理结果: ${state.input}`,
  };
};

workflow.addNode("process", processNode);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();

export default async function handler(
  req: VercelRequest,
  res: VercelResponse
) {
  if (req.method === "POST") {
    const result = await app.invoke({
      input: req.body.input,
      output: "",
    });
    res.status(200).json(result);
  } else {
    res.status(405).json({ error: "Method not allowed" });
  }
}
```

## 环境变量

### 必需的环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `LANGCHAIN_API_KEY` | LangChain API 密钥 | `ls_xxx...` |
| `OPENAI_API_KEY` | OpenAI API 密钥 | `sk-xxx...` |
| `LANGCHAIN_TRACING_V2` | 启用追踪 | `true` |
| `LANGCHAIN_PROJECT` | 项目名称 | `my-project` |

### 环境变量配置

```typescript
// config.ts
export const config = {
  langchain: {
    apiKey: process.env.LANGCHAIN_API_KEY || "",
    tracing: process.env.LANGCHAIN_TRACING_V2 === "true",
    project: process.env.LANGCHAIN_PROJECT || "default",
    endpoint: process.env.LANGCHAIN_ENDPOINT || "https://api.smith.langchain.com",
  },
  openai: {
    apiKey: process.env.OPENAI_API_KEY || "",
  },
  server: {
    port: parseInt(process.env.PORT || "8000"),
    host: process.env.HOST || "0.0.0.0",
  },
};
```

### 使用 dotenv

```typescript
// 在开发环境中使用 dotenv
import dotenv from "dotenv";

dotenv.config();

import { StateGraph, END } from "@langchain/langgraph";

// 现在可以访问环境变量
console.log("API Key:", process.env.LANGCHAIN_API_KEY ? "已配置" : "未配置");
```

### .env 文件示例

```bash
# .env.example
LANGCHAIN_API_KEY=your_langsmith_api_key
OPENAI_API_KEY=your_openai_api_key
LANGCHAIN_TRACING_V2=true
LANGCHAIN_PROJECT=my-langgraph-project
PORT=8000
NODE_ENV=production
```

> **注意**：永远不要将 `.env` 文件提交到版本控制系统中。确保在 `.gitignore` 中添加 `.env`。

> **建议**：在生产环境中使用密钥管理服务（如 AWS Secrets Manager、Google Secret Manager）来管理敏感配置。

> **提示**：使用 `config.ts` 集中管理配置，可以更好地控制不同环境的配置差异。
