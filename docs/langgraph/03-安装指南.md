# 第3章 安装指南

> 来源: [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)

---

## 系统要求

在安装 LangGraph 之前，请确保你的开发环境满足以下要求：

| 要求 | 最低版本 | 推荐版本 |
|:---:|:---:|:---:|
| Node.js | 18.0.0 | 20.x LTS |
| TypeScript | 4.7.0 | 5.x |
| npm | 8.0.0 | 10.x |
| 操作系统 | Windows 10+, macOS 10.15+, Ubuntu 20.04+ | 最新稳定版 |

### 检查 Node.js 版本

```bash
node --version
# 输出示例：v20.11.0
```

### 检查 npm 版本

```bash
npm --version
# 输出示例：10.2.4
```

---

## 安装 LangGraph

### 使用 npm 安装（推荐）

```bash
npm install @langchain/langgraph @langchain/core
```

### 使用 yarn 安装

```bash
yarn add @langchain/langgraph @langchain/core
```

### 使用 pnpm 安装

```bash
pnpm add @langchain/langgraph @langchain/core
```

> **注意**：`@langchain/core` 是必需的对等依赖（peer dependency），必须同时安装。

## 包说明

### 核心包

| 包名 | 说明 | 必需 |
|:---:|:---|:---:|
| `@langchain/langgraph` | LangGraph 核心库 | ✅ |
| `@langchain/core` | LangChain 核心抽象 | ✅ |

### 可选依赖

根据你的使用场景，可能需要安装以下可选依赖：

| 包名 | 说明 | 使用场景 |
|:---:|:---|:---|
| `@langchain/openai` | OpenAI 集成 | 使用 OpenAI 模型 |
| `@langchain/anthropic` | Anthropic 集成 | 使用 Claude 模型 |
| `@langchain/community` | 社区集成 | 使用其他 LLM 提供商 |
| `zod` | 数据验证 | 定义严格的类型 |

### 安装可选依赖

```bash
# 如果使用 OpenAI
npm install @langchain/openai

# 如果使用 Anthropic
npm install @langchain/anthropic

# 如果需要数据验证
npm install zod
```

## 环境配置

### 配置 API 密钥

大多数 LLM 提供商需要 API 密钥。建议使用环境变量来管理密钥。

#### 方式 1：使用 .env 文件

创建 `.env` 文件：

```bash
# .env 文件
OPENAI_API_KEY=sk-your-openai-api-key
ANTHROPIC_API_KEY=sk-ant-your-anthropic-api-key
```

安装 dotenv 包：

```bash
npm install dotenv
```

在代码中加载环境变量：

```typescript
import "dotenv/config";
```

#### 方式 2：使用系统环境变量

```bash
# Linux/macOS
export OPENAI_API_KEY="sk-your-openai-api-key"

# Windows PowerShell
$env:OPENAI_API_KEY="sk-your-openai-api-key"

# Windows CMD
set OPENAI_API_KEY=sk-your-openai-api-key
```

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ES2020",
    "moduleResolution": "node",
```

## 验证安装

创建一个简单的测试文件来验证安装是否成功：

```typescript
// test-install.ts
import { Annotation, StateGraph, START, END } from "@langchain/langgraph";

// 定义简单的状态
const StateAnnotation = Annotation.Root({
  input: Annotation<string>,
  output: Annotation<string>,
});

// 定义节点
const processNode = async (state: typeof StateAnnotation.State) => {
  return { output: `处理完成: ${state.input}` };
};

// 构建图
const workflow = new StateGraph(StateAnnotation)
  .addNode("process", processNode)
  .addEdge(START, "process")
  .addEdge("process", END);

// 编译并运行
const app = workflow.compile();

async function test() {
  const result = await app.invoke({
    input: "测试输入",
    output: "",
  });

  console.log("✅ LangGraph 安装成功！");
  console.log("结果:", result);
}

test().catch(console.error);
```

运行测试：

```bash
# 使用 tsx 运行 TypeScript 文件
npx tsx test-install.ts

# 或者先编译再运行
npm run build
node dist/test-install.js
```

## 常见问题

### 1. 模块找不到错误

```
Error: Cannot find module '@langchain/langgraph'
```

**解决方案**：

```bash
# 清除缓存并重新安装
rm -rf node_modules package-lock.json
npm install
```

### 2. 类型错误

```
Type 'xxx' is not assignable to type 'yyy'
```

**解决方案**：确保 `@langchain/core` 版本与 `@langchain/langgraph` 兼容：

```bash
npm ls @langchain/core
```

### 3. ESM/CJS 兼容性问题

如果遇到模块系统问题，确保 `package.json` 中设置了 `"type": "module"`，或者将文件扩展名改为 `.mjs`。
