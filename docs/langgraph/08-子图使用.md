# 第八章 子图使用

> 来源: [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)

---

## 什么是子图

子图（Subgraph）是 LangGraph 中的一个重要概念，它允许你将复杂的图分解为更小、更易管理的模块。子图本质上是一个完整的图，可以被嵌入到另一个图中作为节点使用。

### 子图的优势

| 优势 | 说明 |
|------|------|
| **模块化** | 将复杂逻辑拆分为独立的子图，便于维护和复用 |
| **可复用性** | 同一个子图可以在多个父图中使用 |
| **独立测试** | 可以单独测试子图的功能 |
| **团队协作** | 不同团队可以并行开发不同的子图 |

## 创建子图

### 基本子图创建

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

// 定义子图的状态
const SubgraphState = Annotation.Root({
  messages: Annotation<string[]>({
    reducer: (prev, curr) => [...prev, ...curr],
    default: () => [],
  }),
  result: Annotation<string>,
});

// 创建子图
const subgraph = new StateGraph(SubgraphState)
  .addNode("process", async (state) => {
    const processed = state.messages.map((msg) => `已处理: ${msg}`);
    return {
      result: processed.join(", "),
    };
  })
  .addEdge("__start__", "process")
  .addEdge("process", "__end__")
  .compile();

// 定义父图的状态
const ParentState = Annotation.Root({
  input: Annotation<string>,
  output: Annotation<string>,
});

// 创建父图并嵌入子图
const parentGraph = new StateGraph(ParentState)
  .addNode("subgraph", subgraph)  // 将子图作为节点添加
  .addNode("final", async (state) => {
    return { output: `最终结果: ${state.input}` };
  })
  .addEdge("__start__", "subgraph")
  .addEdge("subgraph", "final")
  .addEdge("final", "__end__")
  .compile();
```

### 带状态映射的子图

当子图和父图的状态结构不同时，需要进行状态映射：

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

// 子图状态
const SubgraphState = Annotation.Root({
  text: Annotation<string>,
  processed: Annotation<boolean>({
    default: () => false,
  }),
});

// 子图定义
const subgraph = new StateGraph(SubgraphState)
  .addNode("analyze", async (state) => {
    return {
      text: `分析结果: ${state.text}`,
      processed: true,
    };
  })
  .addEdge("__start__", "analyze")
  .addEdge("analyze", "__end__")
  .compile();

// 父图状态
const ParentState = Annotation.Root({
  userInput: Annotation<string>,
  result: Annotation<string>,
});

// 父图定义，使用状态映射
const parentGraph = new StateGraph(ParentState)
  .addNode("subgraph", subgraph, {
    // 状态映射：将父图状态转换为子图状态
    input: (state) => ({
      text: state.userInput,
    }),
    // 结果映射：将子图结果转换为父图状态
    output: (state, result) => ({
      result: result.text,
    }),
  })
  .addEdge("__start__", "subgraph")
  .addEdge("subgraph", "__end__")
  .compile();
```

## 嵌套子图

子图可以嵌套多层，形成层次化的图结构：

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

// 第三层子图
const Level3State = Annotation.Root({
  data: Annotation<string>,
});

const level3Graph = new StateGraph(Level3State)
  .addNode("deepProcess", async (state) => {
    return { data: `[深层处理] ${state.data}` };
  })
  .addEdge("__start__", "deepProcess")
  .addEdge("deepProcess", "__end__")
  .compile();

// 第二层子图
const Level2State = Annotation.Root({
  input: Annotation<string>,
  output: Annotation<string>,
});

const level2Graph = new StateGraph(Level2State)
  .addNode("level3", level3Graph, {
    input: (state) => ({ data: state.input }),
    output: (state, result) => ({ output: result.data }),
  })
  .addEdge("__start__", "level3")
  .addEdge("level3", "__end__")
  .compile();

// 第一层（顶层）图
const TopLevelState = Annotation.Root({
  request: Annotation<string>,
  response: Annotation<string>,
});

const topLevelGraph = new StateGraph(TopLevelState)
  .addNode("level2", level2Graph, {
    input: (state) => ({ input: state.request }),
    output: (state, result) => ({ response: result.output }),
  })
  .addEdge("__start__", "level2")
  .addEdge("level2", "__end__")
  .compile();
```

## 子图通信

### 共享状态通信

子图和父图可以通过共享状态进行通信：

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

// 共享状态定义
const SharedState = Annotation.Root({
  messages: Annotation<string[]>({
    reducer: (prev, curr) => [...prev, ...curr],
    default: () => [],
  }),
  metadata: Annotation<Record<string, any>>({
    reducer: (prev, curr) => ({ ...prev, ...curr }),
    default: () => ({}),
  }),
});

// 子图 A：数据收集
const collectorSubgraph = new StateGraph(SharedState)
  .addNode("collect", async (state) => {
    return {
      messages: ["收集到数据 A", "收集到数据 B"],
      metadata: { source: "collector", timestamp: Date.now() },
    };
  })
  .addEdge("__start__", "collect")
  .addEdge("collect", "__end__")
  .compile();

// 子图 B：数据处理
const processorSubgraph = new StateGraph(SharedState)
  .addNode("process", async (state) => {
    return {
      messages: [`处理了 ${state.messages.length} 条消息`],
      metadata: { processed: true, count: state.messages.length },
    };
  })
  .addEdge("__start__", "process")
  .addEdge("process", "__end__")
  .compile();

// 父图：协调子图
const orchestratorGraph = new StateGraph(SharedState)
  .addNode("collector", collectorSubgraph)
  .addNode("processor", processorSubgraph)
  .addEdge("__start__", "collector")
  .addEdge("collector", "processor")
  .addEdge("processor", "__end__")
  .compile();
```

### 通过 Command 通信

使用 `Command` 对象在子图之间传递控制和数据：

```typescript
import { StateGraph, Annotation, Command } from "@langchain/langgraph";

const State = Annotation.Root({
  step: Annotation<string>,
  data: Annotation<any>,
});

// 子图 A：发送 Command
const senderSubgraph = new StateGraph(State)
  .addNode("send", async (state) => {
    return new Command({
      update: {
        step: "已发送",
        data: { message: "来自子图 A 的数据" },
      },
      goto: "receiver",  // 跳转到另一个子图
    });
  })
  .addEdge("__start__", "send")
  .compile();

// 子图 B：接收并处理
const receiverSubgraph = new StateGraph(State)
  .addNode("receive", async (state) => {
    return {
      step: "已接收",
      data: { ...state.data, processed: true },
    };
  })
  .addEdge("__start__", "receive")
  .addEdge("receive", "__end__")
  .compile();
```

## 错误处理

### 子图错误处理

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

const State = Annotation.Root({
  input: Annotation<string>,
  result: Annotation<string>,
  error: Annotation<string | null>({
    default: () => null,
  }),
});

// 带错误处理的子图
const safeSubgraph = new StateGraph(State)
  .addNode("process", async (state) => {
    try {
      // 模拟可能失败的操作
      if (state.input === "error") {
        throw new Error("处理失败");
      }
      return { result: `成功: ${state.input}`, error: null };
    } catch (e) {
      return {
        result: "",
        error: e instanceof Error ? e.message : "未知错误",
      };
    }
  })
  .addNode("errorHandler", async (state) => {
    if (state.error) {
      return { result: `错误已处理: ${state.error}`, error: null };
    }
    return {};
  })
  .addEdge("__start__", "process")
  .addConditionalEdge("process", async (state) => {
    return state.error ? "errorHandler" : "__end__";
  })
  .addEdge("errorHandler", "__end__")
  .compile();

// 父图使用带错误处理的子图
const parentGraph = new StateGraph(State)
  .addNode("safeProcess", safeSubgraph, {
    input: (state) => ({ input: state.input }),
    output: (state, result) => ({
      result: result.result,
      error: result.error,
    }),
  })
  .addEdge("__start__", "safeProcess")
  .addEdge("safeProcess", "__end__")
  .compile();
```

### 重试机制

```typescript
import { StateGraph, Annotation } from "@langchain/langgraph";

const State = Annotation.Root({
  attempts: Annotation<number>({
    default: () => 0,
    reducer: (_, curr) => curr,
  }),
  maxRetries: Annotation<number>({
    default: () => 3,
  }),
  result: Annotation<string>,
  error: Annotation<string | null>({
    default: () => null,
  }),
});

// 带重试的子图
const retryableSubgraph = new StateGraph(State)
  .addNode("process", async (state) => {
    try {
      // 模拟不稳定的操作
      if (state.attempts < 2) {
        throw new Error(`尝试 ${state.attempts + 1} 失败`);
      }
      return {
        result: `第 ${state.attempts + 1} 次尝试成功`,
        error: null,
      };
    } catch (e) {
      return {
        attempts: state.attempts + 1,
        error: e instanceof Error ? e.message : "未知错误",
      };
    }
  })
  .addNode("retry", async (state) => {
    return { error: null };  // 清除错误以便重试
  })
  .addEdge("__start__", "process")
  .addConditionalEdge("process", async (state) => {
    if (state.error && state.attempts < state.maxRetries) {
      return "retry";
    }
    return "__end__";
  })
  .addEdge("retry", "process")
  .compile();
```

> **注意**：子图的状态会自动与父图同步，确保状态的一致性。

> **建议**：对于复杂的业务逻辑，建议将功能拆分为多个子图，提高代码的可维护性和可测试性。

> **提示**：使用状态映射时，确保映射函数正确处理所有必要的状态字段，避免数据丢失。
