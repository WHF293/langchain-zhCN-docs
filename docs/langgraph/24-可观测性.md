# 第24章 可观测性

> 来源: [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)

---

## 概述

可观测性（Observability）是理解和调试复杂系统的关键能力。LangGraph 提供了完整的可观测性支持，包括追踪（Tracing）、日志（Logging）和指标（Metrics）三大支柱。

## 追踪（Tracing）

追踪可以记录每次执行的完整调用链，帮助理解系统行为和定位问题。

### 启用追踪

```typescript
import { StateGraph, END } from "@langchain/langgraph";
import { Client } from "langsmith";

// 配置 LangSmith 追踪
process.env.LANGCHAIN_TRACING_V2 = "true";
process.env.LANGCHAIN_API_KEY = "your-langsmith-api-key";
process.env.LANGCHAIN_PROJECT = "my-langgraph-project";

interface State {
  input: string;
  output: string;
}

// 创建工作流
const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

// 定义节点
const processNode = async (state: State) => {
  // 这个节点的执行会被自动追踪
  return {
    output: `处理: ${state.input}`,
  };
};

workflow.addNode("process", processNode);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();

// 执行并自动记录追踪
const result = await app.invoke({ input: "测试输入", output: "" });
console.log(result);
```

### 追踪数据结构

| 字段 | 说明 |
|------|------|
| run_id | 唯一运行标识符 |
| name | 调用名称 |
| inputs | 输入参数 |
| outputs | 输出结果 |
| start_time | 开始时间 |
| end_time | 结束时间 |
| parent_run_id | 父级运行 ID |
| child_runs | 子级运行列表 |

## 日志（Logging）

LangGraph 使用结构化日志记录系统行为。

### 配置日志级别

```typescript
import { StateGraph, END } from "@langchain/langgraph";

// 设置日志级别
const LOG_LEVELS = {
  DEBUG: 0,
  INFO: 1,
  WARNING: 2,
  ERROR: 3,
};

// 自定义日志器
class Logger {
  private level: number;

  constructor(level: number = LOG_LEVELS.INFO) {
    this.level = level;
  }

  debug(message: string, data?: any) {
    if (this.level <= LOG_LEVELS.DEBUG) {
      console.log(`[DEBUG] ${message}`, data || "");
    }
  }

  info(message: string, data?: any) {
    if (this.level <= LOG_LEVELS.INFO) {
      console.log(`[INFO] ${message}`, data || "");
    }
  }

  warning(message: string, data?: any) {
    if (this.level <= LOG_LEVELS.WARNING) {
      console.warn(`[WARNING] ${message}`, data || "");
    }
  }

  error(message: string, data?: any) {
    if (this.level <= LOG_LEVELS.ERROR) {
      console.error(`[ERROR] ${message}`, data || "");
    }
  }
}

const logger = new Logger(LOG_LEVELS.DEBUG);

interface State {
  data: string;
  result: string;
}

// 带日志的工作流
const workflow = new StateGraph<State>({
  channels: {
    data: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    result: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const loggedNode = async (state: State) => {
  logger.info("节点开始执行", { input: state.data });

  try {
    const result = `处理完成: ${state.data}`;
    logger.debug("节点执行成功", { output: result });
    return { result };
  } catch (error) {
    logger.error("节点执行失败", { error });
    throw error;
  }
};

workflow.addNode("loggedNode", loggedNode);
workflow.addEdge("loggedNode", END);
workflow.setEntryPoint("loggedNode");

const app = workflow.compile();
```

## 指标（Metrics）

指标用于监控系统的性能和健康状态。

### 常用指标

| 指标名称 | 类型 | 说明 |
|----------|------|------|
| latency | Histogram | 请求延迟 |
| request_count | Counter | 请求总数 |
| error_count | Counter | 错误总数 |
| active_runs | Gauge | 当前活跃运行数 |

### 收集指标

```typescript
import { StateGraph, END } from "@langchain/langgraph";

// 指标收集器
class MetricsCollector {
  private metrics: Map<string, number> = new Map();

  increment(name: string, value: number = 1) {
    const current = this.metrics.get(name) || 0;
    this.metrics.set(name, current + value);
  }

  gauge(name: string, value: number) {
    this.metrics.set(name, value);
  }

  getMetrics() {
    return Object.fromEntries(this.metrics);
  }
}

const metrics = new MetricsCollector();

interface State {
  input: string;
  output: string;
}

// 带指标监控的工作流
const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const monitoredNode = async (state: State) => {
  const startTime = Date.now();
  metrics.increment("request_count");

  try {
    const result = `处理: ${state.input}`;
    metrics.increment("success_count");
    return { output: result };
  } catch (error) {
    metrics.increment("error_count");
    throw error;
  } finally {
    const latency = Date.now() - startTime;
    metrics.increment("total_latency", latency);
  }
};

workflow.addNode("monitored", monitoredNode);
workflow.addEdge("monitored", END);
workflow.setEntryPoint("monitored");

const app = workflow.compile();

// 运行并收集指标
async function runWithMetrics(input: string) {
  const result = await app.invoke({ input, output: "" });
  console.log("结果:", result);
  console.log("指标:", metrics.getMetrics());
  return result;
}
```

## LangSmith 集成

LangSmith 是 LangChain 官方的可观测性平台，与 LangGraph 深度集成。

### 配置 LangSmith

```typescript
// 环境变量配置
// .env 文件
// LANGCHAIN_TRACING_V2=true
// LANGCHAIN_API_KEY=your-api-key
// LANGCHAIN_PROJECT=your-project-name
// LANGCHAIN_ENDPOINT=https://api.smith.langchain.com

import { StateGraph, END } from "@langchain/langgraph";

// 或者在代码中配置
process.env.LANGCHAIN_TRACING_V2 = "true";
process.env.LANGCHAIN_API_KEY = "your-api-key";
process.env.LANGCHAIN_PROJECT = "my-project";

interface State {
  query: string;
  response: string;
}

const workflow = new StateGraph<State>({
  channels: {
    query: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    response: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const processQuery = async (state: State) => {
  // LangSmith 会自动记录这次执行
  return {
    response: `回答: ${state.query}`,
  };
};

workflow.addNode("process", processQuery);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();

// 所有调用都会被记录到 LangSmith
const result = await app.invoke({
  query: "什么是 LangGraph?",
  response: "",
});
```

## 调试技巧

### 1. 使用详细日志

```typescript
import { StateGraph, END } from "@langchain/langgraph";

interface State {
  data: string;
  debugInfo: string[];
}

// 启用详细调试日志
process.env.LANGCHAIN_VERBOSE = "true";

const workflow = new StateGraph<State>({
  channels: {
    data: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    debugInfo: {
      value: (prev: string[], next: string[]) => [...prev, ...next],
      default: () => [],
    },
  },
});

const debugNode = async (state: State) => {
  const timestamp = new Date().toISOString();
  return {
    data: state.data.toUpperCase(),
    debugInfo: [`[${timestamp}] 处理数据: ${state.data}`],
  };
};

workflow.addNode("debugNode", debugNode);
workflow.addEdge("debugNode", END);
workflow.setEntryPoint("debugNode");

const app = workflow.compile();
```

### 2. 检查点调试

```typescript
import { StateGraph, END } from "@langchain/langgraph";
import { MemorySaver } from "@langchain/langgraph/checkpoint";

interface State {
  step: number;
  data: string;
}

// 使用检查点保存中间状态
const checkpointer = new MemorySaver();

const workflow = new StateGraph<State>({
  channels: {
    step: {
      value: (prev: number, next: number) => next,
      default: () => 0,
    },
    data: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

const step1 = async (state: State) => {
  console.log("步骤 1 - 输入:", state);
  return { step: 1, data: `步骤1处理: ${state.data}` };
};

const step2 = async (state: State) => {
  console.log("步骤 2 - 输入:", state);
  return { step: 2, data: `步骤2处理: ${state.data}` };
};

workflow.addNode("step1", step1);
workflow.addNode("step2", step2);
workflow.addEdge("step1", "step2");
workflow.addEdge("step2", END);
workflow.setEntryPoint("step1");

// 编译时启用检查点
const app = workflow.compile({ checkpointer });

// 执行并保存检查点
const config = { configurable: { thread_id: "debug-session-1" } };
const result = await app.invoke(
  { step: 0, data: "测试数据" },
  config
);

console.log("最终结果:", result);
```

### 3. 错误处理与调试

```typescript
import { StateGraph, END } from "@langchain/langgraph";

interface State {
  input: string;
  output: string;
  error?: string;
}

const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    error: {
      value: (prev: string | undefined, next: string | undefined) => next,
      default: () => undefined,
    },
  },
});

const riskyNode = async (state: State) => {
  try {
    // 可能出错的操作
    const result = JSON.parse(state.input);
    return { output: JSON.stringify(result), error: undefined };
  } catch (err) {
    console.error("解析错误:", err);
    return {
      output: "",
      error: `解析失败: ${(err as Error).message}`,
    };
  }
};

const errorHandler = (state: State): string => {
  if (state.error) {
    console.log("检测到错误，进入错误处理流程");
    return "handleError";
  }
  return "success";
};

const handleSuccess = (state: State) => {
  console.log("成功:", state.output);
  return { output: state.output };
};

const handleError = (state: State) => {
  console.log("处理错误:", state.error);
  return { output: `错误已处理: ${state.error}` };
};

workflow.addNode("risky", riskyNode);
workflow.addNode("success", handleSuccess);
workflow.addNode("handleError", handleError);

workflow.addConditionalEdges("risky", errorHandler, {
  success: "success",
  handleError: "handleError",
});

workflow.addEdge("success", END);
workflow.addEdge("handleError", END);
workflow.setEntryPoint("risky");

const app = workflow.compile();
```

> **提示**：在开发环境中启用详细日志，在生产环境中使用适当的日志级别以避免性能影响。

> **建议**：定期检查 LangSmith 中的追踪数据，可以帮助发现潜在的性能问题和错误模式。

> **注意**：确保在生产环境中正确配置 API 密钥和项目名称，避免泄露敏感信息。
