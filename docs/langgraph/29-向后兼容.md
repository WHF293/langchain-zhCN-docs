## 版本策略

LangGraph 遵循语义化版本控制（Semantic Versioning），确保版本号清晰地传达变更的性质。

### 版本号格式

```
MAJOR.MINOR.PATCH
```

| 版本类型 | 说明 | 示例 |
|----------|------|------|
| MAJOR | 不兼容的 API 变更 | 1.0.0 -> 2.0.0 |
| MINOR | 向后兼容的功能新增 | 1.0.0 -> 1.1.0 |
| PATCH | 向后兼容的问题修复 | 1.0.0 -> 1.0.1 |

### 版本发布周期

```typescript
// 版本发布计划示例
interface ReleaseSchedule {
  major: {
    frequency: "每 6-12 个月";
    notice: "提前 3 个月通知";
    migrationGuide: "提供详细迁移指南";
  };
  minor: {
    frequency: "每月";
    notice: "提前 2 周通知";
    migrationGuide: "提供变更说明";
  };
  patch: {
    frequency: "按需";
    notice: "无特殊通知";
    migrationGuide: "仅修复说明";
  };
}
```

## 弃用警告

当某个功能即将被移除时，LangGraph 会先标记为弃用（Deprecated），并在后续版本中移除。

### 弃用流程

```
功能标记为弃用 ──► 添加弃用警告 ──► 保持 2-3 个版本 ──► 移除功能
      │                   │                   │               │
      ▼                   ▼                   ▼               ▼
   通知用户          控制台警告         文档更新        清理代码
```

### 弃用警告示例

```typescript
import { StateGraph, END } from "@langchain/langgraph";

// 旧版本 API（已弃用）
class OldWorkflowBuilder {
  /**
   * @deprecated 使用 new StateGraph() 替代
   * 将在未来版本中移除
   */
  static create(config: any) {
    console.warn(
      "警告: OldWorkflowBuilder.create() 已弃用，请使用 new StateGraph() 替代"
    );
    // 返回旧的实现
    return {};
  }
}

// 新版本 API
interface State {
  input: string;
  output: string;
}

const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

// 使用新 API
const processNode = async (state: State) => {
  return { output: `处理: ${state.input}` };
};

workflow.addNode("process", processNode);
workflow.addEdge("process", END);
workflow.setEntryPoint("process");

const app = workflow.compile();
```

### 检测弃用警告

```typescript
// 配置弃用警告处理
process.on("warning", (warning) => {
  if (warning.name === "DeprecationWarning") {
    console.warn("弃用警告:", warning.message);
    console.warn("建议:", warning.message);
    // 可以发送到监控系统
  }
});

// 或者使用环境变量控制
// NODE_OPTIONS="--throw-deprecation" 可以将弃用警告转为错误
// NODE_OPTIONS="--no-deprecation" 可以禁用弃用警告
```

### 从 v0.x 迁移到 v1.x

```typescript
// v0.x 旧写法
import { Graph } from "@langchain/langgraph";

const graph = new Graph();
graph.addNode("start", (state) => state);
graph.addNode("end", (state) => state);
graph.addEdge("start", "end");
graph.setEntryPoint("start");
graph.setFinishPoint("end");

const app = graph.compile();
```

```typescript
// v1.x 新写法
import { StateGraph, END } from "@langchain/langgraph";

interface State {
  input: string;
  output: string;
}

const workflow = new StateGraph<State>({
  channels: {
    input: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
    output: {
      value: (prev: string, next: string) => next,
      default: () => "",
    },
  },
});

workflow.addNode("start", async (state) => state);
workflow.addNode("end", async (state) => state);
workflow.addEdge("start", "end");
workflow.addEdge("end", END);
workflow.setEntryPoint("start");

const app = workflow.compile();
```

### 迁移工具

```typescript
// 迁移脚本示例
import * as fs from "fs";
import * as path from "path";

class MigrationTool {
  // 分析代码中的弃用用法
  static analyzeDeprecations(filePath: string): string[] {
    const content = fs.readFileSync(filePath, "utf-8");
    const warnings: string[] = [];

    // 检查旧的 API 用法
    if (content.includes("new Graph()")) {
      warnings.push("检测到旧的 Graph 类用法，请迁移到 StateGraph");
    }

    if (content.includes("setFinishPoint")) {
      warnings.push(
        "检测到 setFinishPoint 用法，请使用 END 常量替代"
      );
    }

    return warnings;
  }

  // 生成迁移报告
  static generateReport(directory: string) {
    const files = fs.readdirSync(directory);
    const report: Record<string, string[]> = {};

    files.forEach((file) => {
      if (file.endsWith(".ts")) {
        const filePath = path.join(directory, file);
        const warnings = this.analyzeDeprecations(filePath);
        if (warnings.length > 0) {
          report[file] = warnings;
        }
      }
    });

    return report;
  }
}

// 使用迁移工具
const report = MigrationTool.generateReport("./src");
console.log("迁移报告:", report);
```

## 破坏性变更

### 常见破坏性变更类型

| 变更类型 | 说明 | 示例 |
|----------|------|------|
| API 移除 | 移除已弃用的 API | `Graph` 类移除 |
| 参数变更 | 函数参数签名变化 | 构造函数参数调整 |
| 返回值变更 | 返回类型或结构变化 | 状态结构变更 |
| 行为变更 | 默认行为改变 | 错误处理方式变化 |

### 处理破坏性变更

```typescript
// 1. 检查当前版本
import { version } from "@langchain/langgraph/package.json";

console.log("当前 LangGraph 版本:", version);

// 2. 版本兼容性检查
function checkCompatibility(requiredVersion: string) {
  const current = parseVersion(version);
  const required = parseVersion(requiredVersion);

  if (current.major !== required.major) {
    console.warn(
      `警告: 当前版本 ${version} 与所需版本 ${requiredVersion} 不兼容`
    );
    return false;
  }

  return true;
}

function parseVersion(v: string) {
  const [major, minor, patch] = v.split(".").map(Number);
  return { major, minor, patch };
}

// 3. 条件性使用新旧 API
function createWorkflow() {
  const [major] = version.split(".").map(Number);

  if (major >= 1) {
    // 使用新 API
    return new StateGraph({
      channels: {
        input: {
          value: (prev: string, next: string) => next,
          default: () => "",
        },
      },
    });
  } else {
    // 使用旧 API
    return new Graph();
  }
}
```

### 版本锁定策略

```typescript
// 版本锁定建议
interface VersionLockStrategy {
  // 生产环境：精确锁定版本
  production: {
    "@langchain/langgraph": "1.2.3";
  };

  // 开发环境：允许补丁更新
  development: {
    "@langchain/langgraph": "~1.2.3"; // 允许 1.2.x 更新
  };

  // 实验环境：允许次版本更新
  experimental: {
    "@langchain/langgraph": "^1.2.3"; // 允许 1.x.x 更新
  };
}
```

> **提示**：在升级版本前，务必阅读变更日志和迁移指南，避免意外的破坏性变更。

> **建议**：使用版本锁定策略，确保团队成员使用相同的依赖版本。

> **注意**：生产环境应使用精确版本锁定，避免自动升级导致的问题。
