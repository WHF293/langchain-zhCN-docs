# 第4章 设计哲学

> 来源: [LangChain 官方文档](https://python.langchain.com/)

---

LangChain 的设计遵循四个核心原则，这些原则指导着框架的架构决策和 API 设计。

## 模块化（Modularity）

LangChain 采用高度模块化的设计，每个组件都是独立的、可替换的构建块。

### 设计原则

- **单一职责**：每个组件只做一件事
- **松耦合**：组件之间通过标准接口交互
- **可替换**：可以轻松替换任何组件

### 代码示例

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ChatAnthropic } from "@langchain/anthropic";
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { StringOutputParser } from "@langchain/core/output_parsers";

// 提示模板是独立的
const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是一个有帮助的助手"],
  ["user", "{input}"],
]);

// 模型是可替换的
const openaiModel = new ChatOpenAI({ modelName: "gpt-4" });
const anthropicModel = new ChatAnthropic({ modelName: "claude-3-opus" });

// 解析器是独立的
const parser = new StringOutputParser();

// 可以自由组合
const openaiChain = prompt.pipe(openaiModel).pipe(parser);
const anthropicChain = prompt.pipe(anthropicModel).pipe(parser);

// 两者接口完全相同
const result1 = await openaiChain.invoke({ input: "你好" });
const result2 = await anthropicChain.invoke({ input: "你好" });
```

### 模块化的优势

| 优势 | 说明 |
|:---|:---|
| **易于测试** | 可以单独测试每个组件 |
| **易于替换** | 更换模型提供商无需重写代码 |
| **易于扩展** | 添加新功能只需添加新组件 |
| **代码复用** | 组件可以在不同项目中复用 |

## 可组合性（Composability）

组件可以通过 `.pipe()` 方法自由组合，形成复杂的处理链。

### 组合模式

```typescript
import { RunnableSequence, RunnablePassthrough } from "@langchain/core/runnables";

// 简单组合
const chain1 = prompt.pipe(model).pipe(parser);

// 并行组合
const chain2 = RunnableSequence.from([
  {
    original: new RunnablePassthrough(),
    summary: prompt.pipe(model).pipe(parser),
  },
  // 后续处理...
]);

// 分支组合
const chain3 = RunnableSequence.from([
  // 条件分支逻辑...
]);
```

### 管道操作符

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { StringOutputParser } from "@langchain/core/output_parsers";

// 使用 .pipe() 创建管道
const chain = ChatPromptTemplate.fromMessages([
  ["system", "你是一个{role}"],
  ["user", "{input}"],
])
  .pipe(new ChatOpenAI({ modelName: "gpt-4" }))
  .pipe(new StringOutputParser());

// 等价于
const chain2 = new StringOutputParser().pipe(
  new ChatOpenAI({ modelName: "gpt-4" }).pipe(
    ChatPromptTemplate.fromMessages([
      ["system", "你是一个{role}"],
      ["user", "{input}"],
    ])
  )
);
```

### Runnable 接口

所有组件都实现统一的 `Runnable` 接口：

| 方法 | 说明 |
|:---|:---|
| `invoke()` | 单次调用 |
| `batch()` | 批量调用 |
| `stream()` | 流式调用 |
| `streamEvents()` | 流式事件 |
| `pipe()` | 组合组件 |
| `pick()` | 选择输出字段 |
| `map()` | 映射输出 |
| `withConfig()` | 添加配置 |
| `withRetry()` | 添加重试 |
| `bind()` | 绑定参数 |

## 流式优先（Streaming First）

LangChain 将流式处理作为一等公民，所有组件都支持流式输出。

### 基础流式

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ChatPromptTemplate } from "@langchain/core/prompts";

const model = new ChatOpenAI({ modelName: "gpt-4", streaming: true });
const prompt = ChatPromptTemplate.fromMessages([
  ["user", "写一个关于{topic}的故事"],
]);

const chain = prompt.pipe(model);

// 使用 stream 方法
const stream = await chain.stream({ topic: "太空" });

for await (const chunk of stream) {
  process.stdout.write(chunk.content as string);
}
```

### 流式事件

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { StringOutputParser } from "@langchain/core/output_parsers";

const model = new ChatOpenAI({ modelName: "gpt-4" });
const prompt = ChatPromptTemplate.fromMessages([
  ["user", "{input}"],
]);

const chain = prompt.pipe(model).pipe(new StringOutputParser());

// 使用 streamEvents 获取详细事件
const eventStream = await chain.streamEvents(
  { input: "你好" },
  { version: "v2" }
);

for await (const event of eventStream) {
  if (event.event === "on_chat_model_stream") {
    process.stdout.write(event.data.chunk.content as string);
  } else if (event.event === "on_chain_start") {
    console.log("\n[链开始]");
  } else if (event.event === "on_chain_end") {
    console.log("\n[链结束]");
  }
}
```

### 流式的优势

- **更好的用户体验**：用户可以立即看到输出
- **降低延迟**：无需等待完整响应
- **实时反馈**：可以监控处理进度
- **节省内存**：无需存储完整响应

## 开发者体验（Developer Experience）

LangChain 注重提供优秀的开发者体验，包括类型安全、清晰的错误信息和丰富的工具支持。

### 类型安全

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { StringOutputParser } from "@langchain/core/output_parsers";

// TypeScript 提供完整的类型推断
const model = new ChatOpenAI({ modelName: "gpt-4" });
const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是一个{role}"],  // role 会被类型检查
  ["user", "{input}"],           // input 会被类型检查
]);

const chain = prompt.pipe(model).pipe(new StringOutputParser());

// 调用时会检查参数类型
const result = await chain.invoke({
  role: "助手",   // ✅ 类型正确
  input: "你好",  // ✅ 类型正确
  // extra: "xxx", // ❌ 类型错误：多余的参数
});
```

### 清晰的错误信息

```typescript
try {
  const model = new ChatOpenAI({ modelName: "gpt-4" });

} catch (error) {

  console.error(error.message);
```

## 设计原则总结

| 原则 | 核心思想 | 好处 |
|:---|:---|:---|
| **模块化** | 组件独立、可替换 | 易于测试、维护、扩展 |
| **可组合性** | 通过管道自由组合 | 灵活构建复杂应用 |
| **流式优先** | 原生流式支持 | 更好的用户体验 |
| **开发者体验** | 类型安全、清晰错误 | 提高开发效率 |

## 最佳实践

### 遵循模块化

```typescript
// ✅ 好的做法：组件独立
const prompt = ChatPromptTemplate.fromMessages([...]);
const model = new ChatOpenAI({...});
const parser = new StringOutputParser();
const chain = prompt.pipe(model).pipe(parser);

// ❌ 不好的做法：紧耦合
class MyChain {
  private model = new ChatOpenAI({...});
  private prompt = ChatPromptTemplate.fromMessages([...]);
  // 难以替换和测试
}
```

### 利用可组合性

```typescript
// ✅ 好的做法：复用组件
const basePrompt = ChatPromptTemplate.fromMessages([
  ["system", "你是一个{role}"],
  ["user", "{input}"],
]);

const translatorChain = basePrompt
  .pipe(new ChatOpenAI({ modelName: "gpt-4" }))
  .pipe(new StringOutputParser());

const summarizerChain = basePrompt
  .pipe(new ChatAnthropic({ modelName: "claude-3" }))
  .pipe(new StringOutputParser());
```

### 优先使用流式

```typescript
// ✅ 好的做法：流式输出
const stream = await chain.stream(input);
for await (const chunk of stream) {
  // 实时处理
}

// ⚠️ 注意：仅在需要完整结果时使用 invoke
const result = await chain.invoke(input);
```
