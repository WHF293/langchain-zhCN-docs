## 内置中间件列表

LangChain 提供了多种内置中间件，覆盖常见的使用场景。以下是主要的内置中间件：

### 核心中间件

| 中间件名称 | 功能 | 优先级 |
|------------|------|--------|
| AuthenticationMiddleware | 身份认证 | 10 |
| RateLimitMiddleware | 请求限流 | 50 |
| CachingMiddleware | 响应缓存 | 100 |
| RetryMiddleware | 失败重试 | 150 |
| LoggingMiddleware | 日志记录 | 200 |
| ValidationMiddleware | 数据验证 | 250 |
| TransformMiddleware | 数据转换 | 300 |

### 辅助中间件

| 中间件名称 | 功能 | 用途 |
|------------|------|------|
| CorsMiddleware | 跨域处理 | Web 应用 |
| CompressionMiddleware | 响应压缩 | 性能优化 |
| TimeoutMiddleware | 超时控制 | 防止阻塞 |
| MetricsMiddleware | 指标收集 | 监控告警 |
| TracingMiddleware | 链路追踪 | 分布式追踪 |

// 导入内置中间件
import {
  AuthenticationMiddleware,

  CachingMiddleware,

  LoggingMiddleware,
} from "langchain/middleware";

## 内置中间件配置

### AuthenticationMiddleware

身份认证中间件，用于验证请求的合法性。

```typescript
// 认证中间件配置
import { AuthenticationMiddleware } from "langchain/middleware";

const authMiddleware = new AuthenticationMiddleware({
  // 认证方式
  strategy: "jwt",  // jwt | api-key | oauth2

  // JWT 配置
  jwt: {
    secret: process.env.JWT_SECRET,
    algorithms: ["HS256"],
    issuer: "your-app",
    audience: "your-api",
  },

  // API Key 配置
  apiKey: {
    headerName: "X-API-Key",
    validate: async (key: string) => {
      // 自定义验证逻辑
      return await validateApiKey(key);
    },
  },

  // 排除路径（不需要认证）
  excludePaths: [
    "/health",
    "/metrics",
    "/public/*",
  ],

  // 错误处理
  onError: (error) => {
    return {
      status: 401,
      message: "认证失败: " + error.message,
    };
  },
});
```

### RateLimitMiddleware

请求限流中间件，防止 API 被滥用。

```typescript
// 限流中间件配置
import { RateLimitMiddleware } from "langchain/middleware";

const rateLimitMiddleware = new RateLimitMiddleware({
  // 时间窗口（毫秒）
  windowMs: 60 * 1000,  // 1分钟

  // 最大请求数
  max: 100,

  // 限流策略
  strategy: "sliding-window",  // fixed-window | sliding-window | token-bucket

  // 按什么维度限流
  keyGenerator: (request) => {
    // 按用户 ID 限流
    return request.headers.get("X-User-ID") ||
           request.headers.get("X-Forwarded-For") ||
           "anonymous";
  },

  // 自定义限流消息
  message: "请求过于频繁，请稍后再试",

  // 响应头
  headers: true,  // 是否返回限流相关头信息

  // 存储后端
  store: "memory",  // memory | redis

  // Redis 配置（如果使用 Redis 存储）
  redis: {
    host: process.env.REDIS_HOST,
    port: parseInt(process.env.REDIS_PORT || "6379"),
  },

  // 跳过成功的请求
  skipSuccessfulRequests: false,

  // 跳过失败的请求
  skipFailedRequests: false,
});
```

### CachingMiddleware

响应缓存中间件，提高响应速度。

```typescript
// 缓存中间件配置
import { CachingMiddleware } from "langchain/middleware";

const cachingMiddleware = new CachingMiddleware({
  // 缓存策略
  strategy: "response",  // response | semantic

  // 缓存时间（秒）
  ttl: 300,  // 5分钟

  // 缓存键生成
  keyGenerator: (request) => {
    // 基于请求 URL 和参数生成缓存键
    const url = new URL(request.url);
    return `${url.pathname}:${url.search}`;
  },

  // 缓存条件
  condition: (request, response) => {
    // 只缓存成功的 GET 请求
    return (
      request.method === "GET" &&
      response.status === 200
    );
  },

  // 缓存存储
  store: "memory",  // memory | redis | memcached

  // 缓存大小限制
  maxSize: 1000,  // 最多缓存 1000 条

  // 语义缓存配置（用于 LLM 响应）
  semanticCache: {
    enabled: true,
    similarityThreshold: 0.95,  // 相似度阈值
    embeddingModel: "text-embedding-3-small",
  },

  // 缓存穿透保护
  staleWhileRevalidate: true,
});
```

> **提示**：对于 LLM 应用，语义缓存（Semantic Cache）可以显著降低成本，相似的问题可以复用缓存的答案。

### RetryMiddleware

失败重试中间件，提高系统可靠性。

```typescript
// 重试中间件配置
import { RetryMiddleware } from "langchain/middleware";

const retryMiddleware = new RetryMiddleware({
  // 最大重试次数
  maxRetries: 3,

  // 重试延迟策略
  backoff: "exponential",  // fixed | exponential | random

  // 基础延迟（毫秒）
  baseDelay: 1000,

  // 最大延迟（毫秒）
  maxDelay: 30000,

  // 可重试的错误
  retryableErrors: [
    "ECONNRESET",
    "ETIMEDOUT",
    "RATE_LIMIT_EXCEEDED",
    "SERVER_ERROR",
  ],

  // 可重试的状态码
  retryableStatusCodes: [429, 500, 502, 503, 504],

  // 自定义重试条件
  shouldRetry: (error, attempt) => {
    // 只重试网络错误，不重试业务错误
    return error.code === "NETWORK_ERROR" && attempt < 3;
  },

  // 重试回调
  onRetry: (error, attempt) => {
    console.log(`第 ${attempt} 次重试，原因: ${error.message}`);
  },
});
```

### LoggingMiddleware

日志记录中间件，记录请求和响应信息。

```typescript
// 日志中间件配置
import { LoggingMiddleware } from "langchain/middleware";

const loggingMiddleware = new LoggingMiddleware({
  // 日志级别
  level: "info",  // debug | info | warn | error

  // 日志格式
  format: "json",  // json | text | custom

  // 记录请求
  request: {
    enabled: true,
    // 记录哪些信息
    include: [
      "method",
      "url",
      "headers",
      "body",
      "query",
    ],
    // 排除敏感头信息
    excludeHeaders: ["Authorization", "Cookie"],
    // 限制 body 大小
    maxBodySize: 10240,  // 10KB
  },

  // 记录响应
  response: {
    enabled: true,
    include: [
      "status",
      "headers",
      "body",
    ],
    maxBodySize: 10240,
  },

  // 记录性能
  performance: {
    enabled: true,
    // 慢请求阈值（毫秒）
    slowThreshold: 1000,
  },

  // 自定义日志字段
  customFields: (request, response) => {
    return {
      userId: request.headers.get("X-User-ID"),
      requestId: request.headers.get("X-Request-ID"),
      environment: process.env.NODE_ENV,
    };
  },

  // 日志输出
  transport: "console",  // console | file | remote
});
```

## 内置中间件示例

### 完整配置示例

```typescript
// 完整的中间件配置示例
import {
  AuthenticationMiddleware,
  RateLimitMiddleware,
  CachingMiddleware,
  RetryMiddleware,
  LoggingMiddleware,
} from "langchain/middleware";

// 创建中间件实例
const auth = new AuthenticationMiddleware({
  strategy: "jwt",
  jwt: { secret: process.env.JWT_SECRET },
  excludePaths: ["/health"],
});

const rateLimit = new RateLimitMiddleware({
  windowMs: 60000,
  max: 100,
  keyGenerator: (req) => req.headers.get("X-API-Key") || "anonymous",
});

const cache = new CachingMiddleware({
  ttl: 300,
  semanticCache: {
    enabled: true,
    similarityThreshold: 0.95,
  },
});

const retry = new RetryMiddleware({
  maxRetries: 3,
  backoff: "exponential",
});

const logging = new LoggingMiddleware({
  level: "info",
  format: "json",
});

// 注册中间件
const middlewareManager = new MiddlewareManager();
middlewareManager.use(auth);
middlewareManager.use(rateLimit);
middlewareManager.use(cache);
middlewareManager.use(retry);
middlewareManager.use(logging);
```

### 组合使用示例

```typescript
// 中间件组合使用
class MiddlewareComposer {
  private middlewares: Middleware[] = [];

  // 添加中间件
  add(middleware: Middleware): this {
    this.middlewares.push(middleware);
    return this;  // 支持链式调用
  }

  // 批量添加
  addAll(middlewares: Middleware[]): this {
    this.middlewares.push(...middlewares);
    return this;
  }

  // 构建中间件链
  build(): Middleware[] {
    // 按优先级排序
    return this.middlewares.sort((a, b) => a.priority - b.priority);
  }
}

// 使用示例
const composer = new MiddlewareComposer();
const middlewares = composer
  .add(auth)
  .add(rateLimit)
  .add(cache)
  .add(retry)
  .add(logging)
  .build();
```

## 内置中间件最佳实践

### 选择合适的中间件

| 场景 | 推荐中间件 | 说明 |
|------|------------|------|
| API 安全 | AuthenticationMiddleware | 必须配置 |
| 防止滥用 | RateLimitMiddleware | 生产环境必须 |
| 性能优化 | CachingMiddleware | 适合读多写少 |
| 可靠性 | RetryMiddleware | 网络不稳定时使用 |
| 问题排查 | LoggingMiddleware | 开发和生产都建议 |

### 性能优化

```typescript
// 中间件性能优化
class OptimizedMiddlewareManager {
  private middlewares: Middleware[] = [];
  private cache = new Map<string, any>();

  // 批量处理优化
  async executeBatch(requests: Request[]): Promise<Response[]> {
    // 按中间件分组处理，减少上下文切换
    const results: Response[] = [];

    for (const middleware of this.middlewares) {
      // 批量执行 beforeRequest
      if (middleware.onBeforeRequest) {
        await Promise.all(
          requests.map((req) => middleware.onBeforeRequest!(req))
        );
      }
    }

    return results;
  }

  // 条件执行优化
  async executeConditional(
    request: Request,
    handler: (req: Request) => Promise<Response>
  ): Promise<Response> {
    // 只执行必要的中间件
    const requiredMiddlewares = this.middlewares.filter(
      (m) => m.shouldExecute?.(request) !== false
    );

    // 执行中间件链
    let currentRequest = request;
    for (const middleware of requiredMiddlewares) {
      if (middleware.onBeforeRequest) {
        currentRequest = await middleware.onBeforeRequest(currentRequest);
      }
    }

    return handler(currentRequest);
  }
}
```

### 监控和告警

```typescript
// 中间件监控
class MonitoredMiddleware implements Middleware {
  name: string;
  priority: number;
  private metrics = {
    totalRequests: 0,
    successRequests: 0,
    failedRequests: 0,
    totalDuration: 0,
  };

  async onBeforeRequest(request: Request): Promise<Request> {
    this.metrics.totalRequests++;
    (request as any).startTime = Date.now();
    return request;
  }

  async onAfterResponse(response: Response): Promise<Response> {
    const startTime = (response as any).startTime;
    const duration = Date.now() - startTime;

    this.metrics.totalDuration += duration;
    this.metrics.successRequests++;

    // 检查是否需要告警
    if (duration > 1000) {
      this.triggerAlert("slow_request", { duration });
    }

    return response;
  }

  async onError(error: Error): Promise<void> {
    this.metrics.failedRequests++;
    this.triggerAlert("error", { message: error.message });
  }

  private triggerAlert(type: string, data: any): void {
    // 发送告警通知
    console.warn(`[告警] ${type}:`, data);
  }

  // 获取指标
  getMetrics() {
    return {
      ...this.metrics,
      avgDuration: this.metrics.totalDuration / this.metrics.totalRequests,
      successRate: this.metrics.successRequests / this.metrics.totalRequests,
    };
  }
}
```

> **建议**：定期审查中间件配置和性能指标，根据实际使用情况调整配置参数。
