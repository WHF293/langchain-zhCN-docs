# 第8章 工具 (Tools)

> 来源: [LangChain 官方文档](https://python.langchain.com/)

---

## 概述

工具 (Tools) 是 LangChain 中用于扩展模型能力的核心组件。通过工具，语言模型可以与外部世界交互，执行搜索、计算、API 调用等操作。工具是构建 Agent 的基础，使模型能够完成超出纯文本生成的任务。

## 工具定义

### 基本工具创建

在 LangChain.js 中，可以使用 `tool` 函数创建工具：

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 定义一个简单的计算器工具
const calculatorTool = tool(
  async ({ expression }) => {
    try {
      // 使用 Function 构造器安全地计算数学表达式
      const result = new Function(`return ${expression}`)();
      return `计算结果: ${expression} = ${result}`;
    } catch (error) {
      return `计算错误: 无法计算表达式 "${expression}"`;
    }
  },
  {
    name: "calculator",
    description: "用于执行数学计算。输入应为有效的数学表达式。",
    schema: z.object({
      expression: z.string().describe("要计算的数学表达式，例如 '2 + 2' 或 '10 * 5'"),
    }),
  }
);

// 定义一个搜索工具
const searchTool = tool(
  async ({ query }) => {
    // 这里可以集成实际的搜索 API
    return `搜索 "${query}" 的结果: [模拟搜索结果]`;
  },
  {
    name: "web_search",
    description: "在互联网上搜索信息。用于获取实时数据或查找特定主题的资料。",
    schema: z.object({
      query: z.string().describe("搜索查询字符串"),
    }),
  }
);
```

### 工具参数定义

使用 Zod schema 定义工具参数，支持类型验证和描述：

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 定义具有复杂参数的工具
const fileReaderTool = tool(
  async ({ filePath, encoding, maxLines }) => {
    // 读取文件的逻辑
    return `读取文件 ${filePath} 的内容`;
  },
  {
    name: "file_reader",
    description: "读取指定路径的文件内容",
    schema: z.object({
      filePath: z.string().describe("文件的绝对路径"),
      encoding: z
        .enum(["utf-8", "ascii", "base64"])
        .default("utf-8")
        .optional()
        .describe("文件编码格式，默认为 utf-8"),
      maxLines: z
        .number()
        .int()
        .positive()
        .optional()
        .describe("最大读取行数，不设置则读取全部"),
    }),
  }
);
```

### 工具分类

| 工具类型 | 说明 | 示例 |
|---------|------|------|
| 查询工具 | 获取信息，不改变系统状态 | 搜索、数据库查询 |
| 操作工具 | 执行操作，可能改变系统状态 | 发送邮件、创建文件 |
| 计算工具 | 执行数学或逻辑运算 | 计算器、数据分析 |
| 交互工具 | 与用户或外部系统交互 | API 调用、消息发送 |

> **提示**：为工具编写清晰的 `description` 非常重要，模型会根据描述决定何时使用该工具。

## 工具运行时

### 工具绑定

将工具绑定到模型，使模型能够调用：

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 创建工具
const weatherTool = tool(
  async ({ city }) => {
    // 模拟天气 API 调用
    return `${city} 今天晴朗，温度 25°C`;
  },
  {
    name: "get_weather",
    description: "获取指定城市的当前天气信息",
    schema: z.object({
      city: z.string().describe("城市名称"),
    }),
  }
);

// 创建模型并绑定工具
const model = new ChatOpenAI({
  modelName: "gpt-4",
  temperature: 0,
});

const modelWithTools = model.bindTools([weatherTool]);

// 模型会自动决定是否调用工具
const response = await modelWithTools.invoke("北京今天天气怎么样？");

console.log(response.tool_calls);
// 输出: [{ name: "get_weather", args: { city: "北京" } }]
```

### 工具调用执行

手动执行工具调用：

```typescript
import { AIMessage, ToolMessage } from "@langchain/core/messages";

// 假设这是模型返回的消息
const aiMessage = new AIMessage({
  content: "",
  tool_calls: [
    {
      id: "call_123",
      name: "get_weather",
      args: { city: "上海" },
    },
  ],
});

// 执行工具调用
for (const toolCall of aiMessage.tool_calls) {
  const toolResult = await weatherTool.invoke(toolCall.args);
  console.log(`工具 ${toolCall.name} 返回: ${toolResult}`);
}
```

### 工具执行流程

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { HumanMessage, AIMessage, ToolMessage } from "@langchain/core/messages";

async function executeWithTools(model, tools, userMessage) {
  // 1. 绑定工具到模型
  const modelWithTools = model.bindTools(tools);

  // 2. 发送用户消息
  let messages = [new HumanMessage(userMessage)];
  let response = await modelWithTools.invoke(messages);

  // 3. 检查是否有工具调用
  while (response.tool_calls && response.tool_calls.length > 0) {
    messages.push(response);

    // 4. 执行所有工具调用
    for (const toolCall of response.tool_calls) {
      const selectedTool = tools.find((t) => t.name === toolCall.name);
      if (selectedTool) {
        const result = await selectedTool.invoke(toolCall.args);
        messages.push(
          new ToolMessage({
            content: result,
            tool_call_id: toolCall.id!,
          })
        );
      }
    }

    // 5. 再次调用模型，传入工具结果
    response = await modelWithTools.invoke(messages);
  }

  return response;
}
```

## 工具错误处理

### 基本错误处理

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

const safeApiTool = tool(
  async ({ url, method }) => {
    try {
      const response = await fetch(url, { method });

      if (!response.ok) {
        throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      }

      const data = await response.json();
      return JSON.stringify(data, null, 2);
    } catch (error) {
      // 返回错误信息而不是抛出异常
      // 这样模型可以理解发生了什么错误
      if (error instanceof Error) {
        return `工具执行错误: ${error.message}`;
      }
      return "工具执行错误: 未知错误";
    }
  },
  {
    name: "api_caller",
    description: "调用外部 API",
    schema: z.object({
      url: z.string().url().describe("API 端点 URL"),
      method: z.enum(["GET", "POST", "PUT", "DELETE"]).default("GET").describe("HTTP 方法"),
    }),
  }
);
```

### 使用 ErrorCode 处理错误

```typescript
import { tool } from "@langchain/core/tools";
import { ToolException } from "@langchain/core/tools";
import { z } from "zod";

const robustTool = tool(
  async ({ input }) => {
    // 模拟可能失败的操作
    if (input === "invalid") {
      throw new ToolException("输入参数无效");
    }
    if (input === "timeout") {
      throw new ToolException("操作超时");
    }
    return `处理结果: ${input}`;
  },
  {
    name: "robust_tool",
    description: "一个具有健壮错误处理的工具",
    schema: z.object({
      input: z.string(),
    }),
  }
);

// 自定义错误处理
const toolWithCustomErrorHandling = tool(
  async ({ input }) => {
    try {
      return await robustTool.invoke({ input });
    } catch (error) {
      if (error instanceof ToolException) {
        // 工具特定的错误处理
        return `工具错误: ${error.message}. 请检查输入参数后重试。`;
      }
      // 其他未知错误
      return "发生了意外错误，请稍后重试。";
    }
  },
  {
    name: "safe_tool_wrapper",
    description: "包装了错误处理的安全工具",
    schema: z.object({
      input: z.string(),
    }),
  }
);
```

### 错误重试机制

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

function createRetryableTool(maxRetries = 3, delay = 1000) {
  return tool(
    async ({ input }) => {
      let lastError: Error | null = null;

      for (let attempt = 1; attempt <= maxRetries; attempt++) {
        try {
          // 执行可能失败的操作
          const result = await someUnreliableOperation(input);
          return result;
        } catch (error) {
          lastError = error as Error;
          console.warn(`尝试 ${attempt}/${maxRetries} 失败: ${lastError.message}`);

          if (attempt < maxRetries) {
            // 等待后重试
            await new Promise((resolve) => setTimeout(resolve, delay * attempt));
          }
        }
      }

      // 所有重试都失败
      return `操作失败，已重试 ${maxRetries} 次。最后错误: ${lastError?.message}`;
    },
    {
      name: "retryable_tool",
      description: "具有自动重试机制的工具",
      schema: z.object({
        input: z.string(),
      }),
    }
  );
}
```

## 工具缓存

### 基本缓存实现

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";

// 简单的内存缓存
class ToolCache {
  private cache = new Map<string, { result: string; timestamp: number }>();
  private ttl: number; // 缓存过期时间（毫秒）

  constructor(ttl: number = 60000) {
    this.ttl = ttl;
  }

  get(key: string): string | null {
    const cached = this.cache.get(key);
    if (!cached) return null;

    // 检查是否过期
    if (Date.now() - cached.timestamp > this.ttl) {
      this.cache.delete(key);
      return null;
    }

    return cached.result;
  }

  set(key: string, result: string): void {
    this.cache.set(key, {
      result,
      timestamp: Date.now(),
    });
  }

  clear(): void {
    this.cache.clear();
  }
}

// 创建带缓存的工具
function createCachedTool<T extends Record<string, any>>(
  toolFn: (args: T) => Promise<string>,
  toolConfig: {
    name: string;
    description: string;
    schema: z.ZodSchema<T>;
    cacheTTL?: number;
  }
) {
  const cache = new ToolCache(toolConfig.cacheTTL);

  return tool(
    async (args: T) => {
      // 生成缓存键
      const cacheKey = JSON.stringify(args);

      // 检查缓存
      const cachedResult = cache.get(cacheKey);
      if (cachedResult) {
        console.log(`缓存命中: ${toolConfig.name}`);
        return cachedResult;
      }

      // 执行工具函数
      const result = await toolFn(args);

      // 存入缓存
      cache.set(cacheKey, result);

      return result;
    },
    {
      name: toolConfig.name,
      description: toolConfig.description + " (支持结果缓存)",
      schema: toolConfig.schema,
    }
  );
}

// 使用示例
const cachedSearchTool = createCachedTool(
  async ({ query }) => {
    // 模拟耗时的搜索操作
    await new Promise((resolve) => setTimeout(resolve, 1000));
    return `搜索结果: ${query} 相关的内容`;
  },
  {
    name: "cached_search",
    description: "带缓存的搜索工具",
    schema: z.object({
      query: z.string().describe("搜索查询"),
    }),
    cacheTTL: 300000, // 5 分钟缓存
  }
);
```

### 缓存策略

| 缓存策略 | 说明 | 适用场景 |
|---------|------|---------|
| 内存缓存 | 存储在进程内存中 | 单次会话内的重复查询 |
| 文件缓存 | 持久化到文件系统 | 需要跨会话保留的缓存 |
| Redis 缓存 | 使用 Redis 存储 | 分布式系统，需要共享缓存 |
| TTL 策略 | 基于时间的过期策略 | 数据有时效性的场景 |

> **建议**：对于频繁调用且结果相对稳定的工具（如配置查询、元数据获取），启用缓存可以显著提高响应速度并减少外部 API 调用。

### 分布式缓存示例

```typescript
import { tool } from "@langchain/core/tools";
import { z } from "zod";
import { createClient } from "redis";

// Redis 缓存实现
class RedisToolCache {
  private client: ReturnType<typeof createClient>;
  private prefix: string;
  private ttl: number;

  constructor(redisUrl: string, prefix: string = "tool_cache:", ttl: number = 3600) {
    this.client = createClient({ url: redisUrl });
    this.prefix = prefix;
    this.ttl = ttl;
  }

  async connect() {
    await this.client.connect();
  }

  async get(key: string): Promise<string | null> {
    const cached = await this.client.get(this.prefix + key);
    return cached;
  }

  async set(key: string, value: string): Promise<void> {
    await this.client.setEx(this.prefix + key, this.ttl, value);
  }

  async delete(key: string): Promise<void> {
    await this.client.del(this.prefix + key);
  }
}

// 使用 Redis 缓存的工具
async function createDistributedCachedTool() {
  const cache = new RedisToolCache("redis://localhost:6379");
  await cache.connect();

  return tool(
    async ({ query }) => {
      const cacheKey = `search:${query}`;

      // 检查缓存
      const cached = await cache.get(cacheKey);
      if (cached) {
        return cached;
      }

      // 执行查询
      const result = await performExpensiveSearch(query);

      // 存入缓存
      await cache.set(cacheKey, result);

      return result;
    },
    {
      name: "distributed_search",
      description: "带分布式缓存的搜索工具",
      schema: z.object({
        query: z.string(),
      }),
    }
  );
}
```

## 最佳实践

1. **清晰的工具描述**：确保工具描述准确反映其功能和预期输入
2. **参数验证**：使用 Zod schema 严格验证输入参数
3. **错误处理**：优雅地处理错误，返回有用的错误信息
4. **超时控制**：为可能长时间运行的工具设置超时
5. **幂等性**：尽量使工具操作具有幂等性，避免重复调用产生副作用
6. **日志记录**：记录工具调用以便调试和监控

> **注意**：工具的权限管理非常重要。确保工具只能访问必要的资源，避免安全风险。
