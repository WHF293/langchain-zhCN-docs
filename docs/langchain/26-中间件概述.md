# 第26章 中间件概述

> 来源: [LangChain 官方文档](https://python.langchain.com/)

---

## 什么是中间件

中间件（Middleware）是一种在请求处理流程中插入自定义逻辑的机制。在 LangChain 中，中间件允许开发者在 LLM 调用前后执行自定义操作，实现请求/响应的拦截、转换和增强。

### 中间件的作用

| 功能 | 说明 | 示例 |
|------|------|------|
| 请求预处理 | 在发送到 LLM 前修改请求 | 添加系统提示、注入上下文 |
| 响应后处理 | 在返回给用户前修改响应 | 格式转换、内容过滤 |
| 错误处理 | 捕获和处理异常 | 重试逻辑、降级策略 |
| 监控日志 | 记录请求和响应 | 性能监控、审计日志 |
| 认证授权 | 验证用户身份和权限 | API 密钥验证、角色检查 |

### 中间件与回调的区别

| 特性 | 中间件 | 回调 |
|------|--------|------|
| 执行时机 | 请求前后 | 事件触发 |
| 修改能力 | 可修改请求/响应 | 只能观察 |
| 顺序控制 | 支持优先级 | 固定顺序 |
| 适用场景 | 全局逻辑 | 特定事件 |

```typescript
// 中间件基本概念示例
interface Middleware {
  name: string;
  priority: number;

  // 请求前处理
  onBeforeRequest?(request: Request): Promise<Request>;

  // 响应后处理
  onAfterResponse?(response: Response): Promise<Response>;

  // 错误处理
  onError?(error: Error): Promise<void>;
}
```

> **注意**：中间件适合处理全局性的逻辑，如日志、认证、限流等。特定业务逻辑应放在具体的处理函数中。

## 中间件架构

LangChain 的中间件架构采用洋葱模型（Onion Model），请求从外到内穿过中间件，响应从内到外返回。

### 洋葱模型

```
请求 → 中间件1 (before) → 中间件2 (before) → LLM → 中间件2 (after) → 中间件1 (after) → 响应
```

### 架构设计

```typescript
// 中间件管理器
class MiddlewareManager {
  private middlewares: Middleware[] = [];

  // 注册中间件
  use(middleware: Middleware): void {
    this.middlewares.push(middleware);
    // 按优先级排序
    this.middlewares.sort((a, b) => a.priority - b.priority);
  }

  // 执行中间件链
  async execute(
    request: Request,
    handler: (req: Request) => Promise<Response>
  ): Promise<Response> {
    let currentRequest = request;

    // 执行请求前处理
    for (const middleware of this.middlewares) {
      if (middleware.onBeforeRequest) {
        currentRequest = await middleware.onBeforeRequest(currentRequest);
      }
    }

    // 调用处理器
    let response = await handler(currentRequest);

    // 执行响应后处理（反序）
    for (const middleware of this.middlewares.reverse()) {
      if (middleware.onAfterResponse) {
        response = await middleware.onAfterResponse(response);
      }
    }

    return response;
  }
}
```

### 中间件优先级

| 优先级 | 范围 | 用途 | 示例 |
|--------|------|------|------|
| 最高 | 0-100 | 安全相关 | 认证、授权 |
| 高 | 101-200 | 基础设施 | 限流、缓存 |
| 中 | 201-300 | 业务逻辑 | 日志、监控 |
| 低 | 301-400 | 辅助功能 | 格式转换 |

```typescript
// 优先级定义
const Priority = {
  AUTHENTICATION: 10,
  RATE_LIMIT: 50,
  CACHING: 100,
  LOGGING: 200,
  TRANSFORMATION: 300,
} as const;

// 使用示例
const authMiddleware: Middleware = {
  name: "auth",
  priority: Priority.AUTHENTICATION,
  onBeforeRequest: async (req) => {
    // 验证 token
    const token = req.headers.get("Authorization");
    if (!token) {
      throw new Error("未授权");
    }
    return req;
  },
};

const loggingMiddleware: Middleware = {
  name: "logging",
  priority: Priority.LOGGING,
  onBeforeRequest: async (req) => {
    console.log(`[${new Date().toISOString()}] 请求开始:`, req.url);
    return req;
  },
  onAfterResponse: async (res) => {
    console.log(`[${new Date().toISOString()}] 请求完成`);
    return res;
  },
};
```

> **建议**：合理设置中间件优先级，确保安全相关的中间件最先执行。

## 中间件生命周期

中间件的生命周期与请求处理流程紧密相关，理解生命周期有助于正确实现中间件逻辑。

### 生命周期阶段

```typescript
// 中间件生命周期
interface MiddlewareLifecycle {
  // 1. 初始化阶段
  onInit?(): Promise<void>;

  // 2. 请求接收阶段
  onBeforeRequest?(request: Request): Promise<Request>;

  // 3. 处理执行阶段
  onProcess?(request: Request): Promise<Response>;

  // 4. 响应返回阶段
  onAfterResponse?(response: Response): Promise<Response>;

  // 5. 错误处理阶段
  onError?(error: Error, request: Request): Promise<Response>;

  // 6. 清理阶段
  onCleanup?(): Promise<void>;
}
```

### 生命周期示例

```typescript
// 完整生命周期中间件示例
class LifecycleMiddleware implements Middleware {
  name = "lifecycle-example";
  priority = 200;

  async onInit(): Promise<void> {
    console.log("中间件初始化");
    // 初始化资源，如数据库连接、缓存等
  }

  async onBeforeRequest(request: Request): Promise<Request> {
    console.log("请求前处理");

    // 添加请求 ID
    const requestId = crypto.randomUUID();
    request.headers.set("X-Request-ID", requestId);

    // 记录开始时间
    (request as any).startTime = Date.now();

    return request;
  }

  async onAfterResponse(response: Response): Promise<Response> {
    console.log("响应后处理");

    // 计算请求耗时
    const startTime = (response as any).startTime;
    const duration = Date.now() - startTime;

    // 添加响应头
    response.headers.set("X-Request-Duration", `${duration}ms`);

    return response;
  }

  async onError(error: Error, request: Request): Promise<Response> {
    console.error("错误处理:", error.message);

    // 返回友好的错误响应
    return new Response(
      JSON.stringify({
        error: "服务器内部错误",
        requestId: request.headers.get("X-Request-ID"),
      }),
      { status: 500 }
    );
  }

  async onCleanup(): Promise<void> {
    console.log("中间件清理");
    // 清理资源
  }
}
```

### 异步处理

```typescript
// 异步中间件处理
class AsyncMiddleware implements Middleware {
  name = "async-example";
  priority = 200;

  async onBeforeRequest(request: Request): Promise<Request> {
    // 异步操作示例：从缓存获取数据
    const cachedData = await this.getCachedData(request.url);
    if (cachedData) {
      (request as any).cachedData = cachedData;
    }

    return request;
  }

  private async getCachedData(url: string): Promise<any> {
    // 模拟缓存查询
    return new Promise((resolve) => {
      setTimeout(() => resolve(null), 100);
    });
  }
}
```

> **注意**：中间件中的异步操作应设置合理的超时时间，避免阻塞整个请求处理流程。

## 中间件配置

中间件配置允许灵活地控制中间件的行为，支持静态配置和动态配置。

### 配置方式

```typescript
// 中间件配置接口
interface MiddlewareConfig {
  enabled: boolean;
  priority: number;
  options: Record<string, any>;
  // 条件执行
  conditions?: {
    paths?: string[];
    methods?: string[];
    headers?: Record<string, string>;
  };
}

// 配置示例
const configs: Record<string, MiddlewareConfig> = {
  auth: {
    enabled: true,
    priority: 10,
    options: {
      secretKey: process.env.JWT_SECRET,
      excludePaths: ["/health", "/metrics"],
    },
  },
  rateLimit: {
    enabled: true,
    priority: 50,
    options: {
      windowMs: 60000,  // 1分钟窗口
      maxRequests: 100, // 最大请求数
    },
    conditions: {
      paths: ["/api/*"],
    },
  },
  logging: {
    enabled: process.env.NODE_ENV !== "production",
    priority: 200,
    options: {
      level: "debug",
      format: "json",
    },
  },
};
```

### 条件执行

```typescript
// 条件中间件
class ConditionalMiddleware implements Middleware {
  name = "conditional";
  priority = 200;
  private config: MiddlewareConfig;

  constructor(config: MiddlewareConfig) {
    this.config = config;
  }

  async onBeforeRequest(request: Request): Promise<Request> {
    // 检查是否应该执行
    if (!this.shouldExecute(request)) {
      return request;
    }

    // 执行中间件逻辑
    return this.process(request);
  }

  private shouldExecute(request: Request): boolean {
    // 检查是否启用
    if (!this.config.enabled) return false;

    // 检查路径条件
    if (this.config.conditions?.paths) {
      const url = new URL(request.url);
      const matchesPath = this.config.conditions.paths.some((pattern) => {
        const regex = new RegExp(pattern.replace("*", ".*"));
        return regex.test(url.pathname);
      });
      if (!matchesPath) return false;
    }

    // 检查方法条件
    if (this.config.conditions?.methods) {
      if (!this.config.conditions.methods.includes(request.method)) {
        return false;
      }
    }

    return true;
  }

  private async process(request: Request): Promise<Request> {
    // 中间件处理逻辑
    return request;
  }
}
```

### 动态配置

```typescript
// 动态配置管理
class DynamicConfigManager {
  private configs: Map<string, MiddlewareConfig> = new Map();
  private watchers: Map<string, Function[]> = new Map();

  // 获取配置
  get(middlewareName: string): MiddlewareConfig | undefined {
    return this.configs.get(middlewareName);
  }

  // 更新配置
  update(middlewareName: string, config: Partial<MiddlewareConfig>): void {
    const existing = this.configs.get(middlewareName) || {
      enabled: true,
      priority: 200,
      options: {},
    };

    const updated = { ...existing, ...config };
    this.configs.set(middlewareName, updated);

    // 通知监听者
    this.notifyWatchers(middlewareName, updated);
  }

  // 监听配置变化
  watch(middlewareName: string, callback: (config: MiddlewareConfig) => void): void {
    const watchers = this.watchers.get(middlewareName) || [];
    watchers.push(callback);
    this.watchers.set(middlewareName, watchers);
  }

  private notifyWatchers(middlewareName: string, config: MiddlewareConfig): void {
    const watchers = this.watchers.get(middlewareName) || [];
    watchers.forEach((callback) => callback(config));
  }
}

// 使用示例
const configManager = new DynamicConfigManager();

// 监听配置变化
configManager.watch("rateLimit", (config) => {
  console.log("限流配置已更新:", config);
});

// 动态更新配置
configManager.update("rateLimit", {
  options: { maxRequests: 200 },
});
```

> **提示**：在生产环境中，建议使用配置中心（如 Apollo、Nacos）管理中间件配置，支持动态更新而不需重启服务。
