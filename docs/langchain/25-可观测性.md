## 追踪

追踪（Tracing）是可观测性的核心组件，用于记录和分析 LLM 应用的执行流程。LangChain 提供了强大的追踪功能，帮助开发者理解应用行为。

### 追踪的重要性

| 追踪维度 | 说明 | 价值 |
|----------|------|------|
| 执行流程 | 记录每个步骤的输入输出 | 理解应用逻辑 |
| 性能分析 | 测量各步骤耗时 | 识别性能瓶颈 |
| 成本监控 | 追踪 Token 使用量 | 控制 API 成本 |
| 错误诊断 | 记录异常和堆栈 | 快速定位问题 |

### LangSmith 追踪集成

```typescript
// LangSmith 追踪配置
import { Client } from "langsmith";
import { traceable } from "langsmith/traceable";

// 设置环境变量
process.env.LANGCHAIN_TRACING_V2 = "true";
process.env.LANGCHAIN_API_KEY = "your-langsmith-api-key";
process.env.LANGCHAIN_PROJECT = "my-project";

// 创建可追踪的函数
const tracedLLMCall = traceable(
  async (prompt: string) => {
    const llm = new ChatOpenAI({ modelName: "gpt-4" });
    const response = await llm.invoke(prompt);
    return response.content;
  },
  { name: "llm-call" }
);

// 使用追踪函数
const result = await tracedLLMCall("你好，请介绍一下自己");
```

## 日志

日志（Logging）是记录应用运行时信息的重要手段。合理的日志策略可以帮助快速定位问题。

### 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 详细的调试信息 | API 请求参数、中间结果 |
| INFO | 一般运行信息 | 任务开始、完成 |
| WARNING | 警告信息 | 接近超时、降级处理 |
| ERROR | 错误信息 | API 调用失败、异常 |

### 结构化日志

```typescript
// 结构化日志实现
import winston from "winston";

// 创建日志记录器
const logger = winston.createLogger({
  level: "info",
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.File({
      filename: "logs/error.log",
      level: "error",
    }),
    new winston.transports.File({
      filename: "logs/combined.log",
    }),
  ],
});

// 开发环境添加控制台输出
if (process.env.NODE_ENV !== "production") {
  logger.add(new winston.transports.Console({
    format: winston.format.simple(),
  }));
}

// LangChain 日志集成
import { BaseCallbackHandler } from "langchain/callbacks";

class LoggingHandler extends BaseCallbackHandler {
  name = "LoggingHandler";

  async handleLLMStart(llm: any, prompts: string[]) {
    logger.info("LLM 调用开始", {
      model: llm.id,
      promptCount: prompts.length,
    });
  }

  async handleLLMEnd(output: any) {
    logger.info("LLM 调用完成", {
      tokenUsage: output.llmOutput?.tokenUsage,
    });
  }

  async handleLLMError(error: Error) {
    logger.error("LLM 调用失败", {
      error: error.message,
      stack: error.stack,
    });
  }

  async handleChainStart(chain: any) {
    logger.debug("Chain 执行开始", {
      chainName: chain.id,
    });
  }

  async handleToolStart(tool: any, input: string) {
    logger.info("Tool 调用开始", {
      toolName: tool.name,
      input: input.substring(0, 100),  // 限制日志长度
    });
  }
}
```

### 日志脱敏

```typescript
// 日志脱敏处理
class SanitizedLogger {
  private sensitiveFields = ["password", "token", "apiKey", "secret"];
  private patterns = [
    { regex: /\b\d{16,19}\b/g, replacement: "****" },  // 银行卡
    { regex: /\b\w+@\w+\.\w+\b/g, replacement: "***@***.***" },  // 邮箱
    { regex: /\b1[3-9]\d{9}\b/g, replacement: "***********" },  // 手机号
  ];

  sanitize(data: any): any {
    if (typeof data === "string") {
      return this.sanitizeString(data);
    }

    if (Array.isArray(data)) {
      return data.map((item) => this.sanitize(item));
    }

    if (typeof data === "object" && data !== null) {
      const sanitized: any = {};
      for (const [key, value] of Object.entries(data)) {
        // 跳过敏感字段
        if (this.sensitiveFields.includes(key)) {
          sanitized[key] = "***";
        } else {
          sanitized[key] = this.sanitize(value);
        }
      }
      return sanitized;
    }

    return data;
  }

  private sanitizeString(str: string): string {
    let result = str;
    for (const { regex, replacement } of this.patterns) {
      result = result.replace(regex, replacement);
    }
    return result;
  }
}
```

> **注意**：日志中不应包含敏感信息（如密码、API 密钥、个人身份信息等）。

## 指标

指标（Metrics）是量化应用性能和行为的数据。通过指标监控，可以及时发现和解决性能问题。

### 关键指标

| 指标类别 | 具体指标 | 说明 |
|----------|----------|------|
| 延迟指标 | 首次响应时间 (TTFT) | 从请求到首个响应的时间 |
| | 总响应时间 | 请求完成的总时间 |
| | 流式延迟 | 流式响应的逐块延迟 |
| 吞吐指标 | 请求/秒 (QPS) | 每秒处理的请求数 |
| | Token/秒 | 每秒生成的 Token 数 |
| 成本指标 | Token 使用量 | 输入和输出的 Token 数 |
| | API 成本 | 基于 Token 的费用 |
| 质量指标 | 成功率 | 成功完成的请求比例 |
| | 错误率 | 失败请求的比例 |

### 指标收集

```typescript
// 指标收集器
interface MetricPoint {
  name: string;
  value: number;
  timestamp: Date;
  tags: Record<string, string>;
}

class MetricsCollector {
  private metrics: MetricPoint[] = [];
  private flushInterval: NodeJS.Timer;

  constructor() {
    // 定期刷新指标
    this.flushInterval = setInterval(() => {
      this.flush();
    }, 60000);  // 每分钟刷新
  }

  // 记录指标
  record(
    name: string,
    value: number,
    tags: Record<string, string> = {}
  ): void {
    this.metrics.push({
      name,
      value,
      timestamp: new Date(),
      tags,
    });
  }

  // 记录延迟
  recordLatency(operation: string, latencyMs: number): void {
    this.record("latency", latencyMs, { operation });
  }

  // 记录 Token 使用
  recordTokenUsage(
    model: string,
    promptTokens: number,
    completionTokens: number
  ): void {
    this.record("prompt_tokens", promptTokens, { model });
    this.record("completion_tokens", completionTokens, { model });
    this.record("total_tokens", promptTokens + completionTokens, { model });
  }

  // 记录错误
  recordError(operation: string, errorType: string): void {
    this.record("error", 1, { operation, error_type: errorType });
  }

  // 刷新指标到监控系统
  private async flush(): Promise<void> {
    if (this.metrics.length === 0) return;

    const metricsToSend = [...this.metrics];
    this.metrics = [];

    // 发送到 Prometheus、DataDog 等监控系统
    await this.sendToMonitoring(metricsToSend);
  }

  private async sendToMonitoring(metrics: MetricPoint[]): Promise<void> {
    // 实现指标发送逻辑
    console.log(`发送 ${metrics.length} 个指标数据点`);
  }
}

// 使用示例
const metrics = new MetricsCollector();

// 在 LLM 调用中记录指标
async function callLLM(prompt: string): Promise<string> {
  const startTime = Date.now();

  try {
    const llm = new ChatOpenAI({ modelName: "gpt-4" });
    const response = await llm.invoke(prompt);

    const latency = Date.now() - startTime;
    metrics.recordLatency("llm_call", latency);
    metrics.recordTokenUsage(
      "gpt-4",
      response.usage?.promptTokens || 0,
      response.usage?.completionTokens || 0
    );

    return response.content as string;
  } catch (error) {
    metrics.recordError("llm_call", error.constructor.name);
    throw error;
  }
}
```

> **建议**：建立指标告警机制，当关键指标异常时及时通知相关人员。

### 详细调试输出

// 自定义调试处理器
import { BaseCallbackHandler } from "langchain/callbacks";

class DebugHandler extends BaseCallbackHandler {
  name = "DebugHandler";

  }

  async handleChainEnd(outputs: any) {
    console.log("=== Chain 执行完成 ===");
    console.log("输出:", JSON.stringify(outputs, null, 2));

### 调试工具

```typescript
// 调试工具集
class LangChainDebugger {
  // 分析 Chain 执行
  async analyzeChainExecution(chain: any, input: any): Promise<void> {
    console.log("开始分析 Chain 执行...");

    const tracer = new CustomTracer();
    const result = await chain.invoke(input, {
      callbacks: [tracer],
    });

    console.log("执行结果:", result);
  }

  // 测试提示词
  async testPrompt(
    prompt: string,
    variables: Record<string, string>
  ): Promise<void> {
    const llm = new ChatOpenAI({ modelName: "gpt-4" });

    // 格式化提示词
    let formattedPrompt = prompt;
    for (const [key, value] of Object.entries(variables)) {
      formattedPrompt = formattedPrompt.replace(`{${key}}`, value);
    }

    console.log("格式化后的提示词:", formattedPrompt);

    const response = await llm.invoke(formattedPrompt);
    console.log("模型响应:", response.content);
  }

  // 比较不同模型的响应
  async compareModels(
    prompt: string,
    models: string[]
  ): Promise<Map<string, string>> {
    const results = new Map<string, string>();

    for (const model of models) {
      const llm = new ChatOpenAI({ modelName: model });
      const response = await llm.invoke(prompt);
      results.set(model, response.content as string);
    }

    return results;
  }
}
```

> **提示**：在开发阶段启用详细调试日志，但在生产环境中应关闭或降低日志级别，避免影响性能。
