# 第16章 运行时

> 来源: [LangChain 官方文档](https://python.langchain.com/)

---

## 概述

运行时（Runtime）是 LangChain 应用程序执行时的环境和上下文。它管理着配置、依赖、错误处理等关键功能，确保应用程序能够稳定可靠地运行。

## 运行时上下文

运行时上下文（Runtime Context）包含了应用程序执行时所需的各种信息和状态。

### 上下文类型

| 上下文类型 | 说明 | 使用场景 |
|-----------|------|---------|
| 用户上下文 | 当前用户信息和权限 | 个性化响应、权限控制 |
| 会话上下文 | 会话状态和历史 | 多轮对话、状态保持 |
| 请求上下文 | 请求相关信息 | 请求追踪、日志记录 |
| 环境上下文 | 运行环境配置 | 环境特定行为 |

### 创建上下文

```typescript
import { RunnableConfig } from "@langchain/core/runnables";

// 创建运行时配置
const config: RunnableConfig = {
  // 可配置字段
  configurable: {
    userId: "user-123",
    sessionId: "session-456",
    model: "gpt-4"
  },
  // 运行名称
  runName: "my-chain",
  // 标签
  tags: ["production", "v1"],
  // 元数据
  metadata: {
    environment: "production",
    version: "1.0.0"
  }
};

// 使用配置调用链
const result = await chain.invoke(
  { input: "你好" },
  config
);
```

### 访问上下文

```typescript
import { RunnableLambda } from "@langchain/core/runnables";

// 创建可运行对象，在其中访问配置
const myRunnable = RunnableLambda.from(
  async (input: string, config?: RunnableConfig) => {
    // 获取配置中的用户ID
    const userId = config?.configurable?.userId;

    // 根据用户ID进行个性化处理
    const personalizedInput = `用户 ${userId} 说: ${input}`;

    return personalizedInput;
  }
);

// 调用时传入配置
const result = await myRunnable.invoke("你好", {
  configurable: { userId: "user-123" }
});
```

## 运行时配置

运行时配置（Runtime Configuration）允许你在运行时动态调整应用程序的行为。

### 配置选项

```typescript
import { RunnableConfig } from "@langchain/core/runnables";

// 完整配置示例
const fullConfig: RunnableConfig = {
  // 可配置参数 - 用于动态调整行为
  configurable: {
    temperature: 0.7,
    maxTokens: 1000,
    modelName: "gpt-4",
    systemPrompt: "你是一个有帮助的助手"
  },

  // 运行名称 - 用于追踪和调试
  runName: "qa-chain",

  // 标签 - 用于分类和过滤
  tags: ["qa", "production"],

  // 元数据 - 存储额外信息
  metadata: {
    requestId: "req-789",
    timestamp: Date.now(),
    source: "api"
  },

  // 最大并发数
  maxConcurrency: 5,

  // 超时设置（毫秒）
  timeout: 30000,

  // 回调函数
  callbacks: []
};
```

### 动态配置

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { RunnableSequence } from "@langchain/core/runnables";

// 创建支持动态配置的模型
const model = new ChatOpenAI({
  // 使用配置中的模型名称，否则使用默认值
  modelName: "gpt-3.5-turbo",
  temperature: 0.7
});

// 创建链，使用可配置字段
const chain = RunnableSequence.from([
  // 第一步：根据配置调整提示词
  RunnableLambda.from(async (input, config) => {
    const systemPrompt = config?.configurable?.systemPrompt
      || "你是一个有帮助的助手";
    return {
      messages: [
        { role: "system", content: systemPrompt },
        { role: "user", content: input }
      ]
    };
  }),
  // 第二步：调用模型
  model
]);

// 使用不同配置调用
const result1 = await chain.invoke("你好", {
  configurable: {
    systemPrompt: "你是一个专业的翻译助手"
  }
});

const result2 = await chain.invoke("你好", {
  configurable: {
    systemPrompt: "你是一个友好的聊天伙伴"
  }
});
```

### 配置绑定

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { ConfigurableField } from "@langchain/core/runnables";

// 创建可配置的模型
const model = new ChatOpenAI({
  temperature: 0.7
}).configurableFields({
  // 将 temperature 字段设为可配置
  temperature: new ConfigurableField({
    id: "llm_temperature",
    name: "LLM Temperature",
    description: "控制输出的随机性，值越高越随机"
  }),
  // 将 modelName 字段设为可配置
  modelName: new ConfigurableField({
    id: "llm_model",
    name: "LLM Model",
    description: "要使用的模型名称"
  })
});

// 使用默认配置
const result1 = await model.invoke("你好");

// 使用自定义配置
const result2 = await model.invoke("你好", {
  configurable: {
    llm_temperature: 0.9,
    llm_model: "gpt-4"
  }
});
```

> **提示**：使用 `configurableFields` 可以将任何模型参数设为可配置，这在需要动态切换模型或调整参数时非常有用。

## 运行时依赖

运行时依赖（Runtime Dependencies）管理着应用程序所需的各种外部资源和服务。

### 依赖注入

```typescript
import { RunnableConfig } from "@langchain/core/runnables";

// 定义依赖接口
interface AppDependencies {
  llm: ChatOpenAI;
  vectorStore: VectorStore;
  logger: Logger;
  cache: Cache;
}

// 通过配置注入依赖
const config: RunnableConfig = {
  configurable: {
    // 注入依赖
    dependencies: {
      llm: new ChatOpenAI({ modelName: "gpt-4" }),
      vectorStore: new Chroma(embeddings, { collectionName: "docs" }),
      logger: console,
      cache: new RedisCache()
    } as AppDependencies
  }
};

// 在链中使用依赖
const chain = RunnableLambda.from(
  async (input: string, config?: RunnableConfig) => {
    const deps = config?.configurable?.dependencies as AppDependencies;

    // 使用注入的依赖
    deps.logger.info("处理输入:", input);

    // 从向量存储检索
    const docs = await deps.vectorStore.similaritySearch(input, 3);

    // 使用LLM生成回答
    const response = await deps.llm.invoke(
      `基于以下文档回答问题:\n${docs.map(d => d.pageContent).join("\n")}\n\n问题: ${input}`
    );

    return response.content;
  }
);
```

### 外部服务集成

```typescript
import { RunnableLambda } from "@langchain/core/runnables";

// 创建数据库服务
const databaseService = RunnableLambda.from(
  async (query: string, config?: RunnableConfig) => {
    const dbConfig = config?.configurable?.database;

    // 连接数据库
    const connection = await createConnection({
      host: dbConfig?.host || "localhost",
      port: dbConfig?.port || 5432,
      database: dbConfig?.name || "mydb"
    });

    // 执行查询
    const result = await connection.query(query);

    return result;
  }
);

// 创建缓存服务
const cacheService = RunnableLambda.from(
  async (key: string, config?: RunnableConfig) => {
    const cacheConfig = config?.configurable?.cache;

    // 检查缓存
    const cached = await redis.get(key);

    if (cached) {
      return JSON.parse(cached);
    }

    return null;
  }
);
```

## 运行时错误处理

运行时错误处理（Runtime Error Handling）确保应用程序能够优雅地处理各种异常情况。

### 错误类型

| 错误类型 | 说明 | 处理策略 |
|---------|------|---------|
| 网络错误 | API调用失败、超时 | 重试、降级 |
| 验证错误 | 输入格式不正确 | 返回错误提示 |
| 限流错误 | API速率限制 | 等待、重试 |
| 认证错误 | API密钥无效 | 提示更新配置 |
| 业务错误 | 业务逻辑异常 | 返回友好提示 |

### 基本错误处理

```typescript
import { RunnableLambda } from "@langchain/core/runnables";

// 创建带错误处理的链
const safeChain = RunnableLambda.from(
  async (input: string, config?: RunnableConfig) => {
    try {
      // 尝试执行主要逻辑
      const result = await mainLogic(input);
      return { success: true, data: result };
    } catch (error) {
      // 处理不同类型的错误
      if (error instanceof ValidationError) {
        return {
          success: false,
          error: "输入格式不正确",
          details: error.message
        };
      }

      if (error instanceof RateLimitError) {
        return {
          success: false,
          error: "请求过于频繁，请稍后再试"
        };
      }

      // 未知错误
      console.error("未知错误:", error);
      return {
        success: false,
        error: "处理请求时出现错误，请稍后再试"
      };
    }
  }
);
```

### 重试机制

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { RunnableRetry } from "@langchain/core/runnables";

// 创建带重试机制的模型
const modelWithRetry = new ChatOpenAI({
  modelName: "gpt-4",
  temperature: 0.7,
  maxRetries: 3  // 最多重试3次
});

// 或者使用 RunnableRetry
const retriableModel = new ChatOpenAI({
  modelName: "gpt-4"
}).withRetry({
  stopAfterAttempt: 3,  // 最多尝试3次
  waitBetweenAttempts: (attemptNumber) =>
    Math.min(1000 * Math.pow(2, attemptNumber), 10000)  // 指数退避，最大10秒
});

// 使用重试模型
const chain = RunnableSequence.from([
  // 输入处理
  RunnableLambda.from((input: string) => {
    return {
      messages: [{ role: "user", content: input }]
    };
  }),
  // 带重试的模型调用
  retriableModel
]);

const result = await chain.invoke("你好");
```

### 降级策略

```typescript
import { ChatOpenAI } from "@langchain/openai";
import { RunnableWithFallbacks } from "@langchain/core/runnables";

// 创建主模型和备用模型
const primaryModel = new ChatOpenAI({
  modelName: "gpt-4",
  temperature: 0.7
});

const fallbackModel = new ChatOpenAI({
  modelName: "gpt-3.5-turbo",
  temperature: 0.7
});

// 创建带降级的链
const chainWithFallback = primaryModel.withFallbacks([fallbackModel]);

// 如果主模型失败，会自动尝试备用模型
const result = await chainWithFallback.invoke("你好");
```

### 超时处理

```typescript
import { ChatOpenAI } from "@langchain/openai";

// 创建带超时的模型
const modelWithTimeout = new ChatOpenAI({
  modelName: "gpt-4",
  timeout: 10000  // 10秒超时
});

// 使用 AbortController 手动控制超时
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 10000);

try {
  const result = await modelWithTimeout.invoke(
    "你好",
    { signal: controller.signal }
  );
  clearTimeout(timeoutId);
  console.log(result);
} catch (error) {
  if (error.name === "AbortError") {
    console.log("请求超时");
  } else {
    console.error("其他错误:", error);
  }
}
```

> **注意**：在生产环境中，建议同时配置重试机制和超时处理，以确保应用的健壮性。

> **建议**：为不同类型的错误定义清晰的错误码和错误信息，方便前端展示和问题排查。

## 最佳实践

1. **配置管理**：使用配置中心或环境变量管理敏感配置
2. **依赖注入**：通过配置注入依赖，便于测试和维护
3. **错误分类**：区分可重试错误和不可重试错误
4. **日志记录**：记录关键操作和错误信息
5. **监控告警**：对关键指标设置监控和告警
