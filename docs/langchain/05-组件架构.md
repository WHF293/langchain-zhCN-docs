# 第5章 组件架构

> 来源: [LangChain 官方文档](https://python.langchain.com/)

---

LangChain 的组件架构分为三个层次：核心组件、扩展组件和集成组件。这种分层设计使得框架既稳定又灵活。

## 架构概览

```
┌─────────────────────────────────────────────────────┐
│                   应用层 (Application)                │
├─────────────────────────────────────────────────────┤
│              集成组件 (Integrations)                   │
│   @langchain/openai  @langchain/anthropic  ...      │
├─────────────────────────────────────────────────────┤
│              扩展组件 (Extensions)                     │
│      @langchain/langgraph  @langchain/textsplitters  │
├─────────────────────────────────────────────────────┤
│               核心组件 (Core)                          │
│                    @langchain/core                    │
└─────────────────────────────────────────────────────┘
```

## 核心组件

核心组件位于 `@langchain/core` 包中，定义了框架的基础接口和通用功能。

### Runnable 接口

所有 LangChain 组件的基础接口：

```typescript
import { Runnable } from "@langchain/core/runnables";

// Runnable 接口定义的核心方法
interface Runnable<Input, Output> {
  invoke(input: Input, config?: RunnableConfig): Promise<Output>;
  batch(inputs: Input[], config?: RunnableConfig): Promise<Output[]>;
  stream(input: Input, config?: RunnableConfig): Promise<IterableReadableStream<Output>>;
  streamEvents(input: Input, options: StreamEventsOptions): AsyncGenerator<StreamEvent>;
  pipe<Next>(next: Runnable<Output, Next>): RunnableSequence<Input, Next>;
  withConfig(config: RunnableConfig): Runnable<Input, Output>;
  withRetry(retryParams?: RetryParams): Runnable<Input, Output>;
  bind(kwargs: Partial<Input>): Runnable<Input, Output>;
}
```

### 模型接口

定义了与 LLM 交互的标准接口：

```typescript
import { BaseLanguageModel } from "@langchain/core/language_models/base";
import { BaseChatModel } from "@langchain/core/language_models/chat_models";
import { BaseLLM } from "@langchain/core/language_models/llms";

// 聊天模型基类
class BaseChatModel extends BaseLanguageModel {
  abstract _generate(messages, options): Promise<ChatResult>;
  _call(messages, options): Promise<string>;
  invoke(input, config): Promise<BaseMessage>;
}

// 文本补全模型基类
class BaseLLM extends BaseLanguageModel {
  abstract _generate(prompts, options): Promise<LLMResult>;
  _call(prompt, options): Promise<string>;
  invoke(input, config): Promise<string>;
}
```

## 扩展组件

扩展组件提供更高层次的抽象和特定功能。

### LangGraph

状态图编排框架：

```typescript
import { StateGraph, END, START } from "@langchain/langgraph";
import { Annotation } from "@langchain/langgraph";

// 定义状态
const StateAnnotation = Annotation.Root({
  messages: Annotation<Array<{ role: string; content: string }>>({
    reducer: (prev, next) => [...prev, ...next],
  }),
  currentStep: Annotation<string>,
});

// 创建图
const graph = new StateGraph(StateAnnotation)
  .addNode("agent", agentNode)
  .addNode("tools", toolsNode)
  .addEdge(START, "agent")
  .addConditionalEdges("agent", shouldContinue)
  .addEdge("tools", "agent")
  .compile();
```

## 组件交互

### 组件组合模式

```typescript
import { RunnableSequence, RunnablePassthrough } from "@langchain/core/runnables";

// 简单管道
const chain = prompt.pipe(model).pipe(parser);

// 序列组合
const sequence = RunnableSequence.from([prompt, model, parser]);

// 带数据传递的组合
const chainWithDocs = RunnableSequence.from([
  {
    question: new RunnablePassthrough(),
    context: retriever,
  },
  answerPrompt,
  model,
  parser,
]);
```

### 中间件模式

```typescript
import { RunnableLambda } from "@langchain/core/runnables";

// 使用 RunnableLambda 添加自定义逻辑
const preprocessing = RunnableLambda.from(async (input) => {
  // 预处理逻辑
  return { ...input, processed: true };
});

const postprocessing = RunnableLambda.from(async (output) => {
  // 后处理逻辑
  return output.trim();
});

const chain = preprocessing.pipe(prompt).pipe(model).pipe(postprocessing);
```

## 最佳实践

### 选择合适的组件

| 场景 | 推荐组件 |
|:---|:---|
| 简单的 LLM 调用 | `ChatModel` + `PromptTemplate` |
| 多轮对话 | `ChatModel` + `MessagesPlaceholder` |
| RAG 应用 | `Retriever` + `ChatModel` |
| 工具调用 | `Agent` + `Tools` |
| 复杂工作流 | `LangGraph` |

### 组件组合建议

```typescript
// ✅ 好的做法：明确的组件分离
const prompt = ChatPromptTemplate.fromMessages([...]);
const model = new ChatOpenAI({...});
const parser = new StringOutputParser();
const chain = prompt.pipe(model).pipe(parser);

// ❌ 不好的做法：混合关注点
const chain = async (input) => {
  const prompt = ChatPromptTemplate.fromMessages([...]);
  const model = new ChatOpenAI({...});
  const result = await model.invoke(await prompt.invoke({ input }));
  return result.content;
};
```
